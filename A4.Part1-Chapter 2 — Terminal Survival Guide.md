## 2. Beginner Explanation

Terminal ဆိုတာ computer ကို **စာရိုက်ပြီး command ပေးတဲ့နေရာ** ပါ။

သင်က computer ကို mouse နဲ့ click လုပ်ပြီး folder ဖွင့်နိုင်တယ်။ Terminal မှာတော့ folder ဖွင့်တာ၊ file ကြည့်တာ၊ project run တာတွေကို စာရိုက်ပြီးလုပ်တယ်။

ဥပမာ—

```bash
ls
```

ဆိုရင် current folder ထဲက files/folders တွေကိုပြတယ်။

```bash
cd projects
```

ဆိုရင် `projects` folder ထဲဝင်တယ်။

```bash
mkdir notes
```

ဆိုရင် `notes` ဆိုတဲ့ folder အသစ်ဆောက်တယ်။

Terminal ကို အလွယ်ပြောရရင်—

```text
Mouse နဲ့လုပ်တဲ့အလုပ်တွေကို စာရိုက်ပြီးလုပ်တဲ့နေရာ
```

ပါ။

## 3. Terminal အမျိုးအစားများ

Operating system အလိုက် terminal နာမည်ကွာနိုင်တယ်။

| System  | Terminal နာမည်                               |
| ------- | -------------------------------------------- |
| Windows | PowerShell, Command Prompt, Windows Terminal |
| macOS   | Terminal                                     |
| Linux   | Terminal                                     |
| VS Code | Integrated Terminal                          |

ဒီ course မှာ beginner-friendly ဖြစ်အောင် **VS Code Integrated Terminal** ကို အများဆုံးသုံးမယ်။ VS Code ထဲမှာ project folder ဖွင့်ထားပြီး terminal ကိုလည်းတစ်နေရာတည်းမှာသုံးနိုင်တာကြောင့် သင်ရလွယ်တယ်။

## 4. Terminal Prompt ဆိုတာဘာလဲ

Terminal ဖွင့်လိုက်ရင် အောက်ကလို စာကြောင်းတစ်ခုမြင်ရမယ်။

macOS/Linux ပုံစံ—

```bash
aung@MacBook agentic-zero-to-hero %
```

Windows PowerShell ပုံစံ—

```powershell
PS C:\Users\Aung\Desktop\agentic-zero-to-hero>
```

ဒီစာကြောင်းက terminal က command စောင့်နေတယ်ဆိုတဲ့အဓိပ္ပါယ်ပါ။

အဲဒီနောက်မှာ command ရိုက်မယ်။

ဥပမာ—

```bash
pwd
```

ပြီးရင် Enter နှိပ်မယ်။

Terminal မှာ command တစ်ခု run လုပ်တဲ့ flow က—

```text
Command ရိုက်
→ Enter နှိပ်
→ Computer ကလုပ်
→ Result ပြ
→ နောက် command စောင့်
```

ဖြစ်ပါတယ်။

# 5. Command Anatomy

Command တစ်ခုကို ခွဲကြည့်ရင် ဒီလိုပါ။

```bash
mkdir notes
```

ဒီမှာ—

```text
mkdir = command name
notes = argument / command လုပ်မယ့် target
```

အဓိပ္ပါယ်—

```text
notes ဆိုတဲ့ folder အသစ်ဆောက်ပါ
```

နောက်ဥပမာ—

```bash
cd projects
```

ဒီမှာ—

```text
cd = change directory command
projects = ဝင်ချင်တဲ့ folder name
```

အဓိပ္ပါယ်—

```text
projects folder ထဲဝင်ပါ
```

Agent က command တစ်ခုပေးတဲ့အခါ သင်မသေချာရင် ဒီလိုမေးပါ။

```text
ဒီ command က ဘာလုပ်တာလဲ beginner နားလည်အောင်ရှင်းပြပါ။
Run လုပ်ရင် file တွေပြောင်းသွားမလား?
Risk ရှိလား?
```

# 6. Essential Commands

ဒီအခန်းမှာ command ၁၂ ခုလောက်ပဲမှတ်ရင် လုံလောက်ပါတယ်။

## 6.1 Current Folder ကိုကြည့်ခြင်း

macOS/Linux/Git Bash—

```bash
pwd
```

Windows PowerShell—

```powershell
pwd
```

`pwd` ဆိုတာ **print working directory** ပါ။ လက်ရှိရောက်နေတဲ့ folder path ကိုပြတယ်။

ဥပမာ output—

```text
/Users/aung/Desktop/agentic-zero-to-hero
```

သို့မဟုတ်—

```text
C:\Users\Aung\Desktop\agentic-zero-to-hero
```

သိထားရမယ့်အချက်—

```text
Terminal မှာ command run လုပ်တာက current folder ထဲမှာလုပ်တာဖြစ်တယ်။
```

ဥပမာ `npm run dev` ကို project folder ထဲမှာ run မှ အလုပ်လုပ်မယ်။ Desktop ပေါ်မှာ run လိုက်ရင် error တက်နိုင်တယ်။

# 11. Copy / Paste in Terminal

Terminal ထဲ copy/paste လုပ်တာ operating system အလိုက်ကွာနိုင်တယ်။

```text
Windows PowerShell:
Ctrl + C = sometimes copy or cancel command
Ctrl + V = paste

macOS Terminal:
Cmd + C = copy
Cmd + V = paste

Linux Terminal:
Ctrl + Shift + C = copy
Ctrl + Shift + V = paste
```

Terminal မှာ `Ctrl + C` က တစ်ခါတစ်လေ **running command ကိုရပ်ရန်** ဖြစ်တယ်။ ဥပမာ development server run နေတဲ့အခါ `Ctrl + C` နှိပ်ရင် server ရပ်မယ်။

# 13. Safe Commands vs Risky Commands

Terminal command တွေမှာ safe command နဲ့ risky command ရှိတယ်။

## 13.1 Beginner-safe Commands

အောက်ပါ command တွေက များသောအားဖြင့် safe ပါ။

```text
pwd
ls
dir
cd
mkdir
touch
New-Item
cat
Get-Content
clear
cls
code .
```

ဒါတွေက file/folder ကြည့်တာ၊ ဝင်တာ၊ ဆောက်တာ၊ ဖတ်တာတွေပါ။

## 13.2 Caution Commands

အောက်ပါ command တွေက run မလုပ်ခင် နားလည်ရမယ်။

```text
npm install
npm update
git reset
git clean
rm
del
rmdir
sudo
chmod
curl
wget
deploy commands
database migration commands
```

## 13.3 Dangerous Commands

အောက်ပါ command တွေကို beginner အနေနဲ့ မျက်စိမှိတ်မ run ပါနဲ့။

```bash
rm -rf
sudo rm -rf
git reset --hard
git clean -fd
curl ... | bash
drop database
delete from ...
```

Agent က ဒီ command တွေ run ခိုင်းရင် အမြဲမေးပါ။

```text
ဒီ command က အန္တရာယ်ရှိနိုင်လား?
ဘာဖျက်မလဲ?
ပြန်ဆယ်လို့ရလား?
Git commit ရှိလား?
Dry-run ရှိလား?
```

Course rule—

```text
Destructive command မ run ခင် approval လိုမယ်။
```

# 22. Chapter Summary

ဒီအခန်းမှာ terminal ကို beginner-friendly အနေနဲ့လေ့လာခဲ့တယ်။

အရေးကြီးဆုံး commands—

```text
pwd        → current folder ကြည့်
ls / dir   → files/folders ကြည့်
cd         → folder ဝင်/ထွက်
cd ..      → parent folder ပြန်တက်
mkdir      → folder ဆောက်
touch      → file ဆောက်, macOS/Linux/Git Bash
New-Item   → file ဆောက်, PowerShell
cat        → file content ကြည့်
clear/cls  → terminal screen ရှင်း
code .     → current folder ကို VS Code နဲ့ဖွင့်
```

အရေးကြီးဆုံး rule—

```text
Command မ run ခင် pwd + ls နဲ့ current folder စစ်ပါ။
```

Agentic Coding အတွက် takeaway—

```text
Terminal command ကို မျက်စိမှိတ်မ run ပါနဲ့။
Command က ဘာလုပ်တာလဲ နားလည်အောင် အရင်မေးပါ။
Risky command ဆို approval, backup, Git commit, dry-run စစ်ပါ။
```
