## 2. Beginner Explanation

Internet ကို အလွယ်ဆုံးပြောရရင်—

```text
ကမ္ဘာတစ်ဝန်းက computer/server/device တွေ အချင်းချင်းဆက်သွယ်နိုင်တဲ့ network ကြီး
```

ပါ။

သင်က browser မှာ website တစ်ခုဖွင့်တဲ့အခါ နောက်ကွယ်မှာ ဒီလိုဖြစ်တယ်။

```text
သင့် Browser
   ↓
Internet
   ↓
Website Server
   ↓
Response ပြန်လာ
   ↓
Browser က webpage အဖြစ်ပြ
```

ဥပမာ သင်က event website တစ်ခုဖွင့်တယ်ဆိုပါစို့။

```text
1. Browser မှာ URL ရိုက်တယ်
2. Browser က server ကို request ပို့တယ်
3. Server က HTML/CSS/JavaScript/data ပြန်ပေးတယ်
4. Browser က user မြင်ရတဲ့ page အဖြစ်ပြတယ်
```

ဒီ course ရဲ့ capstone project ဖြစ်တဲ့ Event Management Platform မှာလည်း ဒီ flow ပဲသုံးမယ်။

```text
Public Visitor → Browser → Public Website → Backend API → Database
```

# 6. URL ဆိုတာဘာလဲ

URL ဆိုတာ website/web page ရဲ့လိပ်စာပါ။

ဥပမာ—

```text
https://example.com/events/123/register
```

ဒီ URL ကိုခွဲကြည့်ရင်—

```text
https://      → protocol
example.com   → domain
/events/123/register → path
```

## 6.1 Protocol

```text
http://
https://
```

`https://` က encrypted connection ဖြစ်လို့ production website တွေမှာ https သုံးသင့်တယ်။

Beginner အတွက်မှတ်ရန်—

```text
https = more secure
http  = not encrypted
```

## 6.2 Domain

```text
example.com
my-event-platform.com
```

Domain ဆိုတာ လူတွေမှတ်ရလွယ်တဲ့ website name ပါ။

## 6.3 Path

```text
/events
/events/123
/events/123/register
/admin/events
```

Path က website ထဲက page/route ကိုညွှန်တယ်။

Event Management project မှာ path တွေဒီလိုရှိနိုင်တယ်။

```text
/                         → homepage
/events                   → event list
/events/123               → event detail
/events/123/register      → registration form
/admin                    → admin dashboard
/checkin                  → PWA check-in app
```

# 8. HTTP / HTTPS ဆိုတာဘာလဲ

HTTP ဆိုတာ browser နဲ့ server ကြားမှာ request/response ပို့တဲ့ communication rule ပါ။

Browser က server ကို မေးတယ်—

```text
GET /events
```

Server ကပြန်ပေးတယ်—

```text
200 OK
events data
```

HTTPS က HTTP ကို secure/encrypted connection နဲ့သုံးတာပါ။

Beginner အတွက် အဓိကမှတ်ရန်—

```text
HTTP/HTTPS = Browser နဲ့ Server စကားပြောတဲ့နည်းလမ်း
```

# 9. Request / Response

Web app အားလုံးရဲ့ အဓိက flow က request/response ပါ။

```text
Request  = client က server ကိုမေး/ပို့တာ
Response = server က client ကိုပြန်ပေးတာ
```

ဥပမာ Event list ကြည့်ခြင်း—

```text
Request:
GET /api/events

Response:
[
  {
    "id": 1,
    "title": "AI Workshop",
    "date": "2026-08-01"
  }
]
```

ဥပမာ Registration form submit လုပ်ခြင်း—

```text
Request:
POST /api/events/1/register

Body:
{
  "name": "Aung Aung",
  "phone": "09123456789",
  "email": "aung@example.com"
}

Response:
{
  "success": true,
  "ticketCode": "EVT-2026-0001"
}
```

ဒီ flow ကိုနားလည်ရင် frontend/backend ချိတ်တာကိုနားလည်လာမယ်။

# 10. HTTP Methods

HTTP request တွေမှာ method ရှိတယ်။ Method ဆိုတာ “ဘာလုပ်ချင်လဲ” ကိုပြောတာပါ။

| Method | အဓိပ္ပါယ်                    | Example                |
| ------ | ---------------------------- | ---------------------- |
| GET    | data တောင်း                  | Event list ကြည့်       |
| POST   | data အသစ်ပို့/ဖန်တီး         | Registration submit    |
| PUT    | data အပြည့် update           | Event edit             |
| PATCH  | data တစ်စိတ်တစ်ပိုင်း update | Check-in status update |
| DELETE | data ဖျက်                    | Event delete           |

Event Management project မှာ—

```text
GET    /api/events
GET    /api/events/:id
POST   /api/events/:id/register
POST   /api/check-in
PUT    /api/admin/events/:id
DELETE /api/admin/events/:id
```

လို endpoint တွေသုံးမယ်။

Beginner အတွက် အစပိုင်းမှာ `GET` နဲ့ `POST` ကိုအရင်နားလည်ရင်လုံလောက်တယ်။

# 11. Status Code ဆိုတာဘာလဲ

Server response မှာ status code ပါတတ်တယ်။ Status code က request အောင်မြင်လား၊ error ဖြစ်လားကိုပြတယ်။

အများဆုံးတွေ့မယ့် status code တွေ—

| Status | အဓိပ္ပါယ်                              |
| ------ | -------------------------------------- |
| 200    | OK / အောင်မြင်                         |
| 201    | Created / အသစ်ဖန်တီးပြီး               |
| 400    | Bad Request / user ပို့တဲ့ data မှား   |
| 401    | Unauthorized / login မဝင်ထား           |
| 403    | Forbidden / permission မရှိ            |
| 404    | Not Found / မတွေ့                      |
| 409    | Conflict / duplicate data လို conflict |
| 422    | Validation Error                       |
| 429    | Too Many Requests / rate limit         |
| 500    | Server Error                           |

ဥပမာ—

```text
Registration form မှာ phone မထည့်ဘဲ submit လုပ်တယ်
→ 422 Validation Error
```

```text
Admin မဟုတ်သူက admin page ဝင်ချင်တယ်
→ 403 Forbidden
```

```text
Event မရှိတဲ့ ID ကိုဖွင့်တယ်
→ 404 Not Found
```

နောက်ပိုင်း debugging မှာ status code ဖတ်တတ်ဖို့အရေးကြီးတယ်။

# 18. Port ဆိုတာဘာလဲ

Port ဆိုတာ computer/server တစ်ခုထဲမှာ service အမျိုးမျိုးကိုခွဲပေးတဲ့ number ပါ။

ဥပမာ—

```text
http://localhost:5173
```

ဒီမှာ `5173` က port number ပါ။

```text
Frontend dev server → localhost:5173
Backend API server  → localhost:3000
Database server     → localhost:5432
```

တူညီတဲ့ computer ထဲမှာ service အများကြီး run နိုင်တယ်။ Port number က ဘယ် service ကိုသွားမလဲ ခွဲပေးတယ်။

ဥပမာ—

```text
localhost:5173 → React frontend
localhost:3000 → Backend API
```

Beginner error တစ်ခု—

```text
Port already in use
```

အဓိပ္ပါယ်—

```text
ဒီ port ကို တခြား process တစ်ခုသုံးနေပြီးသား
```

နောက်ပိုင်း debug guide ထဲမှာပိုရှင်းမယ်။

# 28. Common Errors

## Error 1 — Frontend နဲ့ Backend မခွဲနိုင်ခြင်း

မကောင်းတဲ့နားလည်မှု—

```text
Registration feature ဆိုတာ form လုပ်ရုံပဲ။
```

တကယ်တော့—

```text
Frontend:
Form UI, validation message, submit button

Backend:
API endpoint, validation, duplicate check, ticket code generation

Database:
Attendee data save
```

ပြင်နည်း—

```text
Feature တစ်ခုကို frontend / backend / database task အဖြစ်ခွဲပါ။
```

---

## Error 2 — API ဆိုတာ website page လို့ထင်ခြင်း

API endpoint က user ကြည့်တဲ့ page မဟုတ်ပါဘူး။ Software systems တွေ data ပို့/ယူတဲ့ interface ပါ။

```text
Page:
/events

API:
/api/events
```

Browser မှာ `/events` ကိုဖွင့်ရင် user-friendly page ပေါ်မယ်။ `/api/events` ကိုခေါ်ရင် JSON data ပြန်လာမယ်။

---

## Error 3 — Localhost ကို internet ပေါ်တင်ပြီးသားလို့ထင်ခြင်း

```text
http://localhost:5173
```

ဆိုတာ ကိုယ့် computer ထဲမှာပဲ run နေတဲ့ app ဖြစ်တယ်။

အခြားလူတွေကြည့်ချင်ရင် deploy လုပ်ရမယ်။

```text
Localhost = local development
Deployment = public/staging/production environment
```

---

## Error 4 — Status code မဖတ်ခြင်း

Network request error ဖြစ်ရင် status code ကိုကြည့်ရမယ်။

```text
404 = endpoint/page မတွေ့
422 = validation error
500 = backend server error
```

Status code မကြည့်ဘဲ “မရဘူး” လို့ပဲပြောရင် Agent က diagnosis လုပ်ရခက်တယ်။

ကောင်းတဲ့ bug report—

```text
POST /api/events/1/register request က 422 ပြန်ပါတယ်။
Response body မှာ phone is required လို့ပြပါတယ်။
Frontend form မှာ phone value ပို့နေ/မနေ စစ်ပေးပါ။
```

---

## Error 5 — Database ကို frontend ကတိုက်ရိုက်ထိချင်ခြင်း

Frontend က database ကိုတိုက်ရိုက်မထိသင့်ပါဘူး။

မကောင်းတဲ့ model—

```text
Frontend → Database
```

ကောင်းတဲ့ model—

```text
Frontend → Backend API → Database
```

ဒီ separation က security, validation, permissions အတွက်လိုတယ်။

---

## Error 6 — Backend error ကို frontend UI bug လို့ထင်ခြင်း

User က button နှိပ်ပြီး error မြင်ရင် frontend UI bug လို့ထင်နိုင်တယ်။ တကယ်တော့—

