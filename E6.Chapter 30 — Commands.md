## 2. Beginner Explanation

Command ဆိုတာ Agent session ထဲမှာ အလုပ်တစ်ခုကို အမြန်လုပ်ခိုင်းတဲ့ shortcut ပါ။

ဥပမာ Claude Code session ထဲမှာ—

```text
/plan Registration History feature အတွက် implementation plan ပေးပါ။
```

လို့ရိုက်ရင် Claude ကို plan mode workflow နဲ့စေခိုင်းတာပါ။

နောက်ဥပမာ—

```text
/context
```

လို့ရိုက်ရင် current context usage ကိုစစ်ဖို့သုံးနိုင်တယ်။

```text
/compact focus on current task, decisions, changed files, and next safe step
```

လို့ရိုက်ရင် long session ကို summary လုပ်ပြီး context window နေရာလွတ်အောင်လုပ်နိုင်တယ်။

အလွယ်ဆုံး mental model—

```text
Prompt
= တစ်ခါသုံးအမိန့်စာ

Skill
= reusable workflow package

Command
= workflow / tool control / skill ကို အမြန်ခေါ်တဲ့ slash shortcut

Hook
= lifecycle checkpoint မှာ auto-run guardrail

Subagent
= specialist worker role
```

Claude Code docs က Commands ကို Claude Code session ထဲမှာ model switch, permissions management, context clear, workflow run စတာတွေကိုအမြန်လုပ်နိုင်တဲ့ session control mechanism အဖြစ်ဖော်ပြထားပြီး `/` ရိုက်တာနဲ့ available command list ကို filter/ရွေးနိုင်ကြောင်းရှင်းပြထားပါတယ်။ Command name ရဲ့နောက်က text ဟာ command arguments အဖြစ်သတ်မှတ်နိုင်ပါတယ်။ ([Claude][1])

# 3. Command ဆိုတာဘာလဲ

Command ဆိုတာ interactive Agent session ထဲမှာ `/` နဲ့စတဲ့ special instruction ပါ။

ဥပမာ—

```text
/help
/plan
/context
/compact
/clear
/permissions
/hooks
/diff
/code-review
```

Command တွေကို အမျိုးအစား ၃ ခုခွဲနိုင်တယ်။

```text
1. Built-in commands
2. Bundled skill commands
3. Custom commands / custom skills
```

## 3.1 Built-in Commands

Tool ကပေးထားတဲ့ fixed behavior commands ပါ။

ဥပမာ—

```text
/context
/compact
/clear
/permissions
/hooks
/model
```

ဒီ command တွေက session control အတွက်သုံးတယ်။

## 3.2 Bundled Skill Commands

Tool က bundle လုပ်ပေးထားတဲ့ skill-based workflows ပါ။

ဥပမာ Claude Code မှာ—

```text
/code-review
/debug
/batch
/loop
/run
/verify
```

လို bundled skills တွေရှိနိုင်တယ်။ Claude docs က bundled skills တွေကို built-in commands reference ထဲမှာပြထားပြီး command ပုံစံနဲ့ invoke လုပ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude][2])

## 3.3 Custom Commands / Custom Skills

ကိုယ်တိုင်ရေးထားတဲ့ reusable workflows ကို `/name` နဲ့ invoke လုပ်တာပါ။

ယနေ့ Claude Code docs အရ custom commands concept ကို Skills ထဲပေါင်းထားပြီး `.claude/commands/deploy.md` legacy file နဲ့ `.claude/skills/deploy/SKILL.md` skill နှစ်ခုလုံးက `/deploy` command အဖြစ်အလုပ်လုပ်နိုင်ပါတယ်။ Existing `.claude/commands/` files တွေက ဆက်အလုပ်လုပ်ပေမယ့် new reusable workflows အတွက် Skill format ကိုဦးစားပေးသင့်ပါတယ်။ ([Claude][2])

# 4. Commands ဘာကြောင့်လိုလဲ

Commands လိုတဲ့အကြောင်းရင်း—

```text
1. Repeated workflow ကို အမြန်ခေါ်နိုင်
2. Long prompt copy-paste လုပ်စရာမလို
3. Session control ကို keyboard-first လုပ်နိုင်
4. Review, debug, verify, compact, clear စတဲ့ workflow တွေ consistency ရှိ
5. Command arguments နဲ့ dynamic task ပေးနိုင်
6. Team/project workflow ကို standardized လုပ်နိုင်
```

ဥပမာ သင် review prompt တစ်ခုကို အကြိမ်ကြိမ်သုံးနေရတယ်ဆိုပါစို့။

```text
Code မပြင်ပါနဲ့။
git diff ကို review လုပ်ပါ။
scope, secrets, package changes, verification gaps စစ်ပါ။
PASS/WARNING/FAIL ပေးပါ။
```

ဒီကို `diff-review` skill/command အဖြစ်ရေးထားရင်—

```text
/diff-review Chapter 30 lab files only
```

လို့အမြန်ခေါ်နိုင်တယ်။

Command ကောင်းတစ်ခုဟာ—

```text
short to invoke
clear in purpose
safe by default
structured in output
approval-gated for risky actions
```

ဖြစ်သင့်တယ်။

# 5. Command vs Skill

Chapter 29 မှာ Skill ကို reusable workflow package အဖြစ်သင်ခဲ့တယ်။ Chapter 30 မှာ Command ကို invocation interface အဖြစ်မြင်ပါ။

| အချက်                         | Skill                           | Command                                  |
| ----------------------------- | ------------------------------- | ---------------------------------------- |
| Meaning                       | reusable instruction package    | slash shortcut / invocation mechanism    |
| File                          | `SKILL.md`                      | built-in command or skill/legacy command |
| Example                       | `diff-review` skill             | `/diff-review`                           |
| Can auto-trigger?             | Yes, if description matches     | Usually explicitly typed                 |
| Best for                      | reusable procedure              | quick session action / workflow call     |
| Current Claude recommendation | use skills for custom workflows | skills create slash commands too         |

Claude Code skills docs က skill directory name က command name ဖြစ်နိုင်ကြောင်း၊ `.claude/skills/deploy-staging/SKILL.md` က `/deploy-staging` အဖြစ် invoke ဖြစ်နိုင်ကြောင်းရှင်းပြထားပါတယ်။ Legacy `.claude/commands/deploy.md` ကလည်း `/deploy` အဖြစ်အလုပ်လုပ်နိုင်ပေမယ့် skill နဲ့ command name တူရင် skill က precedence ရပါတယ်။ ([Claude][2])

Course rule—

```text
New reusable workflow
→ Skill အဖြစ်ရေးပါ။

User က slash shortcut နဲ့ခေါ်မယ်
→ အဲဒီ Skill က command အဖြစ်လည်းအလုပ်လုပ်မယ်။

Old .claude/commands/ files
→ နားလည်ထားပါ, but new work မှာ skills ကိုဦးစားပေးပါ။
```

# 6. Built-in Commands — Claude Code

Claude Code docs က command workflow ကို session phase အလိုက်ဖော်ပြထားပါတယ်။ Repo ပထမဆုံးစချိန်မှာ `/init`, `/memory`, `/mcp`, `/permissions` သုံးနိုင်ပြီး task အတွင်းမှာ `/plan`, `/model`, `/effort`, `/context`, `/compact`, `/btw` စတာတွေသုံးနိုင်တယ်။ Ship မလုပ်ခင် `/diff`, `/code-review`, `/review`, `/security-review` စတဲ့ review commands တွေသုံးနိုင်ပြီး session အကြား `/clear`, `/resume`, `/branch` တို့သုံးနိုင်ကြောင်း docs မှာဖော်ပြထားပါတယ်။ ([Claude][1])

Beginner အတွက် အရေးကြီးဆုံး Claude Code commands—

```text
/init
/memory
/permissions
/hooks
/plan
/context
/compact
/clear
/diff
/code-review
/security-review
```

## 6.1 `/init`

Project ထဲ starter `CLAUDE.md` ဖန်တီး/အကြံပြုဖို့သုံးနိုင်တယ်။

Use when—

```text
Existing repo ထဲ Claude Code စသုံးမယ်
Project memory scaffold လိုတယ်
```

Rule—

```text
Generated CLAUDE.md ကိုမဖတ်ဘဲ commit မလုပ်ပါနဲ့။
```

## 6.2 `/memory`

Loaded memory files စစ်ရန်သုံးတယ်။

Use when—

```text
CLAUDE.md loaded ဖြစ်/မဖြစ်စစ်မယ်
Rule မလိုက်နာတဲ့ပြဿနာ debug မယ်
```

## 6.3 `/permissions`

Permission rules စစ်/ပြင်ရန်သုံးတယ်။

Use when—

```text
Agent ဘာ tool/command လုပ်ခွင့်ရှိလဲစစ်မယ်
.env / destructive command protection စစ်မယ်
```

## 6.4 `/hooks`

Configured hooks တွေစစ်ရန်သုံးတယ်။

Use when—

```text
PreToolUse / PostToolUse hooks စစ်မယ်
settings.json ပြင်ပြီး verify မယ်
```

## 6.5 `/plan`

Code မပြင်ခင် plan mode workflow သုံးရန်။

Use when—

```text
Feature-level task
Multiple files
Unfamiliar code
Risky changes
```

## 6.6 `/context`

Context window usage စစ်ရန်။

Use when—

```text
Session ကြာလာ
Agent quality ကျလာ
Compact/Clear ဆုံးဖြတ်မယ်
```

## 6.7 `/compact`

Long session ကို summary လုပ်ရန်။

Use when—

```text
Same task ဆက်လုပ်မယ်
Context window နေရာလွတ်ချင်
```

## 6.8 `/clear`

Fresh chat စရန်။

Use when—

```text
Unrelated task အသစ်
Context rot ဖြစ်နေ
Rejected plans များ
```

## 6.9 `/diff`

Current changes ကိုကြည့်/review အတွက်သုံးနိုင်တယ်။

Use when—

```text
Commit မလုပ်ခင်
Agent ပြင်ထားတဲ့ files စစ်မယ်
```

## 6.10 `/code-review`

Current diff ကို review လုပ်ရန်သုံးနိုင်တယ်။ Claude docs မှာ `/code-review` သည် diff correctness bugs/cleanups စစ်နိုင်ပြီး `--fix` option နဲ့ findings apply လုပ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ Beginner course မှာတော့ auto-fix မလုပ်ခင် human review gate ထည့်ပါ။ ([Claude][1])

# 7. Built-in Slash Commands — Codex CLI

Codex CLI docs က slash commands တွေကို interactive terminal session ထဲမှာ model, permissions, context summary စတာတွေကို terminal မထွက်ဘဲထိန်းနိုင်တဲ့ keyboard-first controls အဖြစ်ဖော်ပြထားပါတယ်။ `/` ရိုက်ပြီး command popup ကိုဖွင့်နိုင်ပြီး task running ဖြစ်နေချိန်မှာ command ကို queue လုပ်နိုင်ပါတယ်။ ([OpenAI Developers][3])

Docs မှာ Codex CLI commands အချို့အဖြစ်—

```text
/permissions
/ide
/keymap
/vim
/sandbox-add-read-dir
/agent
/apps
/plugins
/hooks
```

တို့ကိုဖော်ပြထားပါတယ်။ `/permissions` က approval requirements tighten/relax လုပ်ဖို့, `/ide` က open files/current selection/editor context ကို next prompt ထဲထည့်ဖို့, `/agent` က spawned subagent thread စစ်/ဆက်လုပ်ဖို့, `/hooks` က lifecycle hooks ကြည့်/စီမံဖို့သုံးနိုင်ကြောင်း docs မှာဖော်ပြထားပါတယ်။ ([OpenAI Developers][3])

Codex CLI မှာ context/session control အတွက် `/compact`, `/clear`, `/new`, `/fork`, `/side`, `/diff`, `/mention`, `/skills` စတဲ့ workflow commands တွေလည်း Chapter 28 မှာလေ့လာခဲ့သလို အသုံးဝင်ပါတယ်။ Codex slash command docs က `/compact` ကို long conversation summarize လုပ်ရန်, `/clear` ကို fresh chat စရန်, `/new` ကို current CLI session ထဲ fresh conversation စရန်, `/fork` ကို current conversation clone လုပ်ပြီး alternative approach စမ်းရန်, `/side` ကို ephemeral side conversation အဖြစ်သုံးရန်ဖော်ပြထားပါတယ်။ ([OpenAI Developers][3])

# 8. Codex Custom Prompts — Deprecated

Codex မှာ အရင်က `~/.codex/prompts/*.md` custom prompts ကို slash command အဖြစ်သုံးနိုင်တဲ့ feature ရှိခဲ့တယ်။ ဒါပေမယ့် official Codex docs က Custom Prompts ကို deprecated လုပ်ထားပြီး reusable prompts/workflows အတွက် Skills ကိုဦးစားပေးသုံးရန်ဖော်ပြထားပါတယ်။ Custom prompts သည် explicit invocation လိုပြီး local Codex home directory ထဲမှာသာနေတဲ့အတွက် repo နဲ့ share မဖြစ်နိုင်ဘဲ, repo-shareable or implicit invocation လိုချင်ရင် Skills သုံးသင့်ပါတယ်။ ([OpenAI Developers][4])

Legacy concept နားလည်ရန်—

```text
~/.codex/prompts/draftpr.md
→ /prompts:draftpr
```

Custom prompt metadata ထဲမှာ—

```yaml
description:
argument-hint:
```

ထည့်နိုင်ပြီး `$1`–`$9`, `$ARGUMENTS`, named placeholders စတာတွေသုံးနိုင်ကြောင်း docs မှာဖော်ပြထားပါတယ်။ ([OpenAI Developers][4])

Course rule—

```text
Codex custom prompts ကို historical/legacy concept အဖြစ်သိထားပါ။
New reusable workflow အတွက် .agents/skills/<name>/SKILL.md ကိုသုံးပါ။
```

# 9. Command Arguments

Command တွေမှာ arguments ပေးနိုင်တယ်။

Claude Code skills docs က `$ARGUMENTS` placeholder ကို skill content ထဲသုံးနိုင်ပြီး `/fix-issue 123` လို့ invoke လုပ်ရင် `123` ဟာ skill body ထဲက `$ARGUMENTS` နေရာမှာ substitute ဖြစ်ကြောင်းဖော်ပြထားပါတယ်။ `$ARGUMENTS` မပါရင် Claude Code က arguments ကို skill content အဆုံးမှာ append လုပ်ပေးနိုင်တယ်။ Claude Code v2.1.199 မှာ `/skill-a /skill-b do XYZ` လို skill chaining လုပ်နိုင်ပြီး up to six skills ကို အစပိုင်းမှာ stack လုပ်နိုင်ကြောင်း docs မှာဖော်ပြထားပါတယ်။ ([Claude][2])

Example skill—

```md
---
name: review-path
description: Review a specific path for scope, secrets, package changes, and verification gaps.
argument-hint: [path]
---

Review this path: $ARGUMENTS

Use review-only mode.
Do not edit files.

Output:
- Decision
- Findings
- Suggested next step
```

Invocation—

```text
/review-path playground/chapter-30-commands/
```

Result—

```text
Review this path: playground/chapter-30-commands/
```

Arguments use cases—

```text
/diff-review Chapter 30 lab files only
/bug-report Register button မရဘူး
/context-pack Registration History feature
/review-path projects/event-management/apps/web/
/fix-issue 123
```

# 10. Command Chaining

Claude Code docs အရ skill invocation တွေကို message အစမှာ stack လုပ်နိုင်ပြီး `/code-review /fix-issue 123` လိုရေးရင် skill နှစ်ခုလုံး load ဖြစ်ပြီး trailing argument ကို `$ARGUMENTS` အဖြစ်ရနိုင်ပါတယ်။ Up to six skills chained ဖြစ်နိုင်ကြောင်း docs ကဖော်ပြထားပါတယ်။ ([Claude][2])

Use carefully.

Good chaining—

```text
/context-pack /diff-review Chapter 30 command lab changes
```

Possible meaning—

```text
context-pack skill + diff-review skill နှစ်ခုလုံး load
trailing text ကို context/task argument အဖြစ်သုံး
```

Beginner caution—

```text
Command chaining ကို အစပိုင်းမှာမသုံးလည်းရပါတယ်။
One command at a time သုံးတာပိုရှင်းတယ်။
```

Course rule—

```text
Beginner:
one command at a time

Builder:
chain only when output interaction is clear

Production:
prefer explicit workflow and review
```

# 11. Command vs Prompt Template

Command နဲ့ prompt template ကဆင်တူပေမယ့် မတူပါ။

| အချက်            | Prompt Template     | Command                         |
| ---------------- | ------------------- | ------------------------------- |
| How used         | copy-paste          | `/name args`                    |
| Storage          | resources/prompts/  | skill/command folder            |
| Invocation       | manual paste        | slash shortcut                  |
| Reuse level      | medium              | high                            |
| Best for         | learning / drafting | repeated workflow               |
| Tool integration | no                  | yes, if skill/command supported |

Prompt template example—

```text
resources/prompts/diff-review.md
```

Command example—

```text
.claude/skills/diff-review/SKILL.md
```

Course strategy—

```text
Learning stage:
resources/prompts/ ထဲ prompt templates စု

Repeated practical workflow:
.claude/skills/ ထဲ skill-command အဖြစ်ပြောင်း
```

# 12. Command vs Hook

Command ကို user/agent က invoke လုပ်တယ်။ Hook က lifecycle event တစ်ခုဖြစ်ရင် auto-run လုပ်တယ်။

| အချက်        | Command                    | Hook                               |
| ------------ | -------------------------- | ---------------------------------- |
| Trigger      | User types `/command`      | Tool event happens                 |
| Example      | `/diff-review`             | PreToolUse blocks `rm -rf`         |
| Purpose      | workflow shortcut          | deterministic guardrail/automation |
| User control | explicit                   | automatic                          |
| Best for     | review/debug/plan workflow | safety/logging/format after edit   |

ဥပမာ—

```text
/diff-review
= user wants review now

PreToolUse guard
= any time Agent tries dangerous Bash, block automatically
```

Rule—

```text
Need human-triggered reusable workflow → Command/Skill
Need always-on safety → Hook/Permission
```

# 13. Command vs Subagent

Command သည် workflow shortcut ဖြစ်တယ်။ Subagent သည် role/work context ဖြစ်တယ်။

```text
/diff-review
= review workflow command

verifier subagent
= independent role that performs verification
```

Command က subagent ကို invoke လုပ်ဖို့ညွှန်နိုင်တယ်။

Example—

```md
# .claude/skills/verify-feature/SKILL.md

Use a verifier subagent if available.
Verify acceptance criteria only.
Do not edit code.
```

Course rule—

```text
Command = what workflow
Subagent = who performs it
```

# 14. Command Safety Principles

Command တစ်ခုဖန်တီးတဲ့အခါ safety ကိုအရင်စဉ်းစားပါ။

Safe command rules—

```text
[ ] Review-only command များသည် code မပြင်ရ
[ ] Plan command များသည် edit မလုပ်ရ
[ ] Deploy command များသည် default planning-only ဖြစ်ရ
[ ] Package install command များ approval gate ပါရ
[ ] Git commit command များ human approval မရှိဘဲ commit မလုပ်ရ
[ ] Secret values မဖတ်/မဖော်ပြရ
[ ] Output format တိကျရ
[ ] Scope/arguments check ပါရ
```

Dangerous command smell—

```text
Run everything automatically.
Commit and push.
Deploy production.
Install whatever needed.
Delete old files.
```

Safe replacement—

```text
Plan only.
Ask for approval.
Review diff.
Provide exact commands but do not run.
```

```text
[ ] Output format ရှိလား?
[ ] Safety rules ပါလား?

[ ] Supporting templates/files လိုလား?
```

Bad names—


Naming rule—

```text

lowercase
hyphen separated
single purpose
```

# 17. Command Description

Description က command menu / skill trigger အတွက်အရေးကြီးတယ်။

Good description—

```yaml
description: Review current git diff before commit. Use when the user asks what changed, wants commit readiness, or requests risk review of uncommitted changes.
```

Bad description—

```yaml
description: Review things.
```

Description ရေးနည်း—

```text
[Action] + [target] + [trigger situation] + [boundary]
```

Example—

```yaml
description: Create a structured bug report prompt from vague error details. Use when the user says something is broken, failing, not working, or gives incomplete bug information. Do not edit code.
```

# 28. Command Debugging

## Command `/` menu မှာမပေါ်

စစ်ရန်—

```text
[ ] folder path မှန်လား
[ ] SKILL.md ရှိလား
[ ] session restart လိုလား
[ ] skill directory existed when session started? Claude live detection caveat
[ ] command name conflict ရှိလား
[ ] user-invocable false ဖြစ်နေသလား
[ ] skillOverrides off ဖြစ်နေသလား
```

Claude Code docs က top-level skills directory session စပြီးနောက်မှအသစ်ဖန်တီးခဲ့ရင် restart လိုနိုင်ပြီး, existing skill directories ထဲက `SKILL.md` text changes ကို live detection လုပ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude][2])

## Command argument မဝင်

စစ်ရန်—

```text
[ ] `$ARGUMENTS` သုံးထားလား
[ ] `$ARGUMENTS[0]` indexing မှန်လား
[ ] arguments quoting လိုလား
[ ] skill body ထဲ output မှာ arguments ထည့်ထားလား
```

## Command မလိုတဲ့အချိန် trigger ဖြစ်

စစ်ရန်—

```text
[ ] description too broad
[ ] disable-model-invocation လိုနိုင်
[ ] paths restriction လိုနိုင်
```

## Command code ပြင်သွား

စစ်ရန်—

```text
[ ] Mode: Review only ပါလား
[ ] Do not edit rule ပါလား
[ ] allowed-tools broad ဖြစ်လား
[ ] command purpose dangerous ဖြစ်လား
```

# 32. Common Errors

## Error 1 — Command ကို Skill နဲ့ရောပြီးမသိခြင်း

Command ဟာ invocation mechanism ဖြစ်ပြီး Skill ဟာ reusable workflow package ဖြစ်တယ်။ Modern Claude custom commands are skills. New workflows ကို `.claude/skills/<name>/SKILL.md` အဖြစ်ရေးပါ။

---

## Error 2 — Description vague ဖြစ်ခြင်း

Bad—

```yaml
description: Helps with things.
```

Good—

```yaml
description: Create manual verification steps for frontend/static labs and browser-based features.
```

---

## Error 3 — Arguments မထည့်ထားခြင်း

Command မှာ dynamic target လိုရင် `$ARGUMENTS` ထည့်ပါ။

Bad—

```md
Review the changes.
```

Good—

```md
Expected scope:
$ARGUMENTS
```

---

## Error 4 — Dangerous command ကို auto-run ခိုင်းခြင်း

Bad—

```text
Deploy production automatically.
```

Good—

```text
Create deployment plan only. Do not deploy. Ask for approval.
```

---

## Error 5 — Review command က code ပြင်ခြင်း

Review-only commands ထဲမှာ—

```text
Do not edit files.
```

မဖြစ်မနေထည့်ပါ။

---

## Error 6 — Legacy `.claude/commands/` ကို new standard လို့ထင်ခြင်း

Legacy commands work, but new reusable workflows should be skills. Claude docs က custom commands merged into skills ဖြစ်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude][2])

---

## Error 7 — Codex custom prompts ကို new workflow အတွက်သုံးခြင်း

Codex custom prompts are deprecated; use skills for reusable instructions. ([OpenAI Developers][4])

# 36. Chapter Summary

ဒီအခန်းမှာ **Commands** ကိုလေ့လာခဲ့တယ်။

အရေးကြီးဆုံး mental model—

```text
Command = Agent session ထဲမှာ workflow/control/action ကို အမြန်ခေါ်တဲ့ slash shortcut
```

Commands အမျိုးအစား—

```text
Built-in commands
= /context, /compact, /clear, /permissions, /hooks စတဲ့ session controls

Bundled skill commands
= /code-review, /debug, /run, /verify စတဲ့ tool-provided workflows

Custom skill commands
= .claude/skills/<name>/SKILL.md ကနေ /name အဖြစ်ခေါ်နိုင်တဲ့ reusable workflows

Legacy custom commands
= .claude/commands/<name>.md still work, but new work should prefer skills
```

Claude Code အတွက် အရေးကြီးတဲ့ update—

```text
Custom commands have been merged into Skills.
New reusable workflows should be written as Skills.
Existing .claude/commands files still work.
```

Codex အတွက် အရေးကြီးတဲ့ update—

```text
Slash commands control interactive sessions.
Custom prompts are deprecated.
Use Skills for reusable instructions and shareable workflows.
```

Command design မှာ အရေးကြီးဆုံး rules—

```text
Command name တိုပြီးရှင်းပါ။
Description trigger-friendly ရေးပါ။
Arguments လိုရင် $ARGUMENTS သုံးပါ။
Output format သတ်မှတ်ပါ။
Review/plan commands သည် code မပြင်ရ။
Risky actions အတွက် approval gate ထည့်ပါ။
Deploy/commit/push/package install commands ကို automatic မလုပ်ပါနဲ့။
```

Agentic Coding အတွက် takeaway—

```text
Prompt template က copy-paste workflow ဖြစ်တယ်။
Skill က reusable workflow package ဖြစ်တယ်။
Command က အဲဒီ workflow ကို အမြန်ခေါ်တဲ့ interface ဖြစ်တယ်။
```
