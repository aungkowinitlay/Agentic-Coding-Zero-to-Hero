```text
Button နှိပ်
```

```text

Setter = state ကို update လုပ်တဲ့ function

```

```jsx

```

# 3. State ဆိုတာဘာလဲ

State ဆိုတာ component ထဲမှာ **ပြောင်းလဲနိုင်ပြီး UI ကိုပြန်ပြောင်းစေမယ့် data** ပါ။

ဥပမာ—

```text
button clicked count
selected category
form input value
show/hide toggle
current event list
registration success message
loading / error status
```

Static variable နဲ့ state ကွာခြားချက်—

```jsx
let count = 0;
```

ဒီလို variable ပြောင်းတာက React ကို UI ပြန် render လုပ်ဖို့မပြောနိုင်ဘူး။

```jsx
const [count, setCount] = useState(0);
```

ဒီလို state သုံးရင် `setCount` ခေါ်တဲ့အခါ React က value အသစ်ကိုမှတ်ပြီး component ကို re-render လုပ်တယ်။ React reference docs မှာ `useState` သည် component ထဲ state variable ထည့်နိုင်တဲ့ Hook ဖြစ်ပြီး convention အနေနဲ့ `[something, setSomething]` array destructuring naming ကိုသုံးကြောင်းဖော်ပြထားပါတယ်။ ([React][2])

# 4. useState Hook

`useState` ကို React ကနေ import လုပ်ရမယ်။

```jsx
import { useState } from "react";
```

သုံးပုံ—

```jsx
const [message, setMessage] = useState("Ready");
```

အဓိပ္ပါယ်—

```text
message
= state value

setMessage
= state update function

"Ready"
= initial state
```

Example—

```jsx
function Greeting() {
  const [message, setMessage] = useState("မင်္ဂလာပါ");

  function handleClick() {
    setMessage("Button နှိပ်ပြီးပါပြီ");
  }

  return (
    <section>
      <p>{message}</p>
      <button type="button" onClick={handleClick}>
        နှိပ်ပါ
      </button>
    </section>
  );
}
```

State update flow—

```text
Initial render:
message = "မင်္ဂလာပါ"

User clicks button:
handleClick runs

setMessage("Button နှိပ်ပြီးပါပြီ"):
state updates

React re-renders:
new message appears
```

# 5. Event Handler ဆိုတာဘာလဲ

Event handler ဆိုတာ user action ဖြစ်တဲ့အခါ run မယ့် function ပါ။

User action examples—

```text
button click
input typing
form submit
mouse hover
focus
keyboard press
```

React docs က event handlers သည် clicking, hovering, focusing form inputs စတဲ့ interactions တွေဖြစ်တဲ့အခါ trigger ဖြစ်တဲ့ ကိုယ်ပိုင် functions တွေဖြစ်ပြီး JSX tag ထဲ event prop အဖြစ်ပေးနိုင်တယ်လို့ဖော်ပြထားပါတယ်။ ([React][3])

Example—

```jsx
function RegisterButton() {
  function handleClick() {
    alert("Register clicked");
  }

  return (
    <button type="button" onClick={handleClick}>
      Register လုပ်မည်
    </button>
  );
}
```

အရေးကြီး—

```jsx
onClick={handleClick}
```

လို့ရေးရမယ်။

မကောင်း—

```jsx
onClick={handleClick()}
```

ဒီလိုရေးရင် component render ဖြစ်တဲ့အချိန် function တန်း run သွားနိုင်တယ်။ User click မစောင့်တော့ဘူး။

Event handler naming convention—

```text
handleClick
handleRegister
handleToggleOpenOnly
handleSubmit
handleChange
```

React docs မှာ event handler functions များကို component ထဲမှာ define လုပ်လေ့ရှိပြီး convention အနေနဲ့ `handle` + event name pattern သုံးလေ့ရှိကြောင်းဖော်ပြထားပါတယ်။ ([React][3])

# 6. onClick Event

Button click ကိုတုံ့ပြန်ချင်ရင် `onClick` သုံးပါ။

```jsx
function Counter() {
  const [count, setCount] = useState(0);

  function handleIncrease() {
    setCount(count + 1);
  }

  return (
    <button type="button" onClick={handleIncrease}>
      Count: {count}
    </button>
  );
}
```

Inline handler သုံးနိုင်တယ်—

```jsx
<button type="button" onClick={() => setCount(count + 1)}>
  Count: {count}
</button>
```

Beginner rule—

```text
Logic တစ်ကြောင်းပဲဆို inline handler ok
Logic ၂–၃ ကြောင်းကျော်လာရင် named function ရေးပါ
```

ဒီ course lab မှာ `handleRegister`, `handleToggleOpenOnly` လို named handlers သုံးမယ်။

# 7. State Design: ဘာကို State ထဲထားမလဲ

React app မှာ state design မှားရင် bug များနိုင်တယ်။

State ထဲထားသင့်တာ—

```text
[✓] user action ကြောင့်ပြောင်းတဲ့ data
[✓] UI ကမှတ်ထားရမယ့် data
[✓] server/API response, later
[✓] form input value, later
[✓] toggle/filter selection
```

State ထဲမထားသင့်တာ—

```text
[✗] existing state ကနေတွက်လို့ရတဲ့ value
[✗] props ကနေတွက်လို့ရတဲ့ value
[✗] constant data
[✗] duplicate / redundant information
```

ဥပမာ—

```jsx
const remainingSeats = event.capacity - event.registeredCount;
```

ဒီ value ကို state ထဲမသိမ်းပါ။ `capacity` နဲ့ `registeredCount` ကနေတွက်လို့ရတယ်။

မကောင်း—

```jsx
const [remainingSeats, setRemainingSeats] = useState(75);
```

ဘာကြောင့်မကောင်းလဲ?

```text
registeredCount ပြောင်းပြီး remainingSeats မ update မိရင် data မကိုက်တော့ဘူး
```

React docs မှာ state structure ရွေးတဲ့အခါ redundant/duplicated information မထားသင့်ကြောင်း၊ မလိုအပ်တဲ့ state ရှိရင် update မေ့ပြီး bug ထည့်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([React][4])

Course rule—

```text
Store minimal state.
Derive everything else.
```

# 8. Array State Update

ဒီ chapter lab မှာ event list ကို state ထဲထားမယ်။

```jsx
const [eventList, setEventList] = useState(initialEvents);
```

Register button နှိပ်တဲ့အခါ event တစ်ခုရဲ့ `registeredCount` ကို update လုပ်မယ်။

မကောင်းတဲ့နည်း—

```jsx
event.registeredCount += 1;
setEventList(eventList);
```

ဒီဟာက existing array/object ကို mutate လုပ်တာဖြစ်တယ်။

ကောင်းတဲ့နည်း—

```jsx
setEventList((currentEvents) =>
  currentEvents.map((event) => {
    if (event.id !== eventId) {
      return event;
    }

    return {
      ...event,
      registeredCount: event.registeredCount + 1
    };
  })
);
```

React docs မှာ state ထဲရှိ arrays တွေကို read-only လိုသဘောထားသင့်ပြီး `push()` / direct assignment လို mutate လုပ်မယ့်နည်းတွေမသုံးဘဲ `map()`, `filter()` လို non-mutating methods နဲ့ array အသစ်ဖန်တီးပြီး setter ထဲပေးသင့်ကြောင်းဖော်ပြထားပါတယ်။ ([React][5])

Mental model—

```text
Old array ကိုမပြင်ပါ
New array တစ်ခုဖန်တီးပါ
Updated object ကို copy လုပ်ပါ
setEventList(newArray) လုပ်ပါ
```

# 10. Replace `src/App.jsx`

`playground/chapter-33-react-vite-starter/react-vite-starter/src/App.jsx` ကိုအောက်ပါ code နဲ့ replace လုပ်ပါ။

```jsx
import { useState } from "react";
import "./App.css";

const initialEvents = [
  {
    id: "evt-ai-workshop",
    title: "AI Workshop",
    date: "2026-08-01",
    location: "Yangon",
    category: "AI",
    capacity: 100,
    registeredCount: 25
  },
  {
    id: "evt-react-meetup",
    title: "React Meetup",
    date: "2026-08-15",
    location: "Mandalay",
    category: "Frontend",
    capacity: 80,
    registeredCount: 72
  },
  {
    id: "evt-pwa-checkin",
    title: "PWA Check-in Lab",
    date: "2026-09-05",
    location: "Online",
    category: "Mobile Web",
    capacity: 60,
    registeredCount: 60
  }
];

function App() {
  const [eventList, setEventList] = useState(initialEvents);
  const [message, setMessage] = useState("Register လုပ်ရန် event တစ်ခုကိုရွေးပါ။");
  const [showOnlyOpen, setShowOnlyOpen] = useState(false);

  const totalCapacity = eventList.reduce((total, event) => total + event.capacity, 0);
  const totalRegistered = eventList.reduce(
    (total, event) => total + event.registeredCount,
    0
  );

  const openEvents = eventList.filter(
    (event) => event.registeredCount < event.capacity
  );

  const visibleEvents = showOnlyOpen ? openEvents : eventList;

  function handleRegister(eventId) {
    const selectedEvent = eventList.find((event) => event.id === eventId);

    if (!selectedEvent) {
      setMessage("Event မတွေ့ပါ။");
      return;
    }

    const remainingSeats = selectedEvent.capacity - selectedEvent.registeredCount;

    if (remainingSeats <= 0) {
      setMessage(`${selectedEvent.title} သည် seats ပြည့်ပြီးပါပြီ။`);
      return;
    }

    setEventList((currentEvents) =>
      currentEvents.map((event) => {
        if (event.id !== eventId) {
          return event;
        }

        return {
          ...event,
          registeredCount: event.registeredCount + 1
        };
      })
    );

    setMessage(`${selectedEvent.title} အတွက် registration အောင်မြင်ပါသည်။`);
  }

  function handleToggleOpenOnly() {
    setShowOnlyOpen((currentValue) => !currentValue);
  }

  function handleReset() {
    setEventList(initialEvents);
    setMessage("Event data ကို reset လုပ်ပြီးပါပြီ။");
    setShowOnlyOpen(false);
  }

  return (
    <main className="page">
      <PageHeader
        eyebrow="Part 6 — Frontend Foundation"
        title="State & Events"
        description="Button click events နဲ့ useState ကိုသုံးပြီး React UI ကို interactive ဖြစ်အောင်လုပ်ပါမယ်။"
      />

      <StatsSummary
        totalEvents={eventList.length}
        totalCapacity={totalCapacity}
        totalRegistered={totalRegistered}
      />

      <ActionPanel
        message={message}
        showOnlyOpen={showOnlyOpen}
        onToggleOpenOnly={handleToggleOpenOnly}
        onReset={handleReset}
      />

      <EventList
        events={visibleEvents}
        openEventCount={openEvents.length}
        onRegister={handleRegister}
      />
    </main>
  );
}

function PageHeader({ eyebrow, title, description }) {
  return (
    <section className="hero">
      <p className="eyebrow">{eyebrow}</p>
      <h1>{title}</h1>
      <p>{description}</p>
    </section>
  );
}

function StatsSummary({ totalEvents, totalCapacity, totalRegistered }) {
  const remainingSeats = totalCapacity - totalRegistered;

  return (
    <section className="stats" aria-label="Event summary">
      <StatCard label="Events" value={totalEvents} />
      <StatCard label="Registered" value={totalRegistered} />
      <StatCard label="Remaining Seats" value={remainingSeats} />
    </section>
  );
}

function StatCard({ label, value }) {
  return (
    <article className="stat-card">
      <p>{label}</p>
      <strong>{value}</strong>
    </article>
  );
}

function ActionPanel({ message, showOnlyOpen, onToggleOpenOnly, onReset }) {
  return (
    <section className="action-panel" aria-label="Event actions">
      <p className="status-message">{message}</p>

      <div className="action-row">
        <button
          className="secondary-button"
          type="button"
          onClick={onToggleOpenOnly}
        >
          {showOnlyOpen ? "Event အားလုံးပြမည်" : "Open events ပဲပြမည်"}
        </button>

        <button className="secondary-button" type="button" onClick={onReset}>
          Reset
        </button>
      </div>
    </section>
  );
}

function EventList({ events, openEventCount, onRegister }) {
  if (events.length === 0) {
    return <EmptyState message="Register လုပ်နိုင်သော event မရှိပါ။" />;
  }

  return (
    <section className="event-section" aria-labelledby="events-heading">
      <div className="section-header">
        <div>
          <p className="eyebrow">Upcoming Events</p>
          <h2 id="events-heading">လာမည့် Event များ</h2>
        </div>

        <p className="section-note">
          Register လုပ်နိုင်သော event {openEventCount} ခု ရှိသည်။
        </p>
      </div>

      <div className="event-grid">
        {events.map((event) => (
          <EventCard key={event.id} event={event} onRegister={onRegister} />
        ))}
      </div>
    </section>
  );
}

function EventCard({ event, onRegister }) {
  const remainingSeats = event.capacity - event.registeredCount;
  const isFull = remainingSeats === 0;

  return (
    <article className="event-card" aria-labelledby={`${event.id}-title`}>
      <div className="event-card-header">
        <div>
          <p className="event-category">{event.category}</p>
          <h3 id={`${event.id}-title`}>{event.title}</h3>
        </div>

        <StatusBadge isFull={isFull} remainingSeats={remainingSeats} />
      </div>

      <dl className="event-meta">
        <EventMetaItem label="ရက်စွဲ" value={event.date} />
        <EventMetaItem label="နေရာ" value={event.location} />
        <EventMetaItem label="ကျန်ရှိသောနေရာ" value={remainingSeats} />
      </dl>

      <PrimaryButton disabled={isFull} onClick={() => onRegister(event.id)}>
        {isFull ? "Seats ပြည့်ပါပြီ" : "Register လုပ်မည်"}
      </PrimaryButton>
    </article>
  );
}

function EventMetaItem({ label, value }) {
  return (
    <div>
      <dt>{label}</dt>
      <dd>{value}</dd>
    </div>
  );
}

function StatusBadge({ isFull, remainingSeats }) {
  if (isFull) {
    return <span className="status-badge status-badge-full">Full</span>;
  }

  if (remainingSeats <= 10) {
    return <span className="status-badge status-badge-warning">Few seats</span>;
  }

  return <span className="status-badge status-badge-open">Open</span>;
}

function PrimaryButton({ disabled = false, onClick, children }) {
  return (
    <button
      className="primary-button"
      type="button"
      disabled={disabled}
      onClick={onClick}
    >
      {children}
    </button>
  );
}

function EmptyState({ message }) {
  return (
    <section className="empty-state">
      <p>{message}</p>
    </section>
  );
}

export default App;
```

# 15. Common State Mistakes

## Error 1 — Direct mutation

မကောင်း—

```jsx
event.registeredCount += 1;
setEventList(eventList);
```

ဘာကြောင့်မကောင်းလဲ?

```text
existing object/array ကိုပြောင်းတာဖြစ်
React update detect မလုပ်နိုင်/bug ဖြစ်နိုင်
old state ကိုမသန့်ရှင်းတော့
```

ကောင်း—

```jsx
setEventList((currentEvents) =>
  currentEvents.map((event) => {
    if (event.id !== eventId) {
      return event;
    }

    return {
      ...event,
      registeredCount: event.registeredCount + 1
    };
  })
);
```

React docs မှာ arrays in state ကို immutable/read-only လိုသဘောထားပြီး update လုပ်ချင်ရင် original array ကို mutate မလုပ်ဘဲ new array ကို setter function ထဲပေးရမယ်လို့ရှင်းပြထားပါတယ်။ ([React][5])

## Error 2 — Derived value ကို state ထဲသိမ်းခြင်း

မကောင်း—

```jsx
const [remainingSeats, setRemainingSeats] = useState(75);
```

ကောင်း—

```jsx
const remainingSeats = event.capacity - event.registeredCount;
```

## Error 3 — Event handler ကို render ချိန် run မိခြင်း

မကောင်း—

```jsx
<button onClick={handleRegister(event.id)}>
```

ကောင်း—

```jsx
<button onClick={() => handleRegister(event.id)}>
```

ပထမ code က render ဖြစ်တဲ့အချိန် `handleRegister` ကိုတန်း run နိုင်တယ်။ ဒုတိယ code က click ဖြစ်မှ run မယ်။

## Error 4 — Setter function မသုံးဘဲ state variable ပြင်ခြင်း

မကောင်း—

```jsx
message = "Done";
```

ကောင်း—

```jsx
setMessage("Done");
```

## Error 5 — `useState` ကို condition ထဲခေါ်ခြင်း

မကောင်း—

```jsx
if (someCondition) {
  const [message, setMessage] = useState("");
}
```

ကောင်း—

```jsx
const [message, setMessage] = useState("");
```

React docs မှာ Hooks တွေကို component top level မှာ unconditional ခေါ်ရမယ်၊ conditional block ထဲမခေါ်ရဘူးလို့ဖော်ပြထားပါတယ်။ ([React][1])

# 16. State Naming Rules

State naming convention—

```jsx
const [message, setMessage] = useState("");
const [eventList, setEventList] = useState(initialEvents);
const [showOnlyOpen, setShowOnlyOpen] = useState(false);
```

Pattern—

```text
[stateValue, setStateValue]
```

Examples—

```text
count / setCount
message / setMessage
events / setEvents
isOpen / setIsOpen
selectedCategory / setSelectedCategory
```

Boolean state naming—

```text
isFull
isOpen
isLoading
hasError
showOnlyOpen
```

# 17. Event Handler Naming Rules

Event handler naming—

```text
handle + Action
```

Examples—

```jsx
function handleRegister(eventId) {}
function handleToggleOpenOnly() {}
function handleReset() {}
function handleSubmit(event) {}
function handleNameChange(event) {}
```

Props for event callbacks naming—

```text
on + Action
```

Example—

```jsx
<EventCard event={event} onRegister={handleRegister} />
```

Child calls—

```jsx
onRegister(event.id);
```

Pattern—

```text
Parent handler = handleRegister
Child callback prop = onRegister
```

ဒီ pattern က React code ဖတ်ရတာလွယ်စေတယ်။

# 24. Common Beginner Questions

## 24.1 ဘာကြောင့် remainingSeats ကို state ထဲမထားတာလဲ?

`remainingSeats` က `capacity - registeredCount` နဲ့တွက်လို့ရတယ်။ ဒါကြောင့် state ထဲမထားသင့်ပါ။ Redundant state ဖြစ်ရင် update မေ့ပြီး bug ဖြစ်နိုင်တယ်။ React docs က state ထဲ redundant/duplicated information မထားဖို့အကြံပြုထားပါတယ်။ ([React][4])

## 24.2 `setEventList` ထဲ callback function ဘာကြောင့်သုံးတာလဲ?

```jsx
setEventList((currentEvents) => ...)
```

ဒီ pattern က current state value ကိုအခြေခံပြီး update လုပ်တဲ့အခါ safer ဖြစ်တယ်။ Button click များလာတာ, future async behavior တွေရှိလာတာနဲ့ state update consistency ပိုကောင်းစေတယ်။

## 24.3 `onClick={() => onRegister(event.id)}` ဘာကြောင့် arrow function သုံးတာလဲ?

`onRegister(event.id)` ကို click ဖြစ်မှ run စေချင်လို့ပါ။ `onClick={onRegister(event.id)}` လို့ရေးရင် render ဖြစ်တဲ့အချိန် function တန်း run နိုင်တယ်။

## 24.4 Array update မှာ `push()` မသုံးရတာဘာကြောင့်လဲ?

State ထဲရှိ array ကို read-only လိုသဘောထားရမယ်။ `push()` သည် existing array ကို mutate လုပ်တယ်။ React docs က state array update လုပ်ချင်ရင် `map()`, `filter()` လို non-mutating methods နဲ့ array အသစ်ဖန်တီးသင့်တယ်လို့ရှင်းပြထားပါတယ်။ ([React][5])

## 24.5 State က component တစ်ခုချင်းစီမှာ private လား?

ဟုတ်ပါတယ်။ React docs မှာ state သည် component instance တစ်ခုချင်းစီအတွက် private ဖြစ်ပြီး component ကိုနေရာနှစ်ခုမှာ render လုပ်ရင် copy တစ်ခုချင်းစီက ကိုယ်ပိုင် state ရနိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([React][1])

# 28. Chapter Summary

ဒီအခန်းမှာ **State & Events** ကိုလေ့လာခဲ့တယ်။

အရေးကြီးဆုံး mental model—

```text
State = component memory
Event = user interaction
Event handler = interaction ဖြစ်တဲ့အခါ run မယ့် function
Setter function = state update + re-render trigger
```

ဒီ chapter မှာ Chapter 34 app ကို interactive ဖြစ်အောင်ပြင်ခဲ့တယ်။

```text
eventList state
message state
showOnlyOpen state

handleRegister
handleToggleOpenOnly
handleReset
```

React official docs အရ state variable ကို component က render ကြားမှာ information မှတ်ထားဖို့သုံးပြီး `useState` Hook က current state value နဲ့ setter function ကိုပေးတယ်။ Event handlers တွေက clicking, hovering, focusing input စတဲ့ user interactions တွေဖြစ်တဲ့အခါ run မယ့် functions ဖြစ်ပြီး JSX tag ထဲ `onClick={handleClick}` လိုပေးနိုင်တယ်။ ([React][1])

State design rule—

```text
Store minimal state.
Derive everything else.
Do not duplicate derived values.
```

Array state update rule—

```text
Do not mutate arrays or objects in state.
Create a new array/object with map/filter/spread.
```

React docs က arrays in state ကို read-only လိုသဘောထားသင့်ပြီး mutate မလုပ်ဘဲ new array ကို setter ထဲပေးသင့်ကြောင်းဖော်ပြထားပါတယ်။ ([React][5])

Agentic Coding takeaway—

```text
Agent ကို state/events ခိုင်းတဲ့အခါ:
- state variables ကို list လုပ်ခိုင်းပါ
- derived values ကို state ထဲမထားခိုင်းပါ
- event handlers ကို named functions အဖြစ်ရေးခိုင်းပါ
- direct mutation မလုပ်ကြောင်း review ခိုင်းပါ
- Browser manual tests + npm run build နဲ့ verify လုပ်ပါ
```
