# 3. JSON ဆိုတာဘာလဲ

JSON ရဲ့အပြည့်အစုံက **JavaScript Object Notation** ပါ။ Beginner အတွက် အလွယ်ဆုံးပြောရရင်—

```text
JSON = Data ကို key/value ပုံစံနဲ့ စနစ်တကျရေးတဲ့ text format
```

ပါ။

MDN docs မှာ JSON ကို JavaScript object syntax ပေါ်အခြေခံထားတဲ့ structured data ကို represent လုပ်ရန်သုံးတဲ့ standard text-based format အဖြစ်ရှင်းပြထားပြီး web apps တွေမှာ server က client ကို data ပို့ခြင်း၊ client က server ကို data ပို့ခြင်းတို့အတွက် မကြာခဏအသုံးပြုကြောင်းဖော်ပြထားပါတယ်။ ([MDN Web Docs][1])

Event data example—

```json
{
  "id": 1,
  "title": "AI Workshop for Beginners",
  "location": "Yangon",
  "capacity": 100
}
```

ဒီ JSON object ထဲမှာ—

```text
"id"       → key
1          → value

"title"    → key
"AI Workshop for Beginners" → value
```

ရှိတယ်။

JSON ကို ဒီ course မှာ အောက်ပါနေရာတွေမှာ မကြာခဏတွေ့မယ်။

```text
package.json
data/events.json
API request body
API response body
TASK_SCHEMA.json
TRACEABILITY data
MCP tool input/output schema
```

# 4. JSON Syntax အခြေခံ

JSON မှာ စည်းကမ်းတချို့ရှိတယ်။ ဒီစည်းကမ်းတွေကိုမလိုက်နာရင် syntax error တက်မယ်။

## 4.1 Object

Object က `{}` နဲ့ရေးတယ်။

```json
{
  "name": "Aung Aung",
  "phone": "09123456789"
}
```

Object ထဲမှာ key/value pairs ပါတယ်။

```text
"key": value
```

## 4.2 Array

Array က `[]` နဲ့ရေးတယ်။ List သိမ်းဖို့သုံးတယ်။

```json
[
  "AI Workshop",
  "Startup Meetup",
  "Community Night"
]
```

Object list example—

```json
[
  {
    "id": 1,
    "title": "AI Workshop"
  },
  {
    "id": 2,
    "title": "Startup Meetup"
  }
]
```

Event Management project မှာ event list, attendee list, user list တွေကို array အနေနဲ့မြင်ရမယ်။

## 4.3 String

String က စာသားပါ။ JSON မှာ string ကို double quote `"` နဲ့ရေးရမယ်။

မှန်—

```json
{
  "title": "AI Workshop"
}
```

မှား—

```json
{
  "title": 'AI Workshop'
}
```

JSON မှာ single quote မသုံးပါ။

## 4.4 Number

Number က quote မလိုပါ။

```json
{
  "capacity": 100,
  "price": 5000
}
```

မှား—

```json
{
  "capacity": "100"
}
```

ဒီမှာ `"100"` က string ဖြစ်သွားတယ်။ တကယ့် number မဟုတ်တော့ဘူး။

## 4.5 Boolean

Boolean က `true` သို့မဟုတ် `false` ပါ။

```json
{
  "isPublished": true,
  "isCheckedIn": false
}
```

Quote မထည့်ပါ။

မှား—

```json
{
  "isPublished": "true"
}
```

ဒီမှာ `"true"` က string ဖြစ်သွားတယ်။

## 4.6 Null

Value မရှိသေးတာကို `null` နဲ့ရေးတယ်။

```json
{
  "checkedInAt": null
}
```

ဥပမာ attendee တစ်ယောက် check-in မလုပ်သေးရင်—

```json
{
  "name": "Aung Aung",
  "ticketCode": "EVT-001",
  "checkedInAt": null
}
```

Check-in လုပ်ပြီးရင်—

```json
{
  "name": "Aung Aung",
  "ticketCode": "EVT-001",
  "checkedInAt": "2026-08-01T09:30:00Z"
}
```

# 5. JSON Rules — အရေးကြီးဆုံးစည်းကမ်းများ

## Rule 1 — Key တွေ double quote ပါရမယ်

မှန်—

```json
{
  "title": "AI Workshop"
}
```

မှား—

```json
{
  title: "AI Workshop"
}
```

JavaScript object မှာ key quote မပါလည်းရနိုင်ပေမယ့် JSON မှာ quote ပါရမယ်။

## Rule 2 — String value တွေ double quote ပါရမယ်

မှန်—

```json
{
  "location": "Yangon"
}
```

မှား—

```json
{
  "location": 'Yangon'
}
```

## Rule 3 — Comma ကို item ကြားမှာပဲသုံးရမယ်

မှန်—

```json
{
  "name": "Aung Aung",
  "phone": "09123456789",
  "email": "aung@example.com"
}
```

မှား—

```json
{
  "name": "Aung Aung",
  "phone": "09123456789",
}
```

နောက်ဆုံး item နောက်မှာ comma မထည့်ရပါ။

## Rule 4 — Comment မရေးရ

JSON မှာ comment မရေးရပါ။

မှား—

```json
{
  // Event title
  "title": "AI Workshop"
}
```

Comment လိုချင်ရင် Markdown file ထဲရေးပါ။ JSON ထဲမရေးပါနဲ့။

## Rule 5 — Bracket မပျက်ရ

Opening `{` ရှိရင် closing `}` ရှိရမယ်။

Opening `[` ရှိရင် closing `]` ရှိရမယ်။

မှား—

```json
{
  "title": "AI Workshop"
```

မှန်—

```json
{
  "title": "AI Workshop"
}
```

# 6. JSON နဲ့ JavaScript Object ကွာခြားချက်

Chapter 4 မှာ JavaScript object ကိုတွေ့ခဲ့တယ်။

JavaScript object—

```js
const event = {
  title: "AI Workshop",
  capacity: 100
};
```

JSON—

```json
{
  "title": "AI Workshop",
  "capacity": 100
}
```

အဓိကကွာခြားချက်—

```text
JavaScript object မှာ key quote မလိုနိုင်
JSON မှာ key double quote မဖြစ်မနေလို
JavaScript object မှာ function ပါနိုင်
JSON မှာ function မပါနိုင်
JavaScript object မှာ comment ရေးလို့ရနိုင်
JSON မှာ comment မရ
```

Beginner အတွက် မှတ်ရန်—

```text
package.json, events.json, API body တွေထဲမှာ JSON rules ကိုတိတိကျကျလိုက်နာရမယ်။
```

# 8. Node.js နဲ့ npm ဆိုတာဘာလဲ

## 8.1 Node.js

JavaScript ကို အစပိုင်းမှာ browser ထဲမှာ run တာအများဆုံးသုံးခဲ့တယ်။ Node.js က JavaScript ကို computer/server ပေါ်မှာ run လုပ်နိုင်စေတဲ့ runtime ပါ။

Beginner အတွက်—

```text
Browser JavaScript = webpage behavior အတွက်
Node.js JavaScript = terminal/server/project tools အတွက်
```

React/Vite project တွေ run ဖို့ Node.js လိုမယ်။

## 8.2 npm

npm က Node.js project တွေမှာ packages/dependencies တွေကို install/manage လုပ်ဖို့သုံးတဲ့ package manager ပါ။ Node.js official Learn docs က npm ကို Node.js အတွက် standard package manager အဖြစ်ဖော်ပြထားပြီး project dependencies တွေကို install, update, manage လုပ်နိုင်ကြောင်းရှင်းပြထားပါတယ်။ ([Node.js][2])

Beginner အတွက် npm ကို—

```text
JavaScript project အတွက် app store / package installer / command runner
```

လိုမှတ်နိုင်တယ်။

npm ကိုသုံးပြီး—

```text
project dependencies install လုပ်မယ်
React, Vite, Tailwind စတဲ့ packages တွေယူမယ်
project scripts run မယ်
test/build/dev server command တွေ run မယ်
```

## 8.3 Node.js နဲ့ npm install ရှိ/မရှိစစ်ခြင်း

Terminal မှာ—

```bash
node -v
npm -v
```

run လုပ်ပါ။

Output example—

```text
v22.12.0
10.9.0
```

npm docs က Node.js နဲ့ npm install ရှိ/မရှိစစ်ဖို့ `node -v` နဲ့ `npm -v` command တွေသုံးရန်ဖော်ပြထားပြီး Node.js/npm ကို Node version manager သို့မဟုတ် installer နဲ့ install လုပ်နိုင်ကြောင်းရှင်းပြထားပါတယ်။ ([npm Docs][3])

# 10. package.json ဆိုတာဘာလဲ

`package.json` ဆိုတာ JavaScript/Node project ရဲ့ manifest file ပါ။

npm docs မှာ `package.json` ဟာ actual JSON file ဖြစ်ရမယ်လို့ဖော်ပြထားပြီး `name`, `version`, `scripts`, `dependencies`, `devDependencies`, `private` စတဲ့ fields တွေကို npm ကအသုံးပြုကြောင်း documentation မှာရှင်းပြထားပါတယ်။ ([npm Docs][4])

Beginner အတွက် `package.json` ကို project ရဲ့—

```text
ID card
Command list
Dependency list
Configuration file
```

လိုမှတ်ပါ။

Example—

```json
{
  "name": "event-management",
  "version": "1.0.0",
  "private": true,
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "test": "vitest"
  },
  "dependencies": {
    "react": "^19.0.0",
    "react-dom": "^19.0.0"
  },
  "devDependencies": {
    "vite": "^7.0.0",
    "vitest": "^3.0.0"
  }
}
```

ဒီ file ထဲမှာ—

```text
name             → project/package name
version          → version number
private          → publish မလုပ်အောင်ကာကွယ်
scripts          → npm run နဲ့ run မယ့် commands
dependencies     → app run ဖို့လိုတဲ့ packages
devDependencies  → development/test/build အတွက်လိုတဲ့ packages
```

ရှိတယ်။

# 11. package.json Fields အရေးကြီးဆုံးများ

## 11.1 name

```json
{
  "name": "event-management"
}
```

Project/package name ပါ။

Naming rule—

```text
lowercase
space မပါ
special character မပါ
hyphen သုံး
```

ကောင်းတဲ့ name—

```text
event-management
myaree-store
agentic-zero-to-hero
```

## 11.2 version

```json
{
  "version": "1.0.0"
}
```

Version number ပါ။

အများဆုံးသုံးတဲ့ pattern—

```text
major.minor.patch
```

ဥပမာ—

```text
1.0.0
1.1.0
1.1.1
```

ဒီကို semantic versioning လို့ခေါ်တတ်တယ်။

## 11.3 private

```json
{
  "private": true
}
```

ဒီ project ကို npm registry ပေါ်မတော်တဆ publish မလုပ်မိအောင်ကာကွယ်ဖို့ သုံးတတ်တယ်။ npm package docs မှာ package ကို private/unpublished အဖြစ်ထားချင်ရင် `"private": true` ကိုထည့်ရန်အကြံပြုထားပါတယ်။ ([npm Docs][4])

Beginner project တွေမှာ အများအားဖြင့် ထည့်ထားသင့်ပါတယ်။

## 11.4 scripts

```json
{
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "test": "vitest"
  }
}
```

`scripts` က command shortcuts ပါ။

Terminal မှာ—

```bash
npm run dev
npm run build
npm run test
```

လို့ run မယ်။

# 15. package-lock.json ဆိုတာဘာလဲ

`package-lock.json` က dependency versions တွေကို exact tree အနေနဲ့ lock လုပ်ထားတဲ့ file ပါ။

npm docs အရ `package-lock.json` ဟာ npm က `node_modules` tree သို့မဟုတ် `package.json` ကိုပြင်တဲ့ operation တွေအတွက် auto-generate လုပ်တဲ့ file ဖြစ်ပြီး နောက်ပိုင်း install တွေမှာ identical dependency tree ပြန်ထုတ်နိုင်အောင် exact tree ကိုဖော်ပြထားတယ်လို့ရှင်းပြထားပါတယ်။ ဒီ file ကို source repository ထဲ commit လုပ်ရန် intended ဖြစ်ကြောင်းလည်း docs မှာဖော်ပြထားပါတယ်။ ([npm Docs][5])

Beginner အတွက်—

```text
package.json      = ဘာ packages လိုတယ်ဆိုတာရေးထားတဲ့ list
package-lock.json = တကယ် install လုပ်ထားတဲ့ exact versions map
node_modules      = actual downloaded packages
```

Rule—

```text
package-lock.json ကို commit လုပ်ပါ။
node_modules ကို commit မလုပ်ပါ။
```

# 17. npm audit ဆိုတာဘာလဲ

Package အသစ်တွေထည့်တာက supply chain risk ပါလာနိုင်တယ်။ npm မှာ `npm audit` ဆိုတဲ့ command ရှိပြီး dependencies ထဲက known security vulnerabilities တွေကိုစစ်နိုင်တယ်။ npm docs မှာ security audit ကို package dependencies တွေထဲက vulnerabilities တွေစစ်တဲ့ assessment အဖြစ်ဖော်ပြထားပြီး `npm audit` က configured dependencies description ကို registry ဆီပို့ကာ known vulnerabilities report တောင်းတယ်လို့ရှင်းပြထားပါတယ်။ ([npm Docs][6])

Run—

```bash
npm audit
```

သို့သော် beginner အတွက် အရေးကြီးတဲ့ rule—

```text
npm audit fix --force ကို မျက်စိမှိတ်မ run ပါနဲ့။
```

ဘာကြောင့်လဲဆိုရင် breaking changes ဖြစ်နိုင်တယ်။ npm docs က audit fix recommendations တွေမှာ semantic version major change ဖြစ်နိုင်လျှင် SEMVER warning ပါနိုင်ပြီး package API ပြောင်းသွားရင် code ကိုထပ်ပြင်ရနိုင်တယ်လို့ရှင်းပြထားပါတယ်။ ([npm Docs][6])

Agent prompt—

```text
npm audit report ကိုဖတ်ပါ။
npm audit fix --force မ run ပါနဲ့။
ပထမဆုံး vulnerabilities summary, severity, breaking change risk, recommended safe action ကိုပြပါ။
```

# 18. Version Symbols — ^ နှင့် ~

`package.json` ထဲမှာ version တွေဒီလိုတွေ့နိုင်တယ်။

```json

  "dependencies": {
    "react": "^19.0.0",
```

Beginner အတွက် အနက်ကြီးမသွားဘဲ ဒီလိုမှတ်ပါ။

```text
"^19.0.0" → compatible newer versions လက်ခံနိုင်
"~1.11.0" → patch-level newer versions လက်ခံနိုင်
```

Versioning ကိုနောက်ပိုင်း dependency safety chapter မှာပိုရှင်းမယ်။ အခုအဆင့်မှာ အရေးကြီးတာက—

```text

```

# 31. Chapter Summary

ဒီအခန်းမှာ JSON, npm, `package.json` ကို survival-level လေ့လာခဲ့တယ်။

အရေးကြီးဆုံး concepts—

```text
JSON
Data ကို key/value format နဲ့ရေးတဲ့ text format.

npm
JavaScript/Node project အတွက် package manager + script runner.

package.json
Project metadata, scripts, dependencies သိမ်းထားတဲ့ file.

scripts
npm run နဲ့ run မယ့် command shortcuts.

dependencies
App run ဖို့လိုတဲ့ packages.

devDependencies
Development/testing/build tools.

node_modules
Installed packages folder. Git commit မလုပ်ရ.

package-lock.json
Exact dependency tree သိမ်းထားတဲ့ lock file. Git commit လုပ်သင့်.
```

Agentic Coding အတွက် အဓိက takeaway—

```text
npm command တွေက project files ကိုပြောင်းနိုင်တယ်။
package install မလုပ်ခင် Agent ကို reason, risk, changed files, rollback plan တောင်းပါ။
```

မကောင်းတဲ့ prompt—

```text
လိုတာ install လုပ်ပါ။
```

```text

ကျွန်တော် approve လုပ်မှ install လုပ်ပါ။
```
