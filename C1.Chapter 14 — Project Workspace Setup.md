# 3. Workspace Design Principles

Workspace တည်ဆောက်တဲ့အခါ အောက်ပါစည်းကမ်းတွေလိုက်မယ်။

```text
Rule 1 — One course, one root folder
Rule 2 — Notes, labs, playground, projects ကိုခွဲထား
Rule 3 — Secret values မထည့်ထားတဲ့ .env.example ကိုသာ commit
Rule 4 — .env ကို Git ထဲမထည့်
Rule 5 — node_modules/dist/build ကို Git ထဲမထည့်
Rule 6 — Agent memory/rules ကို CLAUDE.md / AGENTS.md ထဲထား
Rule 7 — Project-specific Claude settings ကို .claude/settings.json ထဲထား
Rule 8 — Personal/local settings ကို Git ထဲမထည့်
Rule 9 — Agent session မစခင် pwd + ls + git status စစ်
Rule 10 — Workspace setup ပြီးတာနဲ့ first commit လုပ်
```

Claude Code docs အရ `settings.json` က Claude Code configuration အတွက် official mechanism ဖြစ်ပြီး user settings, project settings, local settings, managed settings ဆိုပြီး scope တွေခွဲထားပါတယ်။ Project settings ကို `.claude/settings.json` ထဲမှာထားပြီး source control ထဲ commit လုပ်နိုင်သလို personal/local settings ကို `.claude/settings.local.json` ထဲထားနိုင်ပြီး Git ignore လုပ်သင့်ပါတယ်။ ([Claude][1])

# 10. `CLAUDE.md` Starter

`CLAUDE.md` ကို တိုတို၊ရှင်းရှင်းထားပါ။ Session တိုင်းဖတ်မယ့် context ဖြစ်တာကြောင့် အရမ်းရှည်မထားသင့်ပါ။

```md
# CLAUDE.md

## Project

Agentic Coding Zero to Hero course workspace.

## Purpose

This repository is used to learn agentic coding step by step, from beginner foundations to full-stack capstone workflows.

## Important Folders

- `notes/` — chapter notes
- `labs/` — lab outputs
- `playground/` — small experiments
- `projects/myaree-store/` — warm-up project
- `projects/event-management/` — final capstone project
- `.claude/` — Claude Code settings, agents, skills, commands, hooks

## Safety Rules

- Do not read or edit `.env` files.
- Do not expose secret values.
- Do not edit `node_modules/`, `dist/`, or `build/`.
- Do not install packages without explaining why and waiting for approval.
- Do not run destructive commands without approval.
- Do not deploy anything without explicit approval.

## Workflow Rules

- For non-trivial tasks, plan before editing.
- Before editing, run or ask to run `git status`.
- If the working tree is not clean, summarize changes before proceeding.
- After edits, provide changed files summary.
- Provide verification steps before saying a task is done.
- Suggest a commit message but do not commit without approval.

## Definition of Done

A task is done only when:
- scope is respected
- changes are summarized
- verification steps are provided
- risks or assumptions are listed
- Git diff is ready for human review
```

# 12. `.claude/settings.json` Starter

`.claude/settings.json` ထဲမှာ project-level settings/rules ထည့်နိုင်တယ်။ အခုအဆင့်မှာ beginner-safe deny rules အနည်းငယ်ထည့်မယ်။

```json
{
  "$schema": "https://json.schemastore.org/claude-code-settings.json",
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)",
      "Read(./node_modules/**)",
      "Read(./dist/**)",
      "Read(./build/**)",
      "Bash(rm -rf *)",
      "Bash(git reset --hard*)",
      "Bash(git clean -fd*)",
      "Bash(curl * | bash*)",
      "Bash(curl * | sh*)"
    ]
  }
}
```

Claude Code settings docs မှာ sensitive files ကို Claude Code မဖတ်နိုင်အောင် `.claude/settings.json` ထဲက `permissions.deny` ကိုသုံးနိုင်ပြီး `.env`, `.env.*`, `secrets/**`, credentials files စတာတွေကို deny list ထဲထည့်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude][1])

သတိ—

```text
ဒီ settings က safety layer တစ်ခုပါ။
ဒါပေမယ့် human review ကိုအစားမထိုးပါဘူး။
နောက်ပိုင်း Hooks chapter မှာ deterministic enforcement ကိုပိုတိတိကျကျသင်မယ်။
```

# 25. Chapter Summary

ဒီအခန်းမှာ Agentic Coding အတွက် safe project workspace တစ်ခုကိုတည်ဆောက်ခဲ့တယ်။

အရေးကြီးဆုံး structure—

```text
agentic-zero-to-hero/
├── notes/
├── labs/
├── playground/
├── projects/
├── resources/
└── .claude/
```

အရေးကြီးဆုံး root files—

```text
README.md
COURSE_DECISIONS.md
CLAUDE.md
AGENTS.md
.gitignore
.env.example
.claude/settings.json
```

အရေးကြီးဆုံး safety rules—

```text
.env မ commit
.env.example တွင် secret value မထည့်
node_modules/dist/build မ commit
.claude/settings.local.json မ commit
Agent first run = read-only setup check
```

Agentic Coding အတွက် takeaway—

```text
Workspace မရှင်းရင် Agent လမ်းလွဲမယ်။
Workspace ရှင်းရင် Agent ကို context ပေးရလွယ်၊ Git review လုပ်ရလွယ်၊ project တိုးချဲ့ရလွယ်တယ်။
```
