# Chapter 27 — Context Packing

## 1. Learning Objectives

ဒီအခန်းပြီးသွားရင် သင်က အောက်ပါအရာတွေကို နားလည်ပြီး လက်တွေ့အသုံးချနိုင်ရပါမယ်။

```text
[ ] Context ဆိုတာ Agent သိနေတဲ့ session information ဖြစ်ကြောင်းနားလည်မယ်
[ ] Context Packing ဆိုတာ Agent ကိုလိုအပ်တဲ့ information ကို မပိုမလိုပေးခြင်းဖြစ်ကြောင်းရှင်းပြနိုင်မယ်
[ ] Useful context နဲ့ noisy context ကိုခွဲနိုင်မယ်
[ ] Task Brief, Relevant Files, SPEC Excerpts, Error Logs, Git Diff, Constraints, Verification ကို context packet အဖြစ်စုနိုင်မယ်
[ ] Agent ကို file အများကြီးမဖတ်ခိုင်းဘဲ context summary ပေးနိုင်မယ်
[ ] Repo map / file summary / dependency map / acceptance criteria excerpt ရေးနိုင်မယ်
[ ] Error log ကို secret မပါအောင် redact လုပ်ပြီးပေးနိုင်မယ်
[ ] Context window, compaction, /clear, fresh session, subagent context separation concept တွေကိုနားလည်မယ်
[ ] Context overloading / context rot / stale context / conflicting context ကိုရှောင်နိုင်မယ်
[ ] Context Packet template တစ်ခုရေးနိုင်မယ်
[ ] Agent ကို “ဒီ context ကိုသုံးပြီး plan ပေးပါ” လို့ခိုင်းနိုင်မယ်
[ ] Context Packing lab report တစ်ခုရေးနိုင်မယ်
```

ဒီအခန်းက Part 5 — Project Memory & Context Engineering ရဲ့ အရေးကြီးဆုံး practical chapter တွေထဲကတစ်ခုပါ။ Chapter 24–26 မှာ `CLAUDE.md`, `AGENTS.md`, rules/instructions တွေကိုလေ့လာခဲ့တယ်။ အခု Chapter 27 မှာတော့ task တစ်ခုလုပ်ဖို့ Agent ကို **လိုအပ်တဲ့ context ကို မပိုမလို pack လုပ်ပေးနည်း** ကိုသင်မယ်။

## 2. Beginner Explanation

Agent ကိုအလုပ်ခိုင်းတဲ့အခါ context မပေးဘဲ ခိုင်းရင် Agent က guess လုပ်ရတယ်။

Bad prompt—

```text
Registration bug fix လုပ်ပါ။
```

Agent မသိနိုင်တာ—

```text
ဘယ် project ထဲက registration?
Frontend bug လား backend bug လား?
Error message ဘာလဲ?
Expected behavior ဘာလဲ?
ဘယ် files ဆက်စပ်လဲ?
SPEC မှာဘာရေးထားလဲ?
နောက်ဆုံးဘာတွေပြင်ထားလဲ?
```

Context Packing ဆိုတာ အဲဒီလိုမဖြစ်အောင် task တစ်ခုအတွက် လိုအပ်တဲ့ information ကို စနစ်တကျစုထားတာပါ။

```text
Context Packing
= Agent အလုပ်လုပ်ဖို့လိုတဲ့ context ကို
  short, relevant, structured, safe ဖြစ်အောင်စုထုပ်ပေးခြင်း
```

ဥပမာ context packet တစ်ခု—

```md
# Context Packet — Registration Phone Validation Bug

## Task
Fix phone empty validation in the static registration lab.

## Expected
Phone empty ဖြစ်ရင် “ဖုန်းနံပါတ်ဖြည့်ပါ။” ပြရမယ်။

## Actual
Phone empty ဖြစ်ပေမယ့် success message ပြနေတယ်။

## Relevant Files
- playground/chapter-11-coding-agent/index.html
- playground/chapter-11-coding-agent/app.js

## Relevant SPEC
- Registration requires name and phone.
- Valid submit should show success only when both fields are filled.

## Recent Changes
- app.js validation function was refactored.

## Constraints
- Frontend only.
- No package install.
- Do not edit .env, package.json, node_modules.

## Verification
- Empty name
- Empty phone
- Valid submit
- Browser console error check
```

ဒီလို context packet ပေးလိုက်ရင် Agent က file အများကြီးမဖတ်ခင် task ကိုနားလည်နိုင်တယ်။

# 3. Context ဆိုတာဘာလဲ

Context ဆိုတာ Agent က session ထဲမှာ သိနေတဲ့ information အားလုံးပါ။

Claude Code official docs အရ context window ထဲမှာ session အတွက် Claude သိနေတဲ့အရာများ — instructions, Claude ဖတ်ထားတဲ့ files, Claude ကိုယ်တိုင်ဖြေထားတဲ့ responses, terminal ထဲမမြင်ရတဲ့ internal content တချို့ — ပါဝင်နိုင်ပါတယ်။ Session startup မှာ `CLAUDE.md`, auto memory, MCP tool names, skill descriptions စတာတွေ load ဖြစ်နိုင်ပြီး Claude ဖိုင်ဖတ်တာ၊ rules load ဖြစ်တာ၊ hooks fire ဖြစ်တာတွေက context ထဲကိုထပ်ထည့်သွားနိုင်ကြောင်း docs ကရှင်းပြထားပါတယ်။ ([Claude][1])

Context source တွေ—

```text
User prompt
CLAUDE.md / AGENTS.md
SPEC.md / README.md / COURSE_DECISIONS.md
Files Agent reads
Git diff
Terminal output
Error logs
Previous conversation
Auto memory
Tool results
Subagent summaries
Skills / commands descriptions
```

အရေးကြီးတဲ့ mindset—

```text
Agent quality = Model quality + Tool access + Context quality
```

Context မကောင်းရင် model ကောင်းလည်းလမ်းလွဲနိုင်တယ်။

# 4. Context Packing ဆိုတာဘာလဲ

Context Packing ဆိုတာ—

```text
Task တစ်ခုအတွက် Agent သိရန်လိုတဲ့ context ကို
တိတိကျကျရွေး၊ အတိုချုပ်၊ structure ချ၊ secret မပါအောင် sanitize လုပ်ပြီးပေးခြင်း
```

ပါ။

Context Packing ရဲ့ရည်ရွယ်ချက်—

```text
[ ] Agent guess လုပ်ရတာလျော့
[ ] irrelevant files/logs မဖတ်ရ
[ ] scope မကျော်
[ ] acceptance criteria မမေ့
[ ] error evidence မပျောက်
[ ] secret မပေါက်
[ ] context window မပွ
[ ] review/verification လုပ်ရလွယ်
```

Bad context—

```text
Project ထဲက file အားလုံး paste လုပ်ထား
Log အရှည်ကြီး secret ပါနေ
SPEC တစ်ခုလုံး paste
Error ဘာလဲမရှင်း
Recent changes မပါ
```

Good context—

```text
Task summary
Relevant files ၂–၅ ခု
SPEC excerpt တိုတို
Error snippet only
Git diff summary
Constraints
Acceptance criteria
Verification steps
```

# 5. Context Packing ဘာကြောင့်အရေးကြီးလဲ

## 5.1 Agent focus ပိုကောင်းစေတယ်

Context ပိုများတာက အမြဲပိုကောင်းတာမဟုတ်ပါဘူး။ Irrelevant context များရင် Agent လမ်းလွဲနိုင်တယ်။

```text
Relevant context = signal
Irrelevant context = noise
Conflicting context = danger
Stale context = wrong direction
```

## 5.2 Token / cost / latency ကိုထိန်းပေးတယ်

Long sessions တွေကြာလာရင် context ကြီးလာတတ်တယ်။ OpenAI compaction docs မှာ long-running interactions တွေမှာ context size ကိုလျှော့ပြီး နောက် turns တွေအတွက်လိုအပ်တဲ့ state ကိုထိန်းထားဖို့ compaction ကိုသုံးနိုင်ကြောင်း, context ကြီးလာတာက quality, cost, latency balance ကိုထိခိုက်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([OpenAI Developers][2])

## 5.3 Safety ပိုကောင်းစေတယ်

Context packet ကို sanitize လုပ်ရင်—

```text
.env values မပါ
API key မပါ
database URL မပါ
PII full logs မပါ
```

ဖြစ်အောင်ထိန်းနိုင်တယ်။

## 5.4 Review လုပ်ရလွယ်စေတယ်

Context packet က task document တစ်ခုလိုဖြစ်တယ်။ နောက်ပိုင်း review လုပ်တဲ့အခါ—

```text
Agent ဘာသိပြီးလုပ်ခဲ့လဲ
ဘာ acceptance criteria နဲ့လုပ်ခဲ့လဲ
ဘာ constraints ရှိခဲ့လဲ
```

ပြန်စစ်လို့ရတယ်။

# 6. Useful Context vs Noisy Context

## Useful Context

```text
Task goal
User-visible expected behavior
Relevant file paths
Short relevant code snippets
SPEC acceptance criteria
Recent git diff summary
Exact error message
Command that failed
Environment version, if relevant
Constraints and non-goals
Verification requirements
```

## Noisy Context

```text
Unrelated files
Full raw logs
Old plans that changed
Outdated requirements
Long tutorial text
Entire package-lock.json
node_modules content
Screenshot text unrelated to bug
Spec sections unrelated to task
Repeated conversation history
```

## Dangerous Context

```text
.env content
API keys
Bearer tokens
Database URLs with passwords
Private keys
Production credentials
Full PII exports
```

Course rule—

```text
Pack relevant context.
Summarize noisy context.
Redact dangerous context.
```

# 7. Context Packet Anatomy

Context packet ကောင်းတစ်ခုမှာ အောက်ပါ section တွေပါနိုင်တယ်။

```text
1. Task
2. Current Goal
3. Current State
4. Relevant Files
5. Relevant SPEC / Decisions
6. Error / Evidence
7. Recent Changes
8. Constraints / Non-goals
9. Safety Notes
10. Acceptance Criteria
11. Verification
12. Requested Agent Output
```

Template—

```md
# Context Packet — [Task Name]

## Task

[What the agent should help with]

## Current Goal

[Desired outcome]

## Current State

[What exists now]

## Relevant Files

- [path]: [why relevant]
- [path]: [why relevant]

## Relevant SPEC / Decisions

- [short excerpt]
- [short excerpt]

## Error / Evidence

[paste exact error or summary, no secrets]

## Recent Changes

- [changed file / behavior]
- [git diff summary]

## Constraints / Non-goals

- [constraint]
- [non-goal]

## Safety Notes

- Do not read .env
- Do not install packages
- Do not touch package.json unless approved

## Acceptance Criteria

- [criteria]
- [criteria]

## Verification

- [manual test]
- [command]

## Requested Output

[plan / debug / review / fix plan / verification table]
```

# 8. Context Packing vs Prompt Anatomy

Prompt Anatomy က Agent ကို task အမိန့်ပေးဖို့ structure ပါ။

Context Packing က prompt ထဲမှာပါမယ့် evidence/context ကို စနစ်တကျပြင်ဆင်တာပါ။

```text
Prompt Anatomy
= command structure

Context Packing
= information selection and compression
```

Example—

```text
Prompt Anatomy:
Mode: Plan first
Goal: Fix phone validation
Allowed files: app.js
Output: root cause + fix plan

Context Packet:
Expected/Actual
Relevant SPEC excerpt
Console error
Recent git diff summary
```

အကောင်းဆုံး workflow—

```text
Task Decomposition
→ Context Packing
→ Plan-first Prompt
→ Review
→ Implement
→ Verify
```

# 10. Context Budget Mindset

Agent context က infinite မဟုတ်ပါ။ Context window ရှိပေမယ့် အဲဒါထဲမှာ—

```text
system instructions
CLAUDE.md / AGENTS.md
tools descriptions
skills descriptions
conversation history
file contents
terminal output
your prompt
```

အကုန်နေရာယူတယ်။

Claude Code context window docs မှာ startup content, file reads, rules, hooks, subagent summaries, `/compact` summary စတာတွေက context timeline ထဲဝင်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ Subagent တစ်ခုက separate context window ထဲမှာ research လုပ်ပြီး summary ပဲ main context ပြန်ပေးနိုင်တာကြောင့် large reads တွေကို main context မပွအောင်ထိန်းနိုင်တယ်လို့ docs ကဖော်ပြထားပါတယ်။ ([Claude][1])

Context budget rule—

```text
Need-to-know only.
Relevant snippets over full files.
Summaries over raw logs.
SPEC excerpts over whole SPEC.
Subagent summaries over raw exploration.
```

# 11. Progressive Disclosure

Progressive disclosure ဆိုတာ context အားလုံးကိုအစကတည်းကမပေးဘဲ လိုလာတဲ့အခါမှအသေးစိတ်ဖွင့်ပေးခြင်းပါ။

Codex Skills docs မှာ skills တွေကို context ထိန်းဖို့ progressive disclosure နဲ့သုံးကြောင်းဖော်ပြထားပါတယ်။ Codex က skill တစ်ခုချင်းစီရဲ့ name, description, file path ကိုသာအရင် context ထဲထည့်ပြီး task နဲ့ကိုက်ညီမှ full `SKILL.md` instructions ကိုဖတ်တယ်လို့ docs ကရှင်းပြထားပါတယ်။ ([OpenAI Developers][3])

ဒီ principle ကို context packing မှာလည်းသုံးနိုင်တယ်။

Bad—

```text
ဒီ task အတွက် project တစ်ခုလုံး paste လုပ်ထား
```

Good—

```text
1. Repo map ပေး
2. Relevant files list ပေး
3. Agent ကို ဘယ် file အတိအကျဖတ်မလဲ plan တောင်း
4. Need ဖြစ်မှ file snippets ပေး
```

Progressive context workflow—

```text
Overview
→ Relevant file map
→ Specific excerpts
→ Error details
→ Git diff
→ Full file only if needed
```

# 15. Relevant Snippets

Full file မဟုတ်ဘဲ relevant snippet ပေးပါ။

Example—

```js
function validateRegistrationForm(formData) {
  const name = formData.name.trim();
  const phone = formData.phone.trim();

  if (name === "") {
    return "နာမည်ဖြည့်ပါ။";
  }

  if (phone === "") {
    return "ဖုန်းနံပါတ်ဖြည့်ပါ။";
  }

  return null;
}
```

Snippet ပေးတဲ့အခါ—

```text
file path
line/section purpose
what omitted
```

ပါရင်ကောင်းတယ်။

```md
Snippet from:
playground/chapter-11-coding-agent/app.js

Purpose:
Current validation logic.

Omitted:
DOM selectors and submit handler are below this function.
```

# 17. Error Log Packing

Error logs ကို pack လုပ်တဲ့အခါ first meaningful error ကိုထုတ်ပါ။

Bad—


# 27. Common Context Packing Mistakes

## Error 1 — Everything Paste လုပ်ခြင်း

Project/file/log အကုန် paste လုပ်တာက Agent ကိုမကူညီဘဲ noise ဖြစ်နိုင်တယ်။

Fix—

```text
Relevant excerpts only.
Summarize first.
Full file only if needed.
```

## Error 2 — Error message မပါခြင်း

Bug context packet မှာ exact error မပါရင် diagnosis ခက်တယ်။

Fix—

```text
First meaningful error ထည့်ပါ။
```

## Error 3 — Acceptance criteria မပါခြင်း

Agent က “done” ဘာလဲမသိဘူး။

Fix—

```text
Criteria checklist ထည့်ပါ။
```

## Error 4 — Old plan ထည့်မိခြင်း

Rejected plan ကို context ထဲထားရင် Agent ကအဲဒါကိုပြန်လိုက်နိုင်တယ်။

Fix—

```text
Rejected / outdated context ကို “Ignore” section ထဲသာရေးပါ။
```

## Error 5 — Secret ပါသွားခြင်း

Logs/configs ထဲ secret ပါနိုင်တယ်။

Fix—

```text
Redact before paste.
.env မပေးပါ။
.env.example ကိုသာပေးပါ။
```

## Error 6 — Relevant files မဖော်ပြခြင်း

Agent က repo တစ်ခုလုံး explore လုပ်ရမယ်။

Fix—

```text
Relevant files + reason list ထည့်ပါ။
```

# 35. Chapter Summary

ဒီအခန်းမှာ **Context Packing** ကိုလေ့လာခဲ့တယ်။

အရေးကြီးဆုံး mental model—

```text
Context Packing = Agent အလုပ်လုပ်ဖို့လိုတဲ့ information ကို
မပိုမလို၊ safe, structured, current ဖြစ်အောင်ပေးခြင်း
```

Context packet ကောင်းတစ်ခုမှာပါသင့်တာ—

```text
Task
Goal
Current State
Relevant Files
SPEC / Decisions Excerpts
Error / Evidence
Recent Changes
Constraints / Non-goals
Safety Notes
Acceptance Criteria
Verification
Requested Output
```

အရေးကြီးဆုံး rules—

```text
Relevant context ပေးပါ။
Noisy context ကို summarize လုပ်ပါ။
Dangerous context ကို redact လုပ်ပါ။
Stale context ကိုဖယ်ပါ။
Conflicting context ကို report ခိုင်းပါ။
Full files/logs မပေးခင် summaries/excerpts သုံးပါ။
```

Agentic Coding အတွက် takeaway—

```text
Prompt ကောင်းတာက instruction ကိုရှင်းစေတယ်။
Context packet က Agent ရဲ့နားလည်မှုကိုမှန်စေတယ်။
Context မကောင်းရင် prompt ကောင်းတောင် Agent လမ်းလွဲနိုင်တယ်။
```
