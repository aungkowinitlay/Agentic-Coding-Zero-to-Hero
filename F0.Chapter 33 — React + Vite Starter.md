## 2. Beginner Explanation

React ဆိုတာ website / web app UI ကို component တွေအဖြစ်တည်ဆောက်ဖို့သုံးတဲ့ JavaScript library ပါ။ အရင် HTML/CSS/JavaScript survival အခန်းတွေမှာ `index.html`, `style.css`, `app.js` သုံးပြီး static page တွေတည်ဆောက်ခဲ့တယ်။ React မှာတော့ UI ကို component လို့ခေါ်တဲ့ function-like pieces အဖြစ်ရေးမယ်။

Vite ဆိုတာ React app ကို local development server နဲ့မြန်မြန် run လုပ်ပေးတဲ့ frontend build tool ပါ။ React official docs က React app တစ်ခုကို scratch ကစတည်ဆောက်မယ်ဆိုရင် Vite, Parcel, Rsbuild လို build tool တစ်ခု install လုပ်ပြီး local development server နဲ့ production build command ရနိုင်ကြောင်းဖော်ပြထားပါတယ်။ React docs ထဲမှာ Vite ကို modern web projects အတွက် faster and leaner development experience ပေးတဲ့ build tool အဖြစ်ဖော်ပြပြီး Vite command example ကိုလည်းပြထားပါတယ်။ ([React][1])

Mental model—

```text
React
= UI ကို component တွေအဖြစ်ရေးတဲ့ JavaScript library

Vite
= React app ကို run/build/preview လုပ်ပေးတဲ့ development tool

npm
= project dependencies install/run scripts လုပ်တဲ့ package manager

Browser
= React app ကိုကြည့်စမ်းတဲ့နေရာ

DevTools
= console error / UI / network စစ်တဲ့နေရာ
```



# 4. React + Vite App Flow

React + Vite project ထဲမှာ browser က `index.html` ကိုဖွင့်တယ်။ `index.html` ထဲက `<div id="root"></div>` ကို React က root container အဖြစ်ယူတယ်။ `src/main.jsx` ထဲမှာ `createRoot` နဲ့ React root တည်ဆောက်ပြီး `<App />` component ကို render လုပ်တယ်။

React official docs က `createRoot` ကို browser DOM node တစ်ခုထဲ React component တွေပြဖို့ root တည်ဆောက်တဲ့ API အဖြစ်ဖော်ပြထားပြီး `root.render(<App />)` ခေါ်မှ App component ကို React root ထဲ display လုပ်ကြောင်းရှင်းပြထားပါတယ်။ React app တစ်ခုလုံးကို React နဲ့တည်ဆောက်တဲ့အခါ root တစ်ခုတည်းကိုသာ create လုပ်လေ့ရှိကြောင်းလည်း docs မှာဖော်ပြထားပါတယ်။ ([React][3])

Flow—

```text
Browser opens index.html
   ↓
index.html has <div id="root"></div>
   ↓
index.html loads /src/main.jsx
   ↓
main.jsx imports React and App
   ↓
createRoot(document.getElementById("root"))
   ↓
root.render(<App />)
   ↓
App.jsx UI appears in browser
```

အရေးကြီးတဲ့ files—

```text
index.html
= browser entry HTML

src/main.jsx
= React app startup file

src/App.jsx
= main UI component

src/App.css
= App component styles

package.json
= npm scripts and dependencies

vite.config.js
= Vite configuration
```

# 5. React Concepts အခြေခံ

## 5.1 Component

Component ဆိုတာ UI အပိုင်းတစ်ခုပါ။

ဥပမာ—

```jsx
function EventCard() {
  return (
    <article>
      <h2>AI Workshop</h2>
      <p>Yangon</p>
    </article>
  );
}
```

ဒီ `EventCard` က component တစ်ခုပါ။ နောက်ပိုင်း Event Management project မှာ components အများကြီးရှိလာမယ်။

```text
EventCard
EventList
RegistrationForm
AdminTable
CheckInButton
```

## 5.2 JSX

JSX ဆိုတာ JavaScript ထဲမှာ HTML-like syntax ရေးတဲ့နည်းပါ။

```jsx
<h1>မင်္ဂလာပါ React</h1>
```

JSX က HTML နဲ့တူပေမယ့် JavaScript rules တွေရှိတယ်။

ဥပမာ—

```jsx
<div className="card">
  <h2>{event.title}</h2>
</div>
```

HTML မှာ `class` သုံးတယ်။ React JSX မှာ `className` သုံးတယ်။

## 5.3 Props

Props ဆိုတာ component တစ်ခုကို data ပေးတဲ့နည်းပါ။

```jsx
function EventCard({ title, location }) {
  return (
    <article>
      <h2>{title}</h2>
      <p>{location}</p>
    </article>
  );
}
```

Use—

```jsx
<EventCard title="AI Workshop" location="Yangon" />
```

## 5.4 State

State ဆိုတာ component ထဲကပြောင်းလဲနိုင်တဲ့ data ပါ။ ဒီအခန်းမှာ `useState` ကိုအတင်းမထည့်သေးပါဘူး။ Starter app ကို static components နဲ့စပြီး နောက် Chapter တွေမှာ state, events, forms ကိုဆက်သင်မယ်။

# 13. Replace `src/App.jsx`

`src/App.jsx` ကိုအောက်ပါ code နဲ့ပြောင်းပါ။

```jsx
import "./App.css";

function App() {
  const event = {
    title: "AI Workshop",
    date: "2026-08-01",
    location: "Yangon",
    capacity: 100,
    registeredCount: 25
  };

  const remainingSeats = event.capacity - event.registeredCount;

  return (
    <main className="page">
      <section className="hero">
        <p className="eyebrow">Part 6 — Frontend Foundation</p>
        <h1>React + Vite Starter</h1>
        <p>
          Event Management project အတွက် React frontend foundation ကို စတင်လေ့ကျင့်ပါမယ်။
        </p>
      </section>

      <section className="event-card" aria-labelledby="event-title">
        <h2 id="event-title">{event.title}</h2>

        <dl className="event-meta">
          <div>
            <dt>ရက်စွဲ</dt>
            <dd>{event.date}</dd>
          </div>

          <div>
            <dt>နေရာ</dt>
            <dd>{event.location}</dd>
          </div>

          <div>
            <dt>ကျန်ရှိသောနေရာ</dt>
            <dd>{remainingSeats}</dd>
          </div>
        </dl>

        <button type="button">Register လုပ်မည်</button>
      </section>
    </main>
  );
}

export default App;
```

ဒီ code မှာ React concepts အခြေခံတွေပါလာတယ်။

```text
function App()
= App component

const event
= component ထဲက local data

remainingSeats
= JavaScript calculation

return (...)
= JSX UI

className
= JSX ထဲ CSS class

{event.title}
= JavaScript value ကို JSX ထဲပြခြင်း

export default App
= main.jsx က import လုပ်နိုင်အောင် export
```

# 22. Common Errors

## Error 1 — `npm create vite@latest` မအောင်ခြင်း

ဖြစ်နိုင်တာ—

```text
Node.js version မလုံလောက်
npm internet access issue
wrong terminal folder
npm cache issue
```

စစ်ရန်—

```bash
node -v
npm -v
pwd
```

Vite docs က Node.js 20.19+ သို့ 22.12+ လိုအပ်ကြောင်းဖော်ပြထားတာကြောင့် version ကိုအရင်စစ်ပါ။ ([vitejs][2])

## Error 2 — `npm run dev` run လုပ်ရာ folder မှား

မကောင်း—

```bash
cd ~/Desktop/agentic-zero-to-hero
npm run dev
```

မှန်—

```bash
cd ~/Desktop/agentic-zero-to-hero/playground/chapter-33-react-vite-starter/react-vite-starter
npm run dev
```

`package.json` ရှိတဲ့ folder ထဲမှာ command run ရမယ်။

## Error 3 — Browser blank screen

စစ်ရန်—

```text
[ ] terminal error ရှိလား
[ ] browser console error ရှိလား
[ ] src/main.jsx import path မှန်လား
[ ] App.jsx export default App ပါလား
[ ] index.html ထဲ #root ရှိလား
```

React docs က `createRoot` target DOM node မဟုတ်ရင် “Target container is not a DOM element” error ဖြစ်နိုင်ကြောင်းဖော်ပြထားပြီး root render မခေါ်ရင်ဘာမှမပြနိုင်ကြောင်း troubleshooting section မှာရှင်းပြထားပါတယ်။ ([React][3])

## Error 4 — JSX မှာ `class` သုံးခြင်း

React JSX မှာ CSS class အတွက် `className` သုံးပါ။

မကောင်း—

```jsx
<div class="card">
```

ကောင်း—

```jsx
<div className="card">
```

## Error 5 — Component function name lowercase

React component name ကို uppercase နဲ့စပါ။

မကောင်း—

```jsx
function app() {
  return <h1>Hello</h1>;
}
```

ကောင်း—

```jsx
function App() {
  return <h1>Hello</h1>;
}
```

## Error 6 — node_modules Git ထဲပါခြင်း

စစ်ပါ—

```bash
git status
```

`node_modules/` ပေါ်နေရင် root `.gitignore` ထဲ `node_modules/` ပါ/မပါစစ်ပါ။

# 27. Common Beginner Questions

## 27.1 React app ကို HTML file double-click လုပ်လို့ရလား?

Vite React app ကို static HTML/CSS/JS lab လို `index.html` double-click နဲ့မ run သင့်ပါ။ Vite dev server နဲ့ run ပါ။

```bash
npm run dev
```

ဘာကြောင့်လဲ?

```text
Vite က module imports, JSX transform, dev server, hot reload စတာတွေစီမံပေးတယ်။
```

## 27.2 `npm install` ကိုနေ့တိုင်း run ရမလား?

မလိုပါ။ Project clone/scaffold ပြီး dependency မရှိသေးတဲ့အချိန်မှာ run ပါ။ `package.json` / lock file ပြောင်းပြီး dependencies မကိုက်တော့ရင်လည်း run လိုနိုင်တယ်။

## 27.3 `npm run dev` နဲ့ `npm run build` ကွာခြားချက်?

```text
npm run dev
= development server, coding while watching browser

npm run build
= production-ready static files generate

npm run preview
= build result ကို local preview
```

Vite docs က `npm run build` က `dist` folder ထုတ်ပြီး `npm run preview` က `dist` ကို local server နဲ့ကြည့်စမ်းနိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([vitejs][6])

## 27.4 `dist/` ကို commit လုပ်ရမလား?

ဒီ course lab အတွက် မလုပ်ပါ။ Source code ကို commit လုပ်ပါ။ Deployment chapter တွင် platform-specific output handling ကိုသီးခြားသင်မယ်။

## 27.5 React component တစ်ခုက function လား HTML လား?

React component သည် JavaScript function ဖြစ်ပြီး JSX UI ပြန်ပေးတယ်။

```jsx
function App() {
  return <h1>Hello</h1>;
}
```

# 30. Chapter Summary

ဒီအခန်းမှာ **React + Vite Starter** ကိုလေ့လာခဲ့တယ်။

အရေးကြီးဆုံး mental model—

```text
React = UI ကို component တွေအဖြစ်ရေးတဲ့ JavaScript library
Vite = React app ကို dev/build/preview လုပ်ပေးတဲ့ frontend tool
```

အရေးကြီးဆုံး commands—

```bash
npm create vite@latest react-vite-starter -- --template react
cd react-vite-starter
npm install
npm run dev
npm run build
npm run preview
```

Vite official docs က `npm create vite@latest` နဲ့ project scaffold လုပ်နိုင်ပြီး `react` template ကို support လုပ်ကြောင်း, current docs အရ Node.js 20.19+ သို့ 22.12+ လိုအပ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ `npm run build` က default `dist` output ထုတ်ပြီး `npm run preview` က build output ကို local preview လုပ်နိုင်ပေမယ့် production server မဟုတ်ကြောင်း docs ကသတိပေးထားပါတယ်။ ([vitejs][2])

အရေးကြီးဆုံး files—

```text
index.html
= browser entry point

src/main.jsx
= React root startup

src/App.jsx
= main UI component

src/App.css
= App styles

src/index.css
= global styles

package.json
= npm scripts and dependencies
```

React official docs က `createRoot` သည် browser DOM node တစ်ခုထဲ React component တွေပြဖို့ root တည်ဆောက်တဲ့ API ဖြစ်ပြီး `root.render(<App />)` ခေါ်မှ App component display ဖြစ်ကြောင်းရှင်းပြထားပါတယ်။ ([React][3])

Agentic Coding takeaway—

```text
React + Vite project တစ်ခုမှာ Agent ကိုခိုင်းတဲ့အခါ package.json, node_modules, dist, .env ကိုအလွယ်မထိခိုင်းပါနဲ့။
Allowed files သတ်မှတ်ပါ။
Plan first လုပ်ခိုင်းပါ။
npm run build / preview / browser console check နဲ့ verify လုပ်ပါ။

```
