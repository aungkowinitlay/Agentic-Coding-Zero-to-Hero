## 2. Beginner Explanation

Form ဆိုတာ user ဆီက information လက်ခံတဲ့ UI ပါ။

Event registration form မှာ—

```text
Event
Name
Phone
Email
Notes
Consent
```

တွေပါနိုင်တယ်။

Static form—

```text
User ရိုက်လို့ရ
ဒါပေမယ့် React က field values မသိ
Validation / dynamic UI လုပ်ရခက်
```

Controlled React form—

```text
User ရိုက်
  ↓
onChange run
  ↓
React state update
  ↓
Input value ကို state ကထိန်း
  ↓

```

အခြေခံ mental model—

```text

Controlled Input
= value/checked ကို React state ကထိန်းထားတဲ့ input

onChange
= User input ပြောင်းတဲ့အခါ state update လုပ်တဲ့ handler

onSubmit
= Form submit ဖြစ်တဲ့အခါ run မယ့် handler

Validation
= Input data လိုအပ်ချက်တွေနဲ့ကိုက်/မကိုက်စစ်ခြင်း

Errors
= ဘယ် field မှားတယ်၊ ဘယ်လိုပြင်ရမယ်ဆိုတာဖော်ပြခြင်း
```

# 3. Controlled Input ဆိုတာဘာလဲ

Controlled input မှာ field ရဲ့လက်ရှိ value ကို React state ကထိန်းထားတယ်။

```jsx
function NameField() {
  const [name, setName] = useState("");

  return (
    <input
      value={name}
      onChange={(event) => setName(event.target.value)}
    />
  );
}
```

ဒီ flow အတိုင်းအလုပ်လုပ်တယ်။

```text
name state = ""

User types "A"
  ↓
onChange runs
  ↓
setName("A")
  ↓
React re-renders
  ↓
input value = "A"
```

React မှာ text input ကို controlled လုပ်မယ်ဆိုရင် `value` နဲ့ `onChange` နှစ်ခုလုံးလိုပါတယ်။ `onChange` က user ရိုက်တဲ့ keystroke တစ်ခုချင်းစီမှာ run ဖြစ်ပြီး state ကို synchronous update လုပ်ရပါတယ်။ ([React][1])

# 4. Controlled vs Uncontrolled Inputs

## 4.1 Uncontrolled Input

Browser ကို value ထိန်းခိုင်းတာပါ။

```jsx
<input
  name="name"
  defaultValue="Aung Aung"
/>
```

React က initial value ပဲပေးတယ်။ နောက်ပိုင်း field value ကို browser ကထိန်းတယ်။

## 4.2 Controlled Input

React state က value ကိုထိန်းတယ်။

```jsx
const [name, setName] = useState("");

<input
  name="name"
  value={name}
  onChange={(event) => setName(event.target.value)}
/>
```

Controlled input က အသုံးဝင်တဲ့အချိန်—

```text
Validation message ပြချင်
Character count ပြချင်
Submit button enable/disable လုပ်ချင်
Input value အပေါ်မူတည်ပြီး UI ပြောင်းချင်
Form reset လုပ်ချင်
Field data ကို component တခြားတစ်ခုနဲ့ share ချင်
```

ဒီ course ရဲ့ registration form မှာ controlled inputs သုံးမယ်။

# 5. Controlled Input Rule များ

## Rule 1 — Text value ကို String နဲ့စပါ

ကောင်း—

```jsx
const [name, setName] = useState("");
```

ရှောင်ရန်—

```jsx
const [name, setName] = useState(undefined);
```

Controlled text input ရဲ့ `value` က component lifetime တစ်လျှောက် string ဖြစ်ရပါတယ်။ `undefined` သို့ `null` ကနေ string သို့ပြောင်းရင် uncontrolled-to-controlled warning ဖြစ်နိုင်ပါတယ်။ Checkbox ရဲ့ `checked` ကလည်း boolean အဖြစ်တည်ငြိမ်ရပါတယ်။ ([React][1])

## Rule 2 — Checkbox အတွက် `checked` သုံးပါ

```jsx
const [consent, setConsent] = useState(false);

<input
  type="checkbox"
  checked={consent}
  onChange={(event) => setConsent(event.target.checked)}
/>
```

Checkbox အတွက်—

```text
event.target.checked
```

ကိုဖတ်ရပါတယ်။ `event.target.value` မဟုတ်ပါဘူး။ ([React][1])

## Rule 3 — Select အတွက် `value` + `onChange`

```jsx
<select
  value={selectedEventId}
  onChange={(event) => setSelectedEventId(event.target.value)}
>
  ...
</select>
```

Controlled `<select>` မှာ `value` နဲ့ `onChange` နှစ်ခုလုံးလိုပါတယ်။ React ထဲမှာ `<option selected>` မသုံးဘဲ `<select value>` ကိုသုံးရပါတယ်။ ([React][2])

## Rule 4 — Textarea အတွက် `value` + `onChange`

```jsx
<textarea
  value={notes}
  onChange={(event) => setNotes(event.target.value)}
/>
```

React controlled textarea မှာ content ကို children အဖြစ်မရေးဘဲ `value` နဲ့ထိန်းရပါတယ်။ ([React][3])

# 6. Form State တစ်ခုတည်းနဲ့ Fields အများကြီးထိန်းခြင်း

Field တစ်ခုချင်းစီအတွက် state သီးခြားရေးနိုင်တယ်။

```jsx
const [name, setName] = useState("");
const [phone, setPhone] = useState("");
const [email, setEmail] = useState("");
```

Field များလာရင် form object တစ်ခုတည်းသုံးတာပိုရှင်းတယ်။

```jsx
const [formData, setFormData] = useState({
  eventId: "",
  name: "",
  phone: "",
  email: "",
  notes: "",
  consent: false
});
```

Generic change handler—

```jsx
function handleFieldChange(event) {
  const {
    name,
    type,
    value,
    checked
  } = event.target;

  const nextValue =
    type === "checkbox"
      ? checked
      : value;

  setFormData((currentData) => ({
    ...currentData,
    [name]: nextValue
  }));
}
```

ဒီမှာ—

```text
...currentData
= Existing fields တွေ copy လုပ်

[name]
= ပြောင်းနေတဲ့ field name ကိုရွေး

nextValue
= Text/select ဆို value
  Checkbox ဆို checked
```

ဖြစ်ပါတယ်။

# 7. `name` Attribute အရေးကြီးပုံ

Input တစ်ခုချင်းစီမှာ `name` ရှိသင့်တယ်။

```jsx

  value={formData.phone}
```


နောက်ပိုင်း—

```text
FormData
Backend request body
API validation errors
Database fields
```

တွေနဲ့လည်း field names တွေကိုတူအောင်ချိတ်နိုင်တယ်။

# 8. Form Submit

Form ကို submit လုပ်တဲ့အခါ `<form onSubmit>` ကိုသုံးပါ။

```jsx
function handleSubmit(event) {
  event.preventDefault();

  console.log("Form submitted");
}

<form onSubmit={handleSubmit}>
  ...
  <button type="submit">
    Register လုပ်မည်
  </button>
</form>
```

Browser ရဲ့ default form behavior က URL သို့ data submit လုပ်ပြီး page refresh ဖြစ်နိုင်ပါတယ်။ Client-side React demo မှာ `event.preventDefault()` သုံးပြီး အဲဒီ default behavior ကိုတားကာ validation နဲ့ state update ကိုကိုယ်တိုင်လုပ်ပါတယ်။ React ရဲ့ `onSubmit` approach က React version မရွေးအလုပ်လုပ်ပြီး submit event ကိုတိုက်ရိုက်အသုံးပြုနိုင်ပါတယ်။ ([React][4])

---

# 9. Submit ကို Button `onClick` မှာမရေးပါနဲ့

ရှောင်ရန်—

```jsx
<button
  type="button"
  onClick={handleSubmit}
>
  Register
</button>
```

ကောင်း—

```jsx
<form onSubmit={handleSubmit}>
  ...
  <button type="submit">
    Register လုပ်မည်
  </button>
</form>
```

Form `onSubmit` သုံးခြင်းရဲ့အားသာချက်—

```text
Enter key နဲ့ submit လုပ်နိုင်
Browser form semantics ရ
Submit behavior တစ်နေရာတည်းမှာထိန်းနိုင်
Button တစ်ခုထက်ပိုရှိရင် type ခွဲနိုင်
Assistive technology နဲ့ပို compatible
```

Form ထဲမှာ submit မလုပ်ရမယ့် button ကို—

```jsx
<button type="button">
  Reset
</button>
```

လို့ရေးရမယ်။ `<button>` ရဲ့ default type က form context အလိုက် submit ဖြစ်နိုင်တာကြောင့် type ကို explicit ရေးတဲ့ habit ထားပါ။

# 10. Validation ဆိုတာဘာလဲ

Validation ဆိုတာ data က product requirements နဲ့ကိုက်/မကိုက်စစ်တာပါ။

ဒီ chapter demo rules—

```text
Event
= Required
= Existing event ဖြစ်ရမယ်
= Seats ကျန်ရမယ်

Name
= Required
= Trim လုပ်ပြီး အနည်းဆုံး 2 characters

Phone
= Required
= Spaces, dashes, parentheses လက်ခံ
= Normalize လုပ်ပြီး 7–15 digits

Email
= Optional
= ထည့်ထားရင် valid email format ဖြစ်ရမယ်

Notes
= Optional
= 200 characters အများဆုံး

Consent
= Required
```

Validation rules တွေကို product SPEC ထဲမှာသတ်မှတ်သင့်တယ်။ Agent ကို သူ့ဘာသာ strict rule တွေတီထွင်ခိုင်းမထားသင့်ပါဘူး။

# 11. Validation Timing

Validation ကို ဘယ်အချိန်လုပ်မလဲဆိုတာ UX အတွက်အရေးကြီးတယ်။

## Option A — Submit အချိန်

```text
User form ဖြည့်
→ Submit
→ Errors စစ်
→ Errors ပြ
```

Beginner အတွက်အကောင်းဆုံး starting point ဖြစ်တယ်။

## Option B — Blur အချိန်

```text
User field ထဲကထွက်
→ အဲဒီ field ကိုစစ်
```

## Option C — Change အချိန်

```text
User keystroke တိုင်း
→ Field validate
```

Typing မစခင်တောင် error ပြတာမျိုးမဖြစ်စေရန်သတိထားရမယ်။

ဒီ chapter strategy—

```text
Submit အချိန်မှာ errors အားလုံး validate
User ပြန်ရိုက်တဲ့အခါ အဲဒီ field ရဲ့ error ကို clear
```

ဒီနည်းက beginner-friendly ဖြစ်ပြီး user ကို keystroke တိုင်းအနှောင့်အယှက်မဖြစ်စေဘူး။

# 12. Native Validation vs Custom Validation

HTML မှာ built-in validation attributes ရှိတယ်။

```jsx
<input required />
<input type="email" />
<input minLength={2} />
<input maxLength={80} />
```

Native validation ကို browser က handle လုပ်နိုင်တယ်။

Custom validation မှာ—

```jsx
<form noValidate onSubmit={handleSubmit}>
```

သုံးပြီး error behavior ကို React ကထိန်းနိုင်တယ်။

ဒီ chapter မှာ—

```text
required / type / maxLength
= Semantic information နဲ့ browser hints အတွက်ထား

noValidate
= Custom validation UI ကိုသင်ဖို့ browser popup ကိုပိတ်

validateRegistration()
= Actual demo validation logic

Server validation
= Production မှာမဖြစ်မနေထပ်လုပ်
```

Custom validation သုံးတဲ့အခါ errors ကို accessible ဖြစ်အောင် notification ပေးရပါတယ်။ Client-side validation တစ်ခုတည်းက security မပေးနိုင်လို့ production data ကို server-side မှာပါပြန် validate လုပ်ရပါတယ်။ ([W3C][5])

# 13. Accessible Form Requirements

Accessible registration form မှာ—

```text
[ ] Field တိုင်း visible label ရှိ
[ ] Required/optional state ကိုစာသားနဲ့ပြ
[ ] Format instructions ကြိုပေး
[ ] Error ကိုစာသားနဲ့ရှင်း
[ ] Error field ကို programmatically ချိတ်
[ ] Error summary ရှိ
[ ] Error summary က field ဆီ link ချိတ်
[ ] aria-invalid သုံး
[ ] aria-describedby သုံး
[ ] Submit success message ကြေညာ
[ ] Keyboard နဲ့အကုန်လုပ်နိုင်
```

W3C forms guidance က user input လိုတဲ့ controls တွေမှာ clear labels/instructions ပေးဖို့၊ required fields နဲ့ expected data formats ကိုကြိုပြဖို့၊ errors ကို text နဲ့ဖော်ပြဖို့အကြံပြုထားပါတယ်။ ([W3C][6])


Overall instructions ကို form မစခင်ပြသပြီး required fields, optional fields, expected formats တွေကိုရှင်းပြသင့်ပါတယ်။ Screen reader တွေဟာ form controls ကိုဖတ်တဲ့ mode သို့ပြောင်းနိုင်တာကြောင့် instructions ကို labels, legends နဲ့ descriptions တွေနဲ့ programmatically ချိတ်ထားဖို့လည်းအရေးကြီးပါတယ်။ ([W3C][6])

# 15. `aria-invalid`

Field မှားနေကြောင်း programmatically ပြဖို့—

```jsx
<input
  aria-invalid={Boolean(errors.name)}
/>
```

သုံးနိုင်တယ်။

```text
No error:
aria-invalid=false

Has error:
aria-invalid=true
```

CSS မှာလည်းသုံးနိုင်တယ်။

```css
input[aria-invalid="true"] {
  border-color: #b91c1c;
}
```

Color တစ်ခုတည်းမဟုတ်ဘဲ error text လည်းပြရမယ်။

# 16. `aria-describedby`

Field ကို help text နဲ့ error text ဆီချိတ်နိုင်တယ်။

```jsx
<input
  aria-describedby="name-help name-error"
/>
```

```jsx
<p id="name-help">
  နာမည်အပြည့်အစုံရေးပါ။
```

```jsx
function createEmptyForm() {
  return {
    eventId: "",
    name: "",
    phone: "",
    email: "",
    notes: "",
    consent: false
  };
}

function normalizePhone(value) {
  return value.replace(/[\s().-]/g, "");
}

function isValidEmail(value) {
  return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value);
}

function validateRegistration(formData, events) {
  const errors = {};
  const selectedEvent = events.find(
    (event) => event.id === formData.eventId
  );

  if (!formData.eventId) {
    errors.eventId = "Event တစ်ခုရွေးပါ။";
  } else if (!selectedEvent) {
    errors.eventId = "ရွေးထားသော Event မတွေ့ပါ။";
  } else if (
    selectedEvent.registeredCount >= selectedEvent.capacity
  ) {
    errors.eventId =
      "ရွေးထားသော Event တွင် seats မကျန်တော့ပါ။";
  }

  const trimmedName = formData.name.trim();

  if (!trimmedName) {
    errors.name = "နာမည်ဖြည့်ပါ။";
  } else if (trimmedName.length < 2) {
    errors.name =
      "နာမည်ကို အနည်းဆုံး စာလုံး ၂ လုံးရေးပါ။";
  }

  const normalizedPhone = normalizePhone(
    formData.phone
  );

  if (!formData.phone.trim()) {
    errors.phone = "ဖုန်းနံပါတ်ဖြည့်ပါ။";
  } else if (!/^\+?\d{7,15}$/.test(normalizedPhone)) {
    errors.phone =
      "ဖုန်းနံပါတ်ကို 7–15 digits ဖြင့်ရေးပါ။ Spaces, dashes နှင့် + ကိုအသုံးပြုနိုင်သည်။";
  }

  const trimmedEmail = formData.email.trim();

  if (
    trimmedEmail &&
    !isValidEmail(trimmedEmail)
  ) {
    errors.email =
      "Email format ကိုပြန်စစ်ပါ။ ဥပမာ name@example.com";
  }

  if (!formData.consent) {
    errors.consent =
      "Registration data အသုံးပြုမှုကို သဘောတူကြောင်းအတည်ပြုပါ။";
  }

  return errors;
}

function createTicketCode(eventId) {
  const eventPart = eventId
    .replace("evt-", "")
    .slice(0, 4)
    .toUpperCase();

  const timePart = Date.now()
    .toString(36)
    .toUpperCase();

  return `${eventPart}-${timePart}`;
}

function maskPhone(phone) {
  const digits = phone.replace(/\D/g, "");

  if (digits.length <= 4) {
    return digits;
  }

  return `••••••${digits.slice(-4)}`;
}

function App() {
  const [eventList, setEventList] =
    useState(initialEvents);

  const [formData, setFormData] =
    useState(createEmptyForm);

  const [formErrors, setFormErrors] =
    useState({});

  const [statusMessage, setStatusMessage] =
    useState(
      "Event တစ်ခုရွေးပြီး registration form ဖြည့်ပါ။"
    );

  const [
    lastRegistration,
    setLastRegistration
  ] = useState(null);

  const eventSelectRef = useRef(null);
  const nameRef = useRef(null);
  const phoneRef = useRef(null);
  const emailRef = useRef(null);
  const consentRef = useRef(null);

  const fieldRefs = {
    eventId: eventSelectRef,
    name: nameRef,
    phone: phoneRef,
    email: emailRef,
    consent: consentRef
  };

  const totalCapacity = eventList.reduce(
    (total, event) => total + event.capacity,
    0
  );

  const totalRegistered = eventList.reduce(
    (total, event) =>
      total + event.registeredCount,
    0
  );

  const openEventCount = eventList.filter(
    (event) =>
      event.registeredCount < event.capacity
  ).length;

  function clearFieldError(fieldName) {
    setFormErrors((currentErrors) => {
      if (!currentErrors[fieldName]) {
        return currentErrors;
      }

      const nextErrors = {
        ...currentErrors
      };

      delete nextErrors[fieldName];

      return nextErrors;
    });
  }

  function handleFieldChange(event) {
    const {
      name,
      type,
      value,
      checked
    } = event.target;

    const nextValue =
      type === "checkbox"
        ? checked
        : value;

    setFormData((currentData) => ({
      ...currentData,
      [name]: nextValue
    }));

    clearFieldError(name);
    setLastRegistration(null);
  }

  function handleChooseEvent(eventId) {
    const selectedEvent = eventList.find(
      (event) => event.id === eventId
    );

    if (!selectedEvent) {
      setStatusMessage("Event မတွေ့ပါ။");
      return;
    }

    const remainingSeats =
      selectedEvent.capacity -
      selectedEvent.registeredCount;

    if (remainingSeats <= 0) {
      setStatusMessage(
        `${selectedEvent.title} တွင် seats မကျန်တော့ပါ။`
      );
      return;
    }

    setFormData((currentData) => ({
      ...currentData,
      eventId
    }));

    clearFieldError("eventId");
    setLastRegistration(null);

    setStatusMessage(
      `${selectedEvent.title} ကိုရွေးထားသည်။ Registration form ကိုဆက်ဖြည့်ပါ။`
    );

    nameRef.current?.focus();
  }

  function handleSubmit(event) {
    event.preventDefault();

    const nextErrors = validateRegistration(
      formData,
      eventList
    );

    setFormErrors(nextErrors);

    const errorCount =
      Object.keys(nextErrors).length;

    if (errorCount > 0) {
      setLastRegistration(null);

      setStatusMessage(
        `Form တွင် error ${errorCount} ခုရှိသည်။ Error messages ကိုပြန်စစ်ပါ။`
      );

      const firstErrorField =
        FIELD_ORDER.find(
          (fieldName) =>
            nextErrors[fieldName]
        );

      fieldRefs[
        firstErrorField
      ]?.current?.focus();

      return;
    }

    const selectedEvent = eventList.find(
      (event) => event.id === formData.eventId
    );

    if (!selectedEvent) {
      setStatusMessage(
        "Selected event ကိုရှာမတွေ့ပါ။"
      );
      return;
    }

    const ticketCode = createTicketCode(
      selectedEvent.id
    );

    const nextRemainingSeats =
      selectedEvent.capacity -
      selectedEvent.registeredCount -
      1;

    setEventList((currentEvents) =>
      currentEvents.map((eventItem) => {
        if (
          eventItem.id !== selectedEvent.id
        ) {
          return eventItem;
        }

        return {
          ...eventItem,
          registeredCount:
            eventItem.registeredCount + 1
        };
      })
    );

    setLastRegistration({
      ticketCode,
      eventTitle: selectedEvent.title,
      name: formData.name.trim(),
      phone: maskPhone(formData.phone),
      email: formData.email.trim(),
      remainingSeats: nextRemainingSeats
    });

    setStatusMessage(
      `${selectedEvent.title} အတွက် registration အောင်မြင်ပါသည်။ Ticket code ${ticketCode}။`
    );

    setFormData(createEmptyForm());
    setFormErrors({});
  }

  function handleResetForm() {
    setFormData(createEmptyForm());
    setFormErrors({});
    setLastRegistration(null);

    setStatusMessage(
      "Registration form ကို reset လုပ်ပြီးပါပြီ။"
    );

    eventSelectRef.current?.focus();
  }

  return (
    <>
      <a
        className="skip-link"
        href="#main-content"
      >
        အဓိကအကြောင်းအရာသို့ ကျော်ရန်
      </a>

      <header className="site-header">
        <div className="site-header-inner">
          <p
            className="brand-name"
            lang="en"
          >
            Event Access Lab
          </p>

          <nav aria-label="စာမျက်နှာအတွင်း လင့်များ">
            <a href="#events-heading">
              Event များ
            </a>

            <a href="#registration-heading">
              Registration Form
            </a>
          </nav>
        </div>
      </header>

      <main
        id="main-content"
        className="page"
        tabIndex={-1}
      >
        <PageHeader />

        <EventSummary
          totalEvents={eventList.length}
          openEventCount={openEventCount}
          remainingSeats={
            totalCapacity - totalRegistered
          }
        />

        <p
          className="status-message"
          role="status"
          aria-atomic="true"
        >
          {statusMessage}
        </p>

        <EventList
          events={eventList}
          onChooseEvent={handleChooseEvent}
        />

        <RegistrationForm
          events={eventList}
          formData={formData}
          errors={formErrors}
          refs={fieldRefs}
          onChange={handleFieldChange}
          onSubmit={handleSubmit}
          onReset={handleResetForm}
        />

        {lastRegistration && (
          <SuccessPanel
            registration={lastRegistration}
          />
        )}
      </main>

      <footer className="site-footer">
        <p>
          Forms & Validation — controlled inputs,
          accessible errors, and submit feedback.
        </p>
      </footer>
    </>
  );
}
```

| Field            | React prop | Event value            |
| ---------------- | ---------- | ---------------------- |
| Event select     | `value`    | `event.target.value`   |
| Name input       | `value`    | `event.target.value`   |
| Phone input      | `value`    | `event.target.value`   |

| Notes textarea   | `value`    | `event.target.value`   |
| Consent checkbox | `checked`  | `event.target.checked` |

Text/select/textarea controlled fields တိုင်းမှာ `value` နဲ့ `onChange` လိုပြီး checkbox မှာ `checked` နဲ့ `onChange` လိုပါတယ်။ ([React][1])

```jsx
<input value={formData.name} />
```

`onChange` မရှိလို့ read-only ဖြစ်နေတယ်။

ပြင်ရန်—

```jsx
<input
  value={formData.name}
  onChange={handleFieldChange}
/>
```

Controlled input မှာ `value` ရှိပြီး `onChange` မရှိရင် React က read-only field warning ပေးနိုင်ပါတယ်။ ([React][1])

---

## Error 2 — Checkbox မပြောင်း

မကောင်း—

```jsx
setFormData({
  ...formData,
  consent: event.target.value
});
```

ကောင်း—

```jsx
setFormData((currentData) => ({
  ...currentData,
  consent: event.target.checked
}));
```

Checkbox အတွက် `checked` value ကိုဖတ်ရပါတယ်။ ([React][1])

---

## Error 3 — Uncontrolled to Controlled Warning

မကောင်း—

```jsx
const [formData, setFormData] = useState({
  name: undefined
});
```

ကောင်း—

```jsx
const [formData, setFormData] = useState({
  name: ""
});
```

---

## Error 4 — Submit လုပ်ရင် Page Refresh

ဖြစ်နိုင်တာ—

```text
event.preventDefault() မရှိ
onSubmit ကို form မှာမပေး
Button type မှား
```

ပြင်ရန်—

```jsx
function handleSubmit(event) {
  event.preventDefault();
}

<form onSubmit={handleSubmit}>
```

---

## Error 5 — Button တိုင်း Submit ဖြစ်

Form ထဲက non-submit button မှာ—

```jsx
type="button"
```

ထည့်ပါ။

```jsx
<button
  type="button"
  onClick={handleReset}
>
  Reset
</button>
```

---

## Error 6 — `<option selected>`

မကောင်း—

```jsx
<option selected value="evt-ai">
  AI Workshop
</option>
```

ကောင်း—

```jsx
<select
  value={formData.eventId}
  onChange={handleFieldChange}
>
```

Controlled select ရဲ့ selected option ကို `<select value>` ကထိန်းရပါတယ်။ ([React][2])

---

## Error 7 — Textarea Children သုံးခြင်း

မကောင်း—

```jsx
<textarea>
  {formData.notes}
</textarea>
```

ကောင်း—

```jsx
<textarea
  value={formData.notes}
  onChange={handleFieldChange}
/>
```

---

## Error 8 — Direct State Mutation

မကောင်း—

```jsx
formData.name = event.target.value;
setFormData(formData);
```

ကောင်း—

```jsx
setFormData((currentData) => ({
  ...currentData,
  name: event.target.value
}));
```

---

## Error 9 — Error ကို Color နဲ့ပဲပြ

မကောင်း—

```text
Input border red
Error text မရှိ
```

ကောင်း—

```text
Red border
+ aria-invalid
+ Error text
+ aria-describedby
+ Error summary
```

WCAG error identification အရ input error ကိုဖြစ်ပွားကြောင်းသာမက ဘာမှားတယ်ဆိုတာကို text နဲ့ဖော်ပြရပါတယ်။ ([W3C][11])

