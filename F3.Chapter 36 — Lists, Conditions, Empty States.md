
```jsx
{events.map((event) => (
  <EventCard key={event.id} event={event} />

```

ဒီနည်းက data array တစ်ခုရှိရင် cards တွေကို auto render လုပ်ပေးတယ်။

# 3. Lists Rendering ဆိုတာဘာလဲ

Lists rendering ဆိုတာ array ထဲက item တစ်ခုချင်းစီကို UI component အဖြစ်ပြောင်းပြတာပါ။

Data—

```jsx
const events = [
  { id: "evt-ai", title: "AI Workshop" },
  { id: "evt-react", title: "React Meetup" }
];
```

Render—

```jsx
<div>
  {events.map((event) => (
    <EventCard key={event.id} event={event} />
  ))}
</div>
```

ဒီမှာ—

```text
events.map(...)
= events array ထဲက item တစ်ခုချင်းစီကို JSX အဖြစ်ပြောင်း

<EventCard ... />
= event တစ်ခုကို card component အဖြစ်ပြ

key={event.id}
= React ကို item တစ်ခုချင်းစီကိုခွဲသိစေတဲ့ stable identity
```

React docs က `map()` ထဲက JSX elements တွေမှာ keys လိုအပ်ပြီး keys က React ကို item တစ်ခုချင်းစီကို match လုပ်နိုင်စေကာ insert/delete/reorder ဖြစ်တဲ့အခါ UI update မှန်စေကြောင်းဖော်ပြထားပါတယ်။ Data ထဲက unique id ကို key အဖြစ်သုံးသင့်ကြောင်းလည်း docs ကရှင်းပြထားပါတယ်။ ([React][1])

# 4. key prop ဘာကြောင့်လိုလဲ

React က list render လုပ်တဲ့အခါ item တစ်ခုချင်းစီကို ခွဲသိဖို့ `key` လိုတယ်။

ကောင်း—

```jsx
{events.map((event) => (
  <EventCard key={event.id} event={event} />
))}
```

မကောင်း—

```jsx
{events.map((event) => (
  <EventCard event={event} />
))}
```

Console warning တက်နိုင်တယ်—

```text
Each child in a list should have a unique "key" prop.
```

Index ကို key အဖြစ်အမြဲမသုံးသင့်ပါ။

မကောင်း—

```jsx
{events.map((event, index) => (
  <EventCard key={index} event={event} />
))}
```

ဘာကြောင့်လဲ?

```text
List item reorder / insert / delete ဖြစ်ရင် index ပြောင်းနိုင်တယ်။
React က wrong item ကို wrong UI state နဲ့ match လုပ်နိုင်တယ်။
```

ကောင်းတဲ့ key—

```text
event.id
user.id
ticketCode
database id
stable slug
```

Course rule—

```text
List render လုပ်ရင် stable unique id ကို key အဖြစ်သုံးပါ။
```

# 5. filter() နဲ့ Visible List

`filter()` က array ထဲက item တချို့ကို condition အတိုင်းရွေးပေးတယ်။

ဥပမာ open events ပဲရွေးမယ်။

```jsx
const openEvents = eventList.filter(
  (event) => event.registeredCount < event.capacity
);
```

Category filter—

```jsx
const filteredByCategory =
  selectedCategory === "All"
    ? eventList
    : eventList.filter((event) => event.category === selectedCategory);
```

Open-only + category filter နှစ်ခုတွဲ—

```jsx
const visibleEvents = showOnlyOpen
  ? filteredByCategory.filter((event) => event.registeredCount < event.capacity)
  : filteredByCategory;
```

အရေးကြီး—

```text
filteredByCategory / visibleEvents ကို state ထဲသိမ်းမထားပါ။
eventList, selectedCategory, showOnlyOpen ကနေတွက်လို့ရတဲ့ derived values ဖြစ်တယ်။
```

Derived values ကို state ထဲထပ်သိမ်းရင် data မကိုက်တဲ့ bug ဖြစ်နိုင်တယ်။

# 6. Conditional Rendering

Conditional rendering ဆိုတာ condition အပေါ်မူတည်ပြီး UI ပြောင်းပြတာပါ။

## 6.1 if statement

```jsx
function StatusBadge({ isFull }) {
  if (isFull) {
    return <span>Full</span>;
  }

  return <span>Open</span>;
}
```

## 6.2 Ternary

```jsx
<button disabled={isFull}>
  {isFull ? "Seats ပြည့်ပါပြီ" : "Register လုပ်မည်"}
</button>
```

## 6.3 Logical AND

```jsx
{remainingSeats <= 10 && <p>နေရာအနည်းငယ်သာကျန်ပါသည်။</p>}
```

React docs က condition အပေါ်မူတည်ပြီး JSX ကို return လုပ်ရန် `if`, conditional operator `? :`, logical AND `&&` တို့ကိုသုံးနိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([React][2])

Course rule—

```text
Big branch → if
Small text/value change → ternary
Optional small UI → &&
```

# 7. Empty / Loading / Error States

Real app တွေမှာ data ရှိတဲ့အချိန်ပဲမဟုတ်ပါဘူး။

```text
Loading
= data မရသေး

Error
= data load မအောင်မြင်

Empty
= data မရှိသေး

No results
= data ရှိပေမယ့် filter နဲ့မကိုက်
```

ဒီ states တွေကို UI ထဲမှာ handle မလုပ်ရင် user က app ပျက်တယ်လို့ထင်နိုင်တယ်။

## 7.1 Loading State

```jsx
if (viewStatus === "loading") {
  return <LoadingState message="Event များကိုဖွင့်နေပါသည်..." />;
}
```

## 7.2 Error State

```jsx
if (viewStatus === "error") {
  return (
    <ErrorState
      title="Event များကိုဖွင့်မရပါ။"
      message="နောက်တစ်ကြိမ်ပြန်စမ်းပါ။"
      onRetry={onRestoreView}
    />
  );
}
```

## 7.3 Empty State

```jsx
if (events.length === 0) {
  return (
    <EmptyState
      title="ကိုက်ညီသော Event မရှိပါ။"
      message="Filter ကိုပြန်ပြောင်းပါ။"
      actionText="Filters reset"
      onAction={onResetFilters}
    />
  );
}
```

Empty state က user ကိုဘာလုပ်ရမလဲပြောသင့်တယ်။

မကောင်း—

```text
No data
```

ကောင်း—

```text
ကိုက်ညီသော Event မရှိပါ။
Category filter သို့ open-only filter ကိုပြန်ပြောင်းပါ။
[Filters reset]
```

`playground/chapter-33-react-vite-starter/react-vite-starter/src/App.jsx` ကိုအောက်ပါ code နဲ့ replace လုပ်ပါ။

```jsx
import { useState } from "react";
import "./App.css";

const initialEvents = [
  {
    id: "evt-ai-workshop",
    title: "AI Workshop",
    date: "2026-08-01",
    location: "Yangon",
    category: "AI",
    capacity: 100,
    registeredCount: 25
  },
  {
    id: "evt-react-meetup",
    title: "React Meetup",
    date: "2026-08-15",
    location: "Mandalay",
    category: "Frontend",
    capacity: 80,
    registeredCount: 72
  },
  {
    id: "evt-pwa-checkin",
    title: "PWA Check-in Lab",
    date: "2026-09-05",
    location: "Online",
    category: "Mobile Web",
    capacity: 60,
    registeredCount: 60
  },
  {
    id: "evt-backend-api",
    title: "Backend API Bootcamp",
    date: "2026-09-20",
    location: "Naypyidaw",
    category: "Backend",
    capacity: 90,
    registeredCount: 12
  },
  {
    id: "evt-ux-a11y",
    title: "UX Accessibility Clinic",
    date: "2026-10-03",
    location: "Yangon",
    category: "UX",
    capacity: 40,
    registeredCount: 40
  }
];

const ALL_CATEGORIES = "All";

function App() {
  const [eventList, setEventList] = useState(initialEvents);
  const [message, setMessage] = useState("Filter ကိုရွေးပြီး event list ကိုစစ်ပါ။");
  const [showOnlyOpen, setShowOnlyOpen] = useState(false);
  const [selectedCategory, setSelectedCategory] = useState(ALL_CATEGORIES);
  const [viewStatus, setViewStatus] = useState("ready");

  const categories = [
    ALL_CATEGORIES,
    ...new Set(eventList.map((event) => event.category))
  ];

  const totalCapacity = eventList.reduce((total, event) => total + event.capacity, 0);
  const totalRegistered = eventList.reduce(
    (total, event) => total + event.registeredCount,
    0
  );

  const filteredByCategory =
    selectedCategory === ALL_CATEGORIES
      ? eventList
      : eventList.filter((event) => event.category === selectedCategory);

  const openEvents = eventList.filter(
    (event) => event.registeredCount < event.capacity
  );

  const visibleEvents = showOnlyOpen
    ? filteredByCategory.filter((event) => event.registeredCount < event.capacity)
    : filteredByCategory;

  const hasActiveFilters = selectedCategory !== ALL_CATEGORIES || showOnlyOpen;

  function handleRegister(eventId) {
    const selectedEvent = eventList.find((event) => event.id === eventId);

    if (!selectedEvent) {
      setMessage("Event မတွေ့ပါ။");
      return;
    }

    const remainingSeats = selectedEvent.capacity - selectedEvent.registeredCount;

    if (remainingSeats <= 0) {
      setMessage(`${selectedEvent.title} သည် seats ပြည့်ပြီးပါပြီ။`);
      return;
    }

    setEventList((currentEvents) =>
      currentEvents.map((event) => {
        if (event.id !== eventId) {
          return event;
        }

        return {
          ...event,
          registeredCount: event.registeredCount + 1
        };
      })
    );

    setMessage(`${selectedEvent.title} အတွက် registration အောင်မြင်ပါသည်။`);
    setViewStatus("ready");
  }

  function handleSelectCategory(category) {
    setSelectedCategory(category);
    setViewStatus("ready");

    if (category === ALL_CATEGORIES) {
      setMessage("Category filter ကိုဖယ်ရှားပြီးပါပြီ။");
      return;
    }

    setMessage(`${category} category ကိုရွေးထားပါတယ်။`);
  }

  function handleToggleOpenOnly() {
    setShowOnlyOpen((currentValue) => !currentValue);
    setViewStatus("ready");
    setMessage("Open-only filter ကိုပြောင်းထားပါတယ်။");
  }

  function handleResetFilters() {
    setSelectedCategory(ALL_CATEGORIES);
    setShowOnlyOpen(false);
    setViewStatus("ready");
    setMessage("Filters reset လုပ်ပြီးပါပြီ။");
  }

  function handleResetData() {
    setEventList(initialEvents);
    setSelectedCategory(ALL_CATEGORIES);
    setShowOnlyOpen(false);
    setViewStatus("ready");
    setMessage("Event data ကို reset လုပ်ပြီးပါပြီ။");
  }

  function handleSimulateLoading() {
    setViewStatus("loading");
    setMessage("Loading state ကိုစမ်းနေပါတယ်။");
  }

  function handleSimulateError() {
    setViewStatus("error");
    setMessage("Error state ကိုစမ်းနေပါတယ်။");
  }

  function handleRestoreView() {
    setViewStatus("ready");
    setMessage("Event list ကိုပြန်ဖွင့်ပြီးပါပြီ။");
  }

  return (
    <main className="page">
      <PageHeader
        eyebrow="Part 6 — Frontend Foundation"
        title="Lists, Conditions, Empty States"
        description="React ထဲမှာ array data ကို list UI အဖြစ်ပြခြင်း၊ filter လုပ်ခြင်း၊ condition အလိုက် UI ပြောင်းခြင်းကိုလေ့ကျင့်ပါမယ်။"
      />

      <StatsSummary
        totalEvents={eventList.length}
        totalCapacity={totalCapacity}
        totalRegistered={totalRegistered}
        openEventCount={openEvents.length}
      />

      <ActionPanel
        categories={categories}
        message={message}
        selectedCategory={selectedCategory}
        showOnlyOpen={showOnlyOpen}
        viewStatus={viewStatus}
        onSelectCategory={handleSelectCategory}
        onToggleOpenOnly={handleToggleOpenOnly}
        onResetFilters={handleResetFilters}
        onResetData={handleResetData}
        onSimulateLoading={handleSimulateLoading}
        onSimulateError={handleSimulateError}
        onRestoreView={handleRestoreView}
      />

      <EventContent
        events={visibleEvents}
        openEventCount={openEvents.length}
        hasActiveFilters={hasActiveFilters}
        viewStatus={viewStatus}
        onRegister={handleRegister}
        onResetFilters={handleResetFilters}
        onRestoreView={handleRestoreView}
      />
    </main>
  );
}

function PageHeader({ eyebrow, title, description }) {
  return (
    <section className="hero">
      <p className="eyebrow">{eyebrow}</p>
      <h1>{title}</h1>
      <p>{description}</p>
    </section>
  );
}

function StatsSummary({
  totalEvents,
  totalCapacity,
  totalRegistered,
  openEventCount
}) {
  const remainingSeats = totalCapacity - totalRegistered;

  return (
    <section className="stats" aria-label="Event summary">
      <StatCard label="Events" value={totalEvents} />
      <StatCard label="Open Events" value={openEventCount} />
      <StatCard label="Remaining Seats" value={remainingSeats} />
    </section>
  );
}

function StatCard({ label, value }) {
  return (
    <article className="stat-card">
      <p>{label}</p>
      <strong>{value}</strong>
    </article>
  );
}

function ActionPanel({
  categories,
  message,
  selectedCategory,
  showOnlyOpen,
  viewStatus,
  onSelectCategory,
  onToggleOpenOnly,
  onResetFilters,
  onResetData,
  onSimulateLoading,
  onSimulateError,
  onRestoreView
}) {
  return (
    <section className="action-panel" aria-label="Event filters and view controls">
      <p className="status-message">{message}</p>

      <CategoryFilter
        categories={categories}
        selectedCategory={selectedCategory}
        onSelectCategory={onSelectCategory}
      />

      <div className="action-row">
        <button
          className="secondary-button"
          type="button"
          onClick={onToggleOpenOnly}
        >
          {showOnlyOpen ? "Event အားလုံးပြမည်" : "Open events ပဲပြမည်"}
        </button>

        <button className="secondary-button" type="button" onClick={onResetFilters}>
          Filters reset
        </button>

        <button className="secondary-button" type="button" onClick={onResetData}>
          Data reset
        </button>

        <button className="secondary-button" type="button" onClick={onSimulateLoading}>
          Loading state
        </button>

        <button className="secondary-button" type="button" onClick={onSimulateError}>
          Error state
        </button>

        {viewStatus !== "ready" && (
          <button className="secondary-button" type="button" onClick={onRestoreView}>
            Event list ပြန်ဖွင့်မည်
          </button>
        )}
      </div>
    </section>
  );
}

function CategoryFilter({ categories, selectedCategory, onSelectCategory }) {
  return (
    <div className="category-filter" aria-label="Category filter">
      {categories.map((category) => (
        <button
          key={category}
          className={
            selectedCategory === category
              ? "filter-chip filter-chip-active"
              : "filter-chip"
          }
          type="button"
          onClick={() => onSelectCategory(category)}
        >
          {category}
        </button>
      ))}
    </div>
  );
}

function EventContent({
  events,
  openEventCount,
  hasActiveFilters,
  viewStatus,
  onRegister,
  onResetFilters,
  onRestoreView
}) {
  if (viewStatus === "loading") {
    return <LoadingState message="Event များကိုဖွင့်နေပါသည်..." />;
  }

  if (viewStatus === "error") {
    return (
      <ErrorState
        title="Event များကိုဖွင့်မရပါ။"
        message="ဒီဟာက error state demo ဖြစ်ပါတယ်။ Event list ကိုပြန်ဖွင့်နိုင်ပါတယ်။"
        onRetry={onRestoreView}
      />
    );
  }

  return (
    <EventList
      events={events}
      openEventCount={openEventCount}
      hasActiveFilters={hasActiveFilters}
      onRegister={onRegister}
      onResetFilters={onResetFilters}
    />
  );
}

function EventList({ events, openEventCount, hasActiveFilters, onRegister, onResetFilters }) {
  if (events.length === 0) {
    return (
      <EmptyState
        title={hasActiveFilters ? "ကိုက်ညီသော Event မရှိပါ။" : "Event မရှိသေးပါ။"}
        message={
          hasActiveFilters
            ? "Category filter သို့ open-only filter ကိုပြန်ပြောင်းပါ။"
            : "Event data ထည့်ပြီးမှ list ပေါ်လာပါမယ်။"
        }
        actionText={hasActiveFilters ? "Filters reset" : undefined}
        onAction={hasActiveFilters ? onResetFilters : undefined}
      />
    );
  }

  return (
    <section className="event-section" aria-labelledby="events-heading">
      <div className="section-header">
        <div>
          <p className="eyebrow">Upcoming Events</p>
          <h2 id="events-heading">လာမည့် Event များ</h2>
        </div>

        <p className="section-note">
          Register လုပ်နိုင်သော event {openEventCount} ခု ရှိသည်။
        </p>
      </div>

      <div className="event-grid">
        {events.map((event) => (
          <EventCard key={event.id} event={event} onRegister={onRegister} />
        ))}
      </div>
    </section>
  );
}

function EventCard({ event, onRegister }) {
  const remainingSeats = event.capacity - event.registeredCount;
  const isFull = remainingSeats === 0;

  return (
    <article className="event-card" aria-labelledby={`${event.id}-title`}>
      <div className="event-card-header">
        <div>
          <p className="event-category">{event.category}</p>
          <h3 id={`${event.id}-title`}>{event.title}</h3>
        </div>

        <StatusBadge isFull={isFull} remainingSeats={remainingSeats} />
      </div>

      <dl className="event-meta">
        <EventMetaItem label="ရက်စွဲ" value={event.date} />
        <EventMetaItem label="နေရာ" value={event.location} />
        <EventMetaItem label="ကျန်ရှိသောနေရာ" value={remainingSeats} />
      </dl>

      {remainingSeats > 0 && remainingSeats <= 10 && (
        <p className="few-seats-note">နေရာအနည်းငယ်သာကျန်ပါသည်။</p>
      )}

      <PrimaryButton disabled={isFull} onClick={() => onRegister(event.id)}>
        {isFull ? "Seats ပြည့်ပါပြီ" : "Register လုပ်မည်"}
      </PrimaryButton>
    </article>
  );
}

function EventMetaItem({ label, value }) {
  return (
    <div>
      <dt>{label}</dt>
      <dd>{value}</dd>
    </div>
  );
}

function StatusBadge({ isFull, remainingSeats }) {
  if (isFull) {
    return <span className="status-badge status-badge-full">Full</span>;
  }

  if (remainingSeats <= 10) {
    return <span className="status-badge status-badge-warning">Few seats</span>;
  }

  return <span className="status-badge status-badge-open">Open</span>;
}

function PrimaryButton({ disabled = false, onClick, children }) {
  return (
    <button
      className="primary-button"
      type="button"
      disabled={disabled}
      onClick={onClick}
    >
      {children}
    </button>
  );
}

function LoadingState({ message }) {
  return (
    <section className="state-card state-card-loading" aria-live="polite">
      <div className="loading-dot" aria-hidden="true" />
      <h2>Loading</h2>
      <p>{message}</p>
    </section>
  );
}

function ErrorState({ title, message, onRetry }) {
  return (
    <section className="state-card state-card-error" role="alert">
      <h2>{title}</h2>
      <p>{message}</p>
      <button className="secondary-button" type="button" onClick={onRetry}>
        ပြန်စမ်းမည်
      </button>
    </section>
  );
}

function EmptyState({ title, message, actionText, onAction }) {
  return (
    <section className="state-card empty-state">
      <h2>{title}</h2>
      <p>{message}</p>

      {actionText && onAction && (
        <button className="secondary-button" type="button" onClick={onAction}>
          {actionText}
        </button>
      )}
    </section>
  );
}
```

# 10. ဒီ Code ထဲက Main Concepts

## 10.1 Category list ကို data ကနေ derive လုပ်တယ်

```jsx
const categories = [
  ALL_CATEGORIES,
  ...new Set(eventList.map((event) => event.category))
];
```

ဒီ code က event data ထဲက categories တွေကိုထုတ်တယ်။ Duplicate category မထပ်အောင် `Set` သုံးတယ်။

```text
eventList
→ category values
→ unique categories
→ filter buttons
```

## 10.2 Category buttons ကို map() နဲ့ render လုပ်တယ်

```jsx
{categories.map((category) => (
  <button key={category} ...>
    {category}
  </button>
))}
```

ဒီမှာ `category` string က unique ဖြစ်လို့ key အဖြစ်သုံးထားတယ်။

## 10.3 Visible events ကို filter() နဲ့တွက်တယ်

```jsx
const filteredByCategory =
  selectedCategory === ALL_CATEGORIES
    ? eventList
    : eventList.filter((event) => event.category === selectedCategory);
```

```jsx
const visibleEvents = showOnlyOpen
  ? filteredByCategory.filter((event) => event.registeredCount < event.capacity)
  : filteredByCategory;
```

ဒီ values တွေကို state ထဲမသိမ်းပါ။ Existing state ကနေတွက်လို့ရတဲ့ derived values ဖြစ်တယ်။

## 10.4 EventContent က Loading / Error / Ready ကိုခွဲတယ်

```jsx
if (viewStatus === "loading") {
  return <LoadingState message="Event များကိုဖွင့်နေပါသည်..." />;
}

if (viewStatus === "error") {
  return <ErrorState ... />;
}

return <EventList ... />;
```

ဒီ pattern က production app တွေမှာအလွန်အသုံးဝင်တယ်။

```text
loading → spinner/skeleton
error → retry UI
ready → actual data list
empty → no data message
```

## 10.5 EventList က Empty State ကို handle လုပ်တယ်

```jsx
if (events.length === 0) {
  return <EmptyState ... />;
}
```

List component တစ်ခုက data မရှိတဲ့အချိန်ကိုမဖြစ်မနေ handle သင့်တယ်။

## 10.6 EventCard က condition အလိုက် badge / button ပြောင်းတယ်

```jsx
const isFull = remainingSeats === 0;
```

```jsx
<PrimaryButton disabled={isFull}>
  {isFull ? "Seats ပြည့်ပါပြီ" : "Register လုပ်မည်"}
</PrimaryButton>
```

# 14. Empty State Design Rules

Empty state က “ဘာမှမရှိ” ဆိုတာကိုပြရုံမဟုတ်ပါဘူး။ User ကို next action ပေးသင့်တယ်။

Good empty state includes—

```text
[ ] Clear title
[ ] Helpful message
[ ] Optional action button
[ ] No blame language
[ ] Current filter/state context
```

Bad—

```text
No data.
```

Good—

```text
ကိုက်ညီသော Event မရှိပါ။
Category filter သို့ open-only filter ကိုပြန်ပြောင်းပါ။
[Filters reset]
```

Course rule—

```text
Empty state should explain why the list is empty and what the user can do next.
```

---

# 15. Loading State Design Rules

Loading state က data ရောက်ဖို့စောင့်နေကြောင်း user ကိုပြတယ်။

Good loading state—

```text
[ ] Short message
[ ] Visual indicator
[ ] No fake error
[ ] aria-live where useful
```

Example—

```jsx
<section className="state-card state-card-loading" aria-live="polite">
  <div className="loading-dot" aria-hidden="true" />
  <h2>Loading</h2>
  <p>Event များကိုဖွင့်နေပါသည်...</p>
</section>
```

`aria-live="polite"` က screen reader user အတွက် update message ကို non-intrusive ဖော်ပြစေတယ်။

---

# 16. Error State Design Rules

Error state က user ကို “ဘာဖြစ်သွားလဲ” နဲ့ “ဘာလုပ်လို့ရလဲ” ပြရမယ်။

Good error state—

```text
[ ] Error title
[ ] Short user-friendly message
[ ] Retry / restore action
[ ] No secret/internal stack trace
[ ] Developer details only in logs, not public UI
```

Bad—

```text
TypeError: Cannot read properties of undefined...
```

Good—

```text
Event များကိုဖွင့်မရပါ။
နောက်တစ်ကြိမ်ပြန်စမ်းပါ။
[ပြန်စမ်းမည်]
```

# 17. Common Errors

## Error 1 — map() ထဲ key မပါ

မကောင်း—

```jsx
{events.map((event) => (
  <EventCard event={event} />
))}
```

ကောင်း—

```jsx
{events.map((event) => (
  <EventCard key={event.id} event={event} />
))}
```

## Error 2 — filter result ကို state ထဲသိမ်းပြီး original list ပျောက်

မကောင်း—

```jsx
setEventList(eventList.filter((event) => event.category === "AI"));
```

ဒီလိုလုပ်ရင် original events ပျောက်သွားနိုင်တယ်။

ကောင်း—

```jsx
const visibleEvents =
  selectedCategory === "All"
    ? eventList
    : eventList.filter((event) => event.category === selectedCategory);
```

## Error 3 — Empty state မထား

မကောင်း—

```jsx
<div className="event-grid">
  {events.map(...)}
</div>
```

`events.length === 0` ဖြစ်ရင် blank area ဖြစ်သွားတယ်။

ကောင်း—

```jsx
if (events.length === 0) {
  return <EmptyState ... />;
}
```

## Error 4 — `&&` နဲ့ number 0 ပေါ်လာနိုင်ခြင်း

မကောင်း—

```jsx
{events.length && <EventList events={events} />}
```

`events.length` သည် 0 ဖြစ်ရင် UI ထဲ `0` ပေါ်နိုင်တယ်။

ကောင်း—

```jsx
{events.length > 0 && <EventList events={events} />}
```

သို့မဟုတ်—

```jsx
if (events.length === 0) {
  return <EmptyState />;
}
```

## Error 5 — Loading/Error state ထဲ에서도 list render လုပ်နေခြင်း

State branch ကိုရှင်းအောင်ခွဲပါ။

```jsx
if (viewStatus === "loading") return <LoadingState />;
if (viewStatus === "error") return <ErrorState />;
return <EventList />;
```

# 23. Common Beginner Questions

## 23.1 filter result ကို state ထဲသိမ်းလို့မရဘူးလား?

ရတယ်, ဒါပေမယ့် beginner အတွက်မကောင်းပါ။ Filter result ကို state ထဲသိမ်းရင် original data ပျောက်နိုင်တယ်။ အကောင်းဆုံးက source data ကို `eventList` state ထဲထားပြီး `selectedCategory`, `showOnlyOpen` ကနေ `visibleEvents` ကိုတွက်ပါ။

```jsx
const visibleEvents = showOnlyOpen ? openEvents : filteredByCategory;
```

## 23.2 Empty state ကို EventList ထဲထားသင့်လား App ထဲထားသင့်လား?

Beginner rule—

```text
List-specific empty state → EventList ထဲထား
Page-level loading/error state → EventContent/App level ထဲထား
```

## 23.3 Loading/Error state ကိုတကယ် API မရှိဘဲဘာကြောင့်သင်လဲ?

နောက်ပိုင်း backend API ချိတ်တဲ့အခါ loading/error states မဖြစ်မနေလိုလာမယ်။ အခု frontend-only demo မှာ simulate လုပ်ပြီး mental model တည်ဆောက်ထားတာပါ။

## 23.4 `&&` ကို ဘယ်အချိန်သုံးသင့်လဲ?

Optional small UI အတွက်သုံးပါ။

```jsx
{remainingSeats <= 10 && <p>Few seats</p>}
```

Big branch, loading/error/empty state တို့အတွက် `if` early return ကပိုရှင်းတယ်။
