## 2. Beginner Explanation

### Agentic Coding ကို အလွယ်ဆုံးနားလည်မယ်ဆိုရင်

Agentic Coding ဆိုတာ—

**
AI ကို code တစ်ကြောင်းချင်းမေးတာမဟုတ်ဘဲ
AI Agent ကို project folder ထဲမှာ အလုပ်တစ်ခုကို အဆင့်လိုက်လုပ်ခိုင်းတာပါ။
*



ဥပမာ သင်က coding မသိသေးဘူးဆိုပါစို့။ သင့်ဆိုင်အတွက် website တစ်ခုလိုချင်တယ်။ ရိုးရိုး ChatGPT ကို မေးရင်—

**
“ဆိုင် website အတွက် HTML code ရေးပေးပါ”
*

လို့မေးမယ်။ ChatGPT က code * ပေးမယ်။ သင်က copy-paste လုပ်ရမယ်။ Error တက်ရင် ဘာဖြစ်မှန်း မသိနိုင်ဘူး။

Agentic Coding မှာတော့—

**
“ဒီ project folder ထဲမှာ product list page တစ်ခုဆောက်ပါ။
မပြင်ခင် plan ပြပါ။
ပြီးရင် npm run build စစ်ပါ။
ဘာ file တွေပြင်ခဲ့လဲ summary ပေးပါ။”
*

လို့ခိုင်းမယ်။

ဒီနေရာမှာ Agent က—

**
1. Project file တွေဖတ်မယ်
2. Plan ဆွဲမယ်
3. File တွေပြင်မယ်
4. Command run မယ်
5. Error ရှိရင်ပြင်မယ်
6. Test/build စစ်မယ်
7. Summary ပြန်ပေးမယ်
*

အဲဒီလို အလုပ်လုပ်လာတဲ့အခါ “AI နဲ့ chat” မဟုတ်တော့ဘဲ **AI နဲ့ software project တည်ဆောက်ခြင်း** ဖြစ်လာပါတယ်။

Claude Code official docs မှာ Claude Code ကို codebase ဖတ်နိုင်၊ file edit လုပ်နိုင်၊ command run နိုင်ပြီး terminal, IDE, desktop app, browser ထဲကနေသုံးနိုင်တဲ့ agentic coding tool အဖြစ်ဖော်ပြထားပါတယ်။ အဲဒီ docs ထဲမှာ feature build, bug fix, tests, git workflows, MCP integration, CLAUDE.md, skills, hooks, multiple agents စတာတွေကိုလည်း development workflow အနေနဲ့ဖော်ပြထားပါတယ်။ ([Claude][1])

## 3. Key Concepts

## 3.1 AI Chat နဲ့ Coding Agent ကွာခြားချက်

AI Chat က သင့်ကို “အကြံပေး” နိုင်တယ်။ Coding Agent ကတော့ project ထဲမှာ “အလုပ်လုပ်” နိုင်တယ်။

| အမျိုးအစား         | လုပ်နိုင်တာ                                              | မလုပ်နိုင်တာ                                 |
| ------------------ | -------------------------------------------------------- | -------------------------------------------- |
| AI Chat            | code example ပေးနိုင်၊ explain လုပ်နိုင်                 | ကိုယ့် project file ကို တိုက်ရိုက်မပြင်နိုင် |
| Autocomplete       | ကိုယ်ရေးနေတဲ့ code ကိုဆက်ရေးပေးနိုင်                     | project တစ်ခုလုံး plan မဆွဲနိုင်             |
| Editor Agent       | editor ထဲက file တွေပြင်နိုင်                             | command/test/deploy workflow ကန့်သတ်နိုင်    |
| CLI Agent          | terminal ထဲကနေ file ဖတ်၊ edit, command run နိုင်         | user review မရှိရင် အမှားလုပ်နိုင်           |
| Coding Agent       | feature တစ်ခုကို plan → code → verify loop နဲ့လုပ်နိုင်  | product owner တာဝန်ကို အစားမထိုးနိုင်        |
| Multi-Agent System | role ခွဲပြီး Planner, Coder, Verifier, Auditor လုပ်နိုင် | orchestration မကောင်းရင် ရှုပ်ထွေးနိုင်      |

ဒီ course မှာ သင်က နောက်ဆုံးမှာ **Coding Agent ကိုစနစ်တကျခိုင်းနိုင်သူ** ဖြစ်လာရပါမယ်။ နောက်ဆုံးပိုင်းမှာတော့ **Subagents / Verifier / Auditor / Deployer** တွေနဲ့ multi-agent workflow အထိတက်သွားမယ်။

Claude Agent SDK docs ထဲမှာ subagents တွေကို main agent က focused subtasks တွေကိုလုပ်ဖို့ spawn လုပ်နိုင်တဲ့ separate agent instances အဖြစ်ဖော်ပြထားပြီး con* isolate လုပ်ခြင်း၊ parallel analyses လုပ်ခြင်း၊ specialized instructions သုံးခြင်းတို့အတွက်အသုံးပြုနိုင်တယ်လို့ရှင်းပြထားပါတယ်။ ([Claude][2])

သင် = Product Owner + Reviewer + Tester

Project = အလုပ်လုပ်ရမယ့်နေရာ

**
ဘာလိုချင်လဲ ဆုံးဖြတ်သူ
ဘယ်အတိုင်းအတာထိလုပ်မလဲ သတ်မှတ်သူ

ပြီး/မပြီး ဆုံးဖြတ်သူ
*

ဖြစ်ပါတယ်။

AI Agent က—

**
file ဖတ်သူ

test run သူ
bug fix သူ
summary ပေးသူ
*

ဖြစ်ပါတယ်။

ဒါကြောင့် Agentic Coding မှာ အမြဲမှတ်ရမယ့်စကားက—

**
AI က အလုပ်လုပ်နိုင်တယ်။
ဒါပေမယ့် တာဝန်ယူရမှာက သင်ပါ။

## 3.3 Agentic Loop

Agentic Coding မှာ အလုပ်တစ်ခုကို တစ်ခါတည်း “ပြီးအောင်” မလုပ်ပါဘူး။ Loop နဲ့လုပ်ပါတယ်။

**
1. Understand
2. Plan
3. Implement
4. Run
5. Test
6. Fix
7. Review
8. Commit
*

မြန်မာလိုပြန်ပြောရင်—

**
နားလည် → အစီအစဉ်ဆွဲ → code ရေး → run ကြည့် → စစ် → ပြင် → review → သိမ်း
*

ဥပမာ feature တစ်ခုဆောက်မယ်ဆိုရင် prompt က ဒီလိုဖြစ်သင့်တယ်။

**
ဒီ project မှာ Event List page ဆောက်ချင်ပါတယ်။

ပထမဆုံး code မရေးပါနဲ့။
SPEC.md ရဲ့ Event List section ကိုဖတ်ပါ။
ဘယ် file တွေပြင်မလဲ plan ပြပါ။
ကျွန်တော် approve လုပ်မှ implement လုပ်ပါ။
ပြီးရင် npm run build run ပြီး result ပြပါ။
*

ဒီ prompt ထဲမှာ Agent ကို “ချက်ချင်း code ရေး” လို့မခိုင်းဘဲ **plan အရင်ပြခိုင်းထားတယ်**။ ဒီလိုလုပ်တာက beginner အတွက် အလွန်အရေးကြီးပါတယ်။

## 3.4 “Done” ဆိုတာ ဘာလဲ

Agent က “ပြီးပြီ” လို့ပြောတာနဲ့ project တကယ်ပြီးတာ မဟုတ်ပါဘူး။

Agentic Coding မှာ “Done” ဆိုတာ—

**
[ ] Code ရေးပြီး
[ ] App run လို့ရပြီး
[ ] Build pass ဖြစ်ပြီး
[ ] Test pass ဖြစ်ပြီး
[ ] Acceptance criteria ကိုက်ပြီး
[ ] User flow ကိုစမ်းပြီး
[ ] Git commit လုပ်ဖို့ ready ဖြစ်ပြီး
*

မှ “Done” ပါ။

ဒီ course မှာ feature တစ်ခုချင်းစီအတွက် **Definition of Done** ရေးတတ်အောင် သင်မယ်။

ဥပမာ—

**
Definition of Done — Event Registration Form

[ ] Name, phone, email fields ပါရမယ်
[ ] Required validation ပါရမယ်
[ ] Submit ပြီးရင် ticket code ပြရမယ်
[ ] Error message မြန်မာလိုပြရမယ်
[ ] Mobile layout မပျက်ရဘူး
[ ] npm run build pass ဖြစ်ရမယ်
[ ] Registration API test pass ဖြစ်ရမယ်
*

## 3.5 Con* ဆိုတာဘာလဲ

AI Agent တစ်ယောက်က project ကိုနားလည်ဖို့ con* လိုတယ်။

Con* ဆိုတာ—

**
Project ရဲ့ အကြောင်းအရာ
File structure
Tech stack
SPEC
Current task
Relevant code
Rules
Past decisions
*

တွေပါ။

မကောင်းတဲ့ prompt—

**
ဒီ project ကိုပြင်ပေး
*

ကောင်းတဲ့ prompt—

**
ဒီ task က Event Registration form validation ကိုပြင်ဖို့ပါ။

ဖတ်ရမယ့် files:
- SPEC.md section 3.2
- apps/web/src/pages/RegisterPage.jsx
- packages/shared/src/validators/registration.ts

မပြင်ရမယ့် files:
- deployment files
- admin pages
- mobile/PWA app

Goal:
Phone number validation ကို Myanmar phone format နဲ့ကိုက်အောင်ပြင်ပါ။
*

Agentic Coding ရဲ့ကျွမ်းကျင်မှုက “AI ကိုမေးတတ်တာ” တစ်ခုတည်းမဟုတ်ဘူး။ **AI ကို con* မှန်မှန်ပေးတတ်တာ** က ပိုအရေးကြီးတယ်။

## 3.6 Memory ဆိုတာဘာလဲ

Agent တွေမှာ session တစ်ခုအတွင်းမှတ်ထားတဲ့ con* ရှိသလို project-level memory လည်းရှိနိုင်တယ်။

Claude Code မှာ `CLAUDE.md` ကို project root ထဲထားပြီး coding standards, architecture decisions, preferred libraries, review checklists စတာတွေသတ်မှတ်နိုင်ကြောင်း official docs မှာဖော်ပြထားပါတယ်။ အလားတူ Codex official docs မှာလည်း Codex က task မစခင် `AGENTS.md` files တွေဖတ်ပြီး global guidance နဲ့ project-specific instructions တွေကို layer လုပ်နိုင်တယ်လို့ဖော်ပြထားပါတယ်။ ([Claude][1]) ([OpenAI Developers][3])

ဒီ course မှာနောက်ပိုင်း သင်ရေးရမယ့် memory files တွေက—

**
CLAUDE.md   → Claude Code အတွက် project memory
AGENTS.md   → Codex / generic agents အတွက် project instructions
SPEC.md     → project ရဲ့ single source of truth
REPO_MAP.md → codebase navigation map
*

## 3.7 Tools ဆိုတာဘာလဲ

Agent တစ်ယောက်က စကားပြောနိုင်ရုံနဲ့ coding agent မဖြစ်သေးပါဘူး။ Tools လိုတယ်။

Tools ဆိုတာ—

**
Read file
Write file
Edit file
Search files
Run terminal command
Run tests
Run build
Use git
Call APIs
Use MCP tools
*

တွေပါ။

Tool များလာလေ Agent အစွမ်းကြီးလာလေပါ။ ဒါပေမယ့် အန္တရာယ်လည်းကြီးလာတယ်။

ဥပမာ—

**
npm install package
rm -rf folder
edit .env
deploy production
delete database
*

လို command တွေက သတိထားရမယ်။

ဒါကြောင့် ဒီ course မှာ “Agent ကိုခိုင်းတတ်ခြင်း” နဲ့အတူ **Agent ကိုထိန်းတတ်ခြင်း** ကိုပါသင်မယ်။

Claude Code hooks docs အရ hooks တွေက Claude Code lifecycle point တွေမှာ automatic run နိုင်ပြီး `PreToolUse` က tool call မလုပ်ခင် block လုပ်နိုင်တယ်၊ `PostToolUse` က tool call အောင်မြင်ပြီးနောက် run နိုင်တယ်၊ `PermissionRequest` က permission dialog ပေါ်ချိန် decision ပေးနိုင်တယ်လို့ဖော်ပြထားပါတယ်။ ([Claude][4])

## 4. Agentic Coding မဟုတ်တာတွေ

ဒီအပိုင်းက အရေးကြီးပါတယ်။ အချို့အရာတွေကို Agentic Coding လို့ထင်တတ်ပေမယ့် မဟုတ်ပါဘူး။

### 4.1 Copy-paste coding မဟုတ်ဘူး

**
ChatGPT က code ပေး
ကိုယ် copy-paste
error တက်ရင် ထပ်မေး
*

ဒါက AI-assisted coding ဖြစ်နိုင်ပေမယ့် Agentic Coding အပြည့်မဟုတ်သေးဘူး။

### 4.2 Magic မဟုတ်ဘူး

AI Agent က အရာရာသိပြီး အမြဲမှန်မယ်မဟုတ်ဘူး။ Bug ထည့်နိုင်တယ်။ Wrong package install လုပ်ချင်နိုင်တယ်။ Requirement ကိုမှားနားလည်နိုင်တယ်။

### 4.3 Human ကိုအစားထိုးတာမဟုတ်ဘူး

Agentic Coding မှာ human မလိုတော့တာမဟုတ်ပါဘူး။ Human role ပြောင်းသွားတာပါ။

အရင်က—

**
Human = code ရေးသူ
*

Agentic Coding မှာ—

**
Human = goal သတ်မှတ်သူ + reviewer + tester + decision maker
Agent = implementation assistant
*

### 4.4 “မသိလည်းရတယ်” မဟုတ်ဘူး

## 8.1 Explain Agentic Coding Prompt

**
Agentic Coding ဆိုတာကို coding လုံးဝမသိသေးတဲ့သူနားလည်အောင်ရှင်းပြပါ။

## 8.2 Safe First Agent Prompt

**
ဒီ project ကိုမပြင်သေးပါနဲ့။
ပထမဆုံး project structure ကိုဖတ်ပြီး beginner နားလည်အောင်ရှင်းပြပါ။

ပြန်ပေးရမယ့်အရာ:
1. Project က ဘာလုပ်တာလဲ
2. Main folders တွေက ဘာအတွက်လဲ
3. Run command ဘာဖြစ်နိုင်လဲ
4. Beginner အနေနဲ့ ပထမဆုံးဖတ်သင့်တဲ့ file ၃ ခု
5. မသေချာတဲ့ assumption တွေ

## 8.3 Plan Before Code Prompt

**
Code မပြင်ပါနဲ့။
ဒီ feature ကို implement လုပ်ဖို့ plan ပေးပါ။

Feature:
[feature name]

ပြန်ပေးရမယ့်အရာ:
- ဘယ် files တွေဖတ်မလဲ
- ဘယ် files တွေပြင်မလဲ
- ဘာ risks ရှိလဲ
- ဘာ tests လိုမလဲ
- ဘာ command run ပြီး verify လုပ်မလဲ
*

## 8.4 Done မဖြစ်သေးတာကိုစစ်ခိုင်းတဲ့ Prompt

**
ဒီ feature ပြီးပြီလို့ယူဆထားပါတယ်။
ဒါပေမယ့် code မပြင်ခင် verification review လုပ်ပါ။

စစ်ရန်:
- Acceptance criteria အားလုံးကိုက်လား
- npm run build pass ဖြစ်လား
- Test ရှိလား / မရှိရင် ဘာ test လိုလဲ
- UI edge cases ရှိလား
- Security/privacy risk ရှိလား

Pass/Fail table နဲ့ပြန်ပေးပါ။
*

## 8.5 Beginner Safety Prompt

**
ကျွန်တော် beginner ပါ။
Command run မလုပ်ခင် command က ဘာလုပ်တာလဲ မြန်မာလိုရှင်းပြပါ။
File delete, package install, .env edit, deployment action, git reset လုပ်မယ်ဆိုရင် approval အရင်တောင်းပါ။
