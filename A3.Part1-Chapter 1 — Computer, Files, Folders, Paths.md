```text
project root ထဲမှာ SPEC.md ထည့်ပါ
```

လို instruction တွေကို နားလည်နိုင်ရမယ်။

## 2. Beginner Explanation

Computer ထဲက အရာတွေကို အလွယ်ဆုံးနားလည်မယ်ဆိုရင်—

```text
Folder = စာရွက်စာတမ်းထည့်တဲ့ ဖိုင်တွဲ
File   = စာရွက်တစ်ရွက် / ပုံတစ်ပုံ / စာတမ်းတစ်ခု / code တစ်ခု
Path   = အဲဒီ file/folder ရှိတဲ့ လိပ်စာ
```

ဥပမာ သင့် computer ထဲမှာ ဒီလိုရှိနိုင်တယ်။

```text
Desktop
  └── AgenticCourse
      └── notes
          └── chapter-1.txt
```

ဒီနေရာမှာ—

```text
AgenticCourse = folder
notes         = folder
chapter-1.txt = file
Desktop/AgenticCourse/notes/chapter-1.txt = path
```

Path ဆိုတာ file/folder ရဲ့ လိပ်စာပါ။

လူတစ်ယောက်ရဲ့အိမ်လိပ်စာလိုပဲ—

```text
အိမ်နံပါတ် → လမ်း → မြို့ → နိုင်ငံ
```

Computer file path ကလည်း—

```text
Drive / Home → Folder → Subfolder → File
```

ဆိုပြီး အဆင့်လိုက်သွားပါတယ်။

## 3. Key Concepts

## 3.1 File ဆိုတာဘာလဲ

File ဆိုတာ computer ထဲမှာ သိမ်းထားတဲ့ data တစ်ခုပါ။

File အမျိုးအစားအများကြီးရှိတယ်။

```text
စာရွက်စာတမ်း file
ပုံ file
အသံ file
video file
spreadsheet file
presentation file
code file
configuration file
```

ဥပမာ—

```text
resume.pdf
photo.jpg
budget.xlsx
presentation.pptx
notes.txt
index.html
App.jsx
package.json
SPEC.md
```

Coding project တွေမှာ file တစ်ခုချင်းစီက တာဝန်တစ်ခုရှိတယ်။

```text
App.jsx       → React app ရဲ့ UI code
package.json  → project commands/dependencies
README.md     → project explanation
SPEC.md       → product requirements
.env          → secret/config values
```

---

## 3.2 Folder ဆိုတာဘာလဲ

Folder ဆိုတာ file တွေကို စုထားတဲ့ container ပါ။

ဥပမာ—

```text
photos/
documents/
downloads/
projects/
```

Coding project ထဲမှာ folder တွေကို purpose အလိုက်ခွဲထားတတ်တယ်။

ဥပမာ React project တစ်ခုမှာ—

```text
src/
  components/
  pages/
  data/
  utils/
```

အဓိပ္ပါယ်က—

```text
src/          → source code တွေထားတဲ့ folder
components/   → reusable UI pieces
pages/        → page-level components
data/         → data files
utils/        → helper functions
```

Folder structure ကောင်းရင် Agent က project ကိုနားလည်ရလွယ်တယ်။ Folder structure ရှုပ်နေရင် Agent လည်းလမ်းလွဲနိုင်တယ်။

## 3.3 Path ဆိုတာဘာလဲ

Path ဆိုတာ file/folder ရဲ့ လိပ်စာပါ။

ဥပမာ—

```text
agentic-zero-to-hero/notes/chapter-1.md
```

ဒီ path ကို ဖတ်မယ်ဆိုရင်—

```text
agentic-zero-to-hero folder ထဲဝင်
notes folder ထဲဝင်
chapter-1.md file ကိုရှာ
```

ဆိုတဲ့ အဓိပ္ပါယ်ပါ။

Agent ကိုခိုင်းတဲ့အခါ path သိရင် အလွန်အရေးကြီးပါတယ်။

မရှင်းတဲ့ prompt—

```text
အဲဒီ file ကိုပြင်ပေး။
```

ရှင်းတဲ့ prompt—

```text
notes/chapter-1.md file ကိုဖတ်ပြီး spelling mistakes တွေပြင်ပေးပါ။
```

ပိုကောင်းတဲ့ prompt—

```text
notes/chapter-1.md file ကိုပဲပြင်ပါ။
တခြား file မပြင်ပါနဲ့။
ပြင်ပြီးရင် ဘာပြင်ခဲ့လဲ summary ပေးပါ။
```

## 3.4 Root Folder ဆိုတာဘာလဲ

Root folder ဆိုတာ project ရဲ့ အဓိက folder ပါ။

ဥပမာ—

```text

  apps/
  services/
  packages/
```

ဒီမှာ `event-management/` က root folder ပါ။

Root folder ကို နားလည်ဖို့အရေးကြီးပါတယ်။ Agentic Coding မှာ အလုပ်အများစုကို root folder ထဲကနေစတင်မယ်။

ဥပမာ—

```text
cd event-management
claude
```

ဆိုရင် Agent က `event-management` project ထဲမှာအလုပ်လုပ်မယ်။

Root folder ထဲမှာ များသောအားဖြင့် ဒီ files တွေရှိတတ်တယ်။

```text
README.md
package.json
SPEC.md
CLAUDE.md
AGENTS.md
.gitignore
.env.example
```

## 3.5 Current Folder ဆိုတာဘာလဲ

Current folder ဆိုတာ သင်လက်ရှိရောက်နေတဲ့ folder ပါ။

Terminal သုံးတဲ့အခါ အရေးကြီးလာမယ်။

ဥပမာ—

```text
Desktop/
  agentic-zero-to-hero/
    notes/
      chapter-1.md
```

သင်က `Desktop` ထဲမှာရှိနေတယ်ဆိုရင် `chapter-1.md` ကိုတိုက်ရိုက်မမြင်နိုင်ဘူး။ `agentic-zero-to-hero/notes/` ထဲဝင်ရမယ်။

Terminal မှာ နောက်အခန်းမှာသင်မယ့် command—

```bash
pwd
```

က current folder path ကိုပြပေးမယ်။

## 3.6 Absolute Path နဲ့ Relative Path

Path အမျိုးအစား ၂ မျိုးရှိတယ်။

```text
Absolute path
Relative path
```

### Absolute Path

Absolute path ဆိုတာ computer ထဲက အပြည့်အစုံလိပ်စာပါ။

Windows ဥပမာ—

```text
C:\Users\Aung\Desktop\agentic-zero-to-hero\notes\chapter-1.md
```

macOS / Linux ဥပမာ—

```text
/Users/aung/Desktop/agentic-zero-to-hero/notes/chapter-1.md
```

### Relative Path

Relative path ဆိုတာ လက်ရှိရောက်နေတဲ့ folder ကနေတွက်တဲ့ လိပ်စာပါ။

သင်က `agentic-zero-to-hero/` folder ထဲမှာရှိနေတယ်ဆိုရင်—

```text
notes/chapter-1.md
```

လို့ရေးရုံနဲ့ရတယ်။

Agentic Coding မှာ relative path ကိုပိုအသုံးများပါတယ်။

ဥပမာ—

```text
src/components/EventCard.jsx ကိုပြင်ပါ။
```

ဒီလိုပြောရင် project root ကနေတွက်တဲ့ path ဖြစ်တတ်တယ်။

## 3.7 File Extension ဆိုတာဘာလဲ

File extension ဆိုတာ file name နောက်ဆုံးမှာပါတဲ့ `.something` ပါ။

ဥပမာ—

```text
notes.txt
README.md
App.jsx
style.css
data.json
photo.png
```

Extension က file အမျိုးအစားကိုပြတယ်။

| Extension      | အဓိပ္ပါယ်                       |
| -------------- | ------------------------------- |
| `.txt`         | plain text                      |
| `.md`          | Markdown document               |
| `.html`        | webpage structure               |
| `.css`         | styling                         |
| `.js`          | JavaScript                      |
| `.jsx`         | React JavaScript component      |
| `.ts`          | TypeScript                      |
| `.tsx`         | React TypeScript component      |
| `.json`        | structured data/config          |
| `.env`         | environment variables / secrets |
| `.png`, `.jpg` | image files                     |
| `.pdf`         | PDF document                    |

Beginner အတွက် သိထားသင့်တာ—

```text
README.md     → project explanation
SPEC.md       → requirements
package.json  → project config/scripts/dependencies
App.jsx       → React component
index.html    → web page entry shell
.env          → secret/config file, သတိထားရန်
```

## 3.8 Hidden Files

Hidden file ဆိုတာ computer က default အနေနဲ့ မပြတဲ့ file ပါ။ များသောအားဖြင့် `.` နဲ့စတယ်။

ဥပမာ—

```text
.gitignore
.env
.claude/
.git/
```

Coding project တွေမှာ hidden files အရေးကြီးပါတယ်။

```text
.gitignore   → Git ထဲမထည့်ချင်တဲ့ files ကိုသတ်မှတ်
.env         → secret/config values
.git/        → Git history
.claude/     → Claude Code settings, skills, agents, hooks
```

အရေးကြီးဆုံးသတိ—

```text
.env file ထဲမှာ API key, password, token တွေရှိနိုင်တယ်။
Agent chat ထဲကို .env content မ paste ပါနဲ့။
Git ထဲ commit မလုပ်ပါနဲ့။
```

## 3.9 File Tree ဆိုတာဘာလဲ

File tree ဆိုတာ folder/file structure ကို သစ်ပင်ပုံစံပြတာပါ။

ဥပမာ—

```text
agentic-zero-to-hero/
├── README.md
├── COURSE_DECISIONS.md
├── notes/
│   ├── chapter-0-1.md
│   ├── chapter-0-2.md
│   └── chapter-0-3.md
└── projects/
    ├── myaree-store/
    └── event-management/
```

ဒီ tree ကိုဖတ်နည်း—

```text
agentic-zero-to-hero/ က root folder
README.md နဲ့ COURSE_DECISIONS.md က root ထဲမှာရှိ
notes/ folder ထဲမှာ chapter notes တွေရှိ
projects/ folder ထဲမှာ project folders တွေရှိ
```

Agent ကို project overview မေးတဲ့အခါ file tree အနေနဲ့ပြခိုင်းတာ အသုံးဝင်တယ်။

Prompt—

```text
ဒီ project ရဲ့ file tree ကို beginner နားလည်အောင်ရှင်းပြပါ။
ဖိုင်တစ်ခုချင်းစီရဲ့တာဝန်ကို တိုတိုပြောပါ။
Code မပြင်ပါနဲ့။
```

## 3.10 Naming Rules

File/folder naming က အရေးကြီးပါတယ်။ နောက်ပိုင်း terminal, Git, deployment, automation tools တွေသုံးတဲ့အခါ space, မြန်မာစာ, special characters တွေက problem ဖြစ်နိုင်တယ်။

Beginner-friendly naming rule—

```text
Use lowercase English
Use hyphen between words
Avoid spaces
Avoid special symbols
Use clear names
```

ကောင်းတဲ့နာမည်—

```text
agentic-zero-to-hero
event-management
myaree-store
chapter-1.md
event-list.md
registration-form.md
```

ရှောင်သင့်တဲ့နာမည်—

```text
My Project Final FINAL
chapter 1 new latest copy.md
ပရောဂျက်အသစ်!!!
event@project#2026
```

မြန်မာစာ folder/file name သုံးလို့ရတဲ့ system တွေရှိပေမယ့် beginner course အတွက် project file/folder name တွေကို English lowercase နဲ့သုံးတာ ပိုလုံခြုံပါတယ်။ UI text, content, documentation ထဲမှာ မြန်မာလိုရေးလို့ရပါတယ်။

# 4. Agentic Coding နဲ့ File/Folder/Path ဘာကြောင့်ဆက်စပ်လဲ

AI Agent ကိုအလုပ်ခိုင်းတဲ့အခါ Agent က file တွေကိုဖတ်ပြီးပြင်မယ်။ ဒါကြောင့် path မသိရင် instruction မရှင်းနိုင်ဘူး။

မကောင်းတဲ့ instruction—

```text
Registration page ကိုပြင်ပေး။
```

Agent အတွက်မရှင်းတာ—

```text
ဘယ် file လဲ?
Frontend လား?
Admin page လား?
PWA page လား?
Backend registration API လား?
```

ကောင်းတဲ့ instruction—

```text
apps/web/src/pages/RegisterPage.jsx file ကိုဖတ်ပါ။
Phone validation message ကိုမြန်မာလိုပြင်ပါ။
တခြား file မပြင်ပါနဲ့။
```

ပိုကောင်းတဲ့ instruction—

```text
Task:
Registration form မှာ phone validation error message ကိုမြန်မာလိုပြင်ချင်ပါတယ်။

Read:
- apps/web/src/pages/RegisterPage.jsx
- packages/shared/src/validators/registration.js

Do not edit:
- services/api/**
- apps/admin/**
- apps/checkin-pwa/**

Before editing:
- ဘာပြင်မလဲ plan ပြပါ။
```

ဒီလို path နဲ့ scope ပေးနိုင်ရင် Agent လမ်းမလွဲတော့ဘူး။

# 6. Important File Types in This Course

နောက်ပိုင်းမှာ အောက်ပါ files တွေ မကြာခဏတွေ့မယ်။

## 6.1 Markdown Files

```text
README.md
SPEC.md
COURSE_DECISIONS.md
FEATURE_BRIEF.md
TRACEABILITY.md
```

Markdown file တွေက documentation အတွက်ပါ။ `.md` နဲ့ဆုံးတယ်။

ဥပမာ—

```md
# Project Title

## Goal

This project helps event organizers manage registrations.
```

## 6.2 JSON Files

```text
package.json
data/events.json
TASK_SCHEMA.json
```

JSON file တွေက structured data/config အတွက်ပါ။

ဥပမာ—

```json
{
  "name": "event-management",
  "version": "1.0.0"
}
```

JSON မှာ comma, quote, bracket မှားရင် error တက်နိုင်တယ်။

## 6.3 JavaScript / React Files

