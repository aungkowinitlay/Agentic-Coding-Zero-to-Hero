## 2. Beginner Explanation

Level 0 ဆိုတာ ChatGPT ကို code example တောင်းပြီး ကိုယ်တိုင် copy-paste လုပ်တဲ့အဆင့်ပါ။

ဥပမာ—

```text
“HTML registration form တစ်ခုရေးပေးပါ”
```

လို့မေးတယ်။ ChatGPT က code block ပေးတယ်။ သင်က code ကို copy လုပ်ပြီး file ထဲ paste လုပ်တယ်။ Browser မှာဖွင့်ကြည့်တယ်။ Error ရှိရင် error message ကိုပြန် paste လုပ်ပြီးမေးတယ်။

ဒီ flow ကို diagram နဲ့ပြရင်—

```text
User
 ↓ prompt
ChatGPT
 ↓ code example
User copy-paste
 ↓
Local file / browser
 ↓
User tests manually
```

ဒီအဆင့်မှာ ChatGPT က **ဆရာ/အကြံပေး/ဥပမာရေးပေးသူ** ပါ။ ဒါပေမယ့် project folder ထဲဝင်ပြီး file ကို ကိုယ်တိုင်ဖတ်၊ ပြင်၊ command run, test run, Git diff review လုပ်ပေးတဲ့ coding agent မဟုတ်သေးပါဘူး။

OpenAI ရဲ့ prompt engineering guide မှာလည်း model ကို ပိုအသုံးဝင်တဲ့ output ရစေဖို့ instruction ကိုရှင်းရှင်းရေးခြင်း၊ context/outcome/format/style ကိုတိတိကျကျဖော်ပြခြင်း၊ desired output format ကိုပြခြင်းတို့ကို အကြံပြုထားပါတယ်။ ဒါကြောင့် Level 0 မှာ အရေးကြီးဆုံး skill က “code မေးတာ” မဟုတ်ဘဲ **code example ကို တိတိကျကျမေးတာ** ဖြစ်ပါတယ်။ ([OpenAI Help Center][1])

# 3. Level 0 ဆိုတာဘာလဲ

Level 0 မှာ AI က project ကို တိုက်ရိုက်မထိဘူး။ User က AI ကို code sample, explanation, pseudo-code, debugging advice မေးတယ်။

Level 0 examples—

```text
- HTML form example ရေးပေးပါ
- CSS card layout example ပေးပါ
- JavaScript validation function ရေးပေးပါ
- ဒီ error ဘာကြောင့်ဖြစ်တာလဲရှင်းပြပါ
- ဒီ code ကို beginner နားလည်အောင်ရှင်းပြပါ
- API request example ပေးပါ
- SQL table design example ပေးပါ
```

Level 0 မှာ AI က—

```text
[✓] code sample ပေးနိုင်တယ်
[✓] concept ရှင်းနိုင်တယ်
[✓] error message ရှင်းနိုင်တယ်
[✓] alternatives နှိုင်းယှဉ်နိုင်တယ်
[✓] prompt/template ရေးပေးနိုင်တယ်
[✓] beginner-friendly explanation ပေးနိုင်တယ်
```

ဒါပေမယ့် Level 0 မှာ AI က—

```text
[✗] project folder ထဲက file တွေကို အလိုအလျောက်မဖတ်နိုင်
[✗] local terminal command မ run နိုင်
[✗] browser မှာ result မစမ်းနိုင်
[✗] Git diff ကိုမမြင်နိုင်
[✗] npm run build/test ကိုမ run နိုင်
[✗] file structure တစ်ခုလုံးကို မသိနိုင်, မပေးထားရင်
[✗] code ကို project context နဲ့အမြဲကိုက်အောင်မရေးနိုင်
```

ဒီအချက်ကြောင့် Level 0 ကို သင်ယူရန်၊ prototype စမ်းရန်, snippet ရရန် အသုံးဝင်တယ်။ ဒါပေမယ့် production project ဆောက်ဖို့ Level 0 တစ်ခုတည်းမလုံလောက်ဘူး။

# 4. Level 0 နဲ့ Level 3/4 Coding Agent ကွာခြားချက်

OpenAI Help Center မှာ Codex ကို “code ရေး၊ review လုပ်၊ ship လုပ်ဖို့ကူညီတဲ့ AI agent” အဖြစ်ဖော်ပြထားပါတယ်။ Level 0 ChatGPT code example က အဲဒီလို project/workflow ကိုတာဝန်ယူလုပ်တဲ့ coding agent မဟုတ်ဘဲ code snippet/explanation ပေးတဲ့ conversation workflow ဖြစ်ပါတယ်။ ([OpenAI Help Center][2])

| အချက်                  | Level 0: ChatGPT Code Example | Level 3/4: CLI/Coding Agent      |
| ---------------------- | ----------------------------- | -------------------------------- |
| Project files ဖတ်ခြင်း | User paste လုပ်ပေးမှ          | Agent က repo ထဲဖတ်နိုင်          |
| File edit              | User copy-paste               | Agent က file edit လုပ်နိုင်      |
| Terminal command       | User ကိုယ်တိုင် run           | Agent က run ခိုင်း/လုပ်နိုင်     |
| Build/test             | User ကိုယ်တိုင်စစ်            | Agent က run + fix loop လုပ်နိုင် |
| Context                | Prompt ထဲပေးသလောက်            | Codebase context ပါဖတ်နိုင်      |
| Git diff               | User paste လုပ်မှ             | Agent က diff review လုပ်နိုင်    |
| Use case               | Learning/snippets             | Real project feature work        |

Level 0 ကို “နည်းပါးတဲ့အဆင့်” လို့မထင်ပါနဲ့။ Coding မသိသေးသူအတွက် Level 0 က အလွန်အသုံးဝင်ပါတယ်။ ဒါပေမယ့် Level 0 ရဲ့နေရာကို နားလည်ဖို့လိုတယ်။

```text
Level 0 = Learn, ask, understand, generate small examples
Level 3/4 = Work inside a real project with tools, tests, files, Git
```

# 6. Level 0 ကို ဘယ်အချိန်မသုံးသင့်လဲ

## 6.1 Project ကြီးတစ်ခုလုံးကို တစ်ခါတည်းဆောက်ခိုင်းခြင်း

မကောင်းတဲ့ prompt—

```text

- Build/test မစစ်နိုင်
```

Level 0 မှာ ဒီလိုမလုပ်သင့်ပါ။

# 7. Good Level 0 Prompt ရဲ့ Anatomy

Level 0 prompt ကောင်းတစ်ခုမှာ အောက်ပါအပိုင်းတွေပါသင့်တယ်။

```text
1. Role / Level
2. Context
3. Goal
4. Constraints
5. Output format
6. Explanation level
7. Verification
```

ဥပမာ—

```text
သင်က beginner-friendly web development tutor ဖြစ်တယ်။

Context:
ကျွန်တော် coding မသိသေးပါ။
Event Management Platform course လေ့လာနေပါတယ်။

Goal:
HTML, CSS, JavaScript သုံးပြီး event registration form example တစ်ခုလိုပါတယ်။

Constraints:
- React မသုံးပါနဲ့
- Backend API မချိတ်ပါနဲ့
- File ၃ ခုခွဲပါ: index.html, style.css, app.js
- မြန်မာ UI text သုံးပါ
- label/input accessibility ပါစေ

Output format:
1. index.html
2. style.css
3. app.js
4. Code explanation
5. How to test in browser

Explanation:
Beginner နားလည်အောင်ရှင်းပါ။
```

ဒီ prompt က “HTML form ရေးပေး” ထက် အများကြီးပိုကောင်းတယ်။

# 8. Bad Prompt vs Good Prompt

## Bad Prompt 1

```text
Website code ရေးပေး။
```

ပြဿနာ—

```text
- ဘာ website လဲမသိ
- ဘာ language/stack လဲမသိ
- ဘယ် file format လဲမသိ
- ဘယ်အထိလုပ်ရမလဲမသိ
```

## Good Prompt 1

```text
HTML, CSS, JavaScript သုံးပြီး simple event listing page example တစ်ခုရေးပေးပါ။

Requirements:
- Event card ၃ ခုပါရမယ်
- Title, date, location ပါရမယ်
- Mobile-first layout ဖြစ်ရမယ်
- File ၃ ခုခွဲပါ: index.html, style.css, app.js
- JavaScript behavior မလိုသေးပါ
- Beginner နားလည်အောင် code explanation ပေးပါ
```

---

## Bad Prompt 2

```text
Code fix ပေး။
```

## Good Prompt 2

```text
အောက်ပါ JavaScript code မှာ error ဖြစ်နေပါတယ်။

Error:
Cannot read properties of null (reading 'addEventListener')

Expected:
Register button နှိပ်ရင် success message ပြရမယ်။

Actual:
Button နှိပ်လို့မရဘဲ console error တက်တယ်။

Code:
[paste HTML + JS]

လုပ်ရန်:
- Code မပြင်ခင် root cause ရှင်းပြပါ
- ဖြစ်နိုင်တဲ့အကြောင်းရင်း ၃ ခုရေးပါ
- ပြင်ထားတဲ့ minimal code ပေးပါ
- Beginner နားလည်အောင်ရှင်းပါ
```

# 9. Level 0 Workflow

Level 0 ကိုအသုံးပြုတဲ့ workflow ကို ဒီလိုမှတ်ပါ။

```text
1. Small task ရွေး
2. Prompt တိတိကျကျရေး
3. Code example ရယူ
4. Code ကိုဖတ်
5. File ထဲ copy-paste
6. Browser/terminal မှာစမ်း
7. Error ရှိရင် error message နဲ့ပြန်မေး
8. Working version ကို Git commit
```

သတိထားရန်—

```text
Level 0 မှာ AI က project ထဲဝင်မပြင်ဘူး။
User က copy-paste တာဝန်ယူရမယ်။
```

# 10. Code Example ကို Copy-Paste မလုပ်ခင် Checklist

ChatGPT code block တစ်ခုရလာရင် အောက်ပါ checklist စစ်ပါ။

```text
[ ] ဒီ code က ဘယ် file အတွက်လဲ
[ ] File name ပါလား
[ ] HTML/CSS/JS ကိုရောထားလား ခွဲထားလား
[ ] Dependencies အသစ်လိုလား
[ ] Package install command ပါလား
[ ] Secret/API key placeholder ပါလား
[ ] Browser မှာ run လို့ရမလား
[ ] Existing project file ကို overwrite လုပ်မှာလား
[ ] Backup/Git commit ရှိလား
```

အထူးသတိ—

```text
Code block တစ်ခုလုံးကို existing file ပေါ် overwrite မလုပ်ခင် backup/Git commit ရှိပါစေ။
```

## app.js

```js
const form = document.querySelector("#registration-form");
const nameInput = document.querySelector("#name");
const phoneInput = document.querySelector("#phone");
const message = document.querySelector("#message");

form.addEventListener("submit", function (event) {
  event.preventDefault();

  const name = nameInput.value.trim();
  const phone = phoneInput.value.trim();

  message.className = "message";

  if (name === "") {
    message.textContent = "နာမည်ဖြည့်ပါ။";
    message.classList.add("error");
    return;
  }

  if (phone === "") {
    message.textContent = "ဖုန်းနံပါတ်ဖြည့်ပါ။";
    message.classList.add("error");
    return;
  }

  message.textContent = `${name} အတွက် registration အောင်မြင်ပါသည်။`;
  message.classList.add("success");
});
```
