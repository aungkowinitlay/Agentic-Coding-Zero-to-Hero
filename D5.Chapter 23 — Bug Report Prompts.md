# Chapter 23 — Bug Report Prompts

## 1. Learning Objectives

ဒီအခန်းပြီးသွားရင် သင်က အောက်ပါအရာတွေကို နားလည်ပြီး လက်တွေ့ရေးနိုင်ရပါမယ်။

```text
[ ] Bug report ဆိုတာဘာလဲ ရှင်းပြနိုင်မယ်
[ ] “မရဘူး” ဆိုတဲ့ vague report ကို structured bug report အဖြစ်ပြောင်းနိုင်မယ်
[ ] Expected / Actual / Steps to Reproduce ကိုရေးနိုင်မယ်
[ ] Error message, console log, terminal output, network status code တို့ကို bug report ထဲထည့်နိုင်မယ်
[ ] Environment information ကိုရေးနိုင်မယ်
[ ] Recent changes / git diff ကို bug diagnosis အတွက်သုံးနိုင်မယ်
[ ] Frontend bug, backend bug, API bug, database bug, build bug, deployment bug prompt တွေကိုခွဲနိုင်မယ်
[ ] Agent ကို code မပြင်ခင် root cause candidates ပေးခိုင်းနိုင်မယ်
[ ] Minimal fix plan နဲ့ approval gate ပါတဲ့ debug prompt ရေးနိုင်မယ်
[ ] Bug finding ကို actionable fix task အဖြစ်ပြောင်းနိုင်မယ်
[ ] Bug ပြင်ပြီး regression test / verification prompt ရေးနိုင်မယ်
```

ဒီအခန်းက Prompting & Agent Control အပိုင်းမှာ အလွန်အရေးကြီးပါတယ်။ Agentic Coding မှာ bug ဖြစ်တာကိုရှောင်လို့မရပါဘူး။ အရေးကြီးတာက bug ဖြစ်တဲ့အခါ Agent ကို **ရှာလို့ရအောင်၊ ပြင်လို့ရအောင်၊ ပြင်ပြီးပြန်စစ်လို့ရအောင်** report ပေးတတ်ဖို့ပါ။

---

## 2. Beginner Explanation

Beginner တွေ bug ဖြစ်တဲ့အခါ မကြာခဏ ဒီလိုပြောတတ်တယ်။

```text
မရဘူး။
Error ဖြစ်နေတယ်။
Button မလုပ်ဘူး။
Page ပျက်နေတယ်။
API မရဘူး။
```

ဒီလို report က Agent အတွက်လည်း၊ human developer အတွက်လည်း မလုံလောက်ပါဘူး။

Agent မသိနိုင်တာတွေ—

```text
ဘာလုပ်ချင်တာလဲ?
ဘာဖြစ်ရမလဲ?
တကယ်ဘာဖြစ်နေလဲ?
ဘယ်အဆင့်မှာဖြစ်တာလဲ?
Error message ဘာလဲ?
Browser console မှာဘာပြလဲ?
Terminal output ဘာလဲ?
Network tab မှာ status code ဘာလဲ?
နောက်ဆုံးဘာတွေပြင်ထားလဲ?
ဘယ် file တွေထိထားလဲ?
```

Good bug report ဆိုတာ—

```text
Bug ကို reproduce လုပ်နိုင်အောင် အချက်အလက်ပြည့်စုံစွာရေးထားတဲ့ report
```

ပါ။

မကောင်းတဲ့ bug report—

```text
Register button မရဘူး။
```

ကောင်းတဲ့ bug report—

```text
Bug:
Register button နှိပ်ပေမယ့် success message မပြပါ။

Steps to Reproduce:
1. playground/chapter-11-coding-agent/index.html ကို browser မှာဖွင့်ပါ။
2. Name field ထဲ “Aung Aung” ရေးပါ။
3. Phone field ထဲ “09123456789” ရေးပါ။
4. Register button နှိပ်ပါ။

Expected:
“Registration အောင်မြင်ပါသည်။” message ပြရမယ်။

Actual:
Message မပြပါ။

Console Error:
Cannot read properties of null (reading 'addEventListener')

Recent Changes:
app.js ထဲမှာ form selector ပြင်ထားပါတယ်။

Task:
Code မပြင်သေးပါနဲ့။
Root cause candidates ၃ ခုနဲ့ minimal fix plan ပေးပါ။
```

ဒီလိုရေးထားရင် Agent က bug ကို diagnosis လုပ်နိုင်တယ်။

# 3. Bug Report Prompt ဆိုတာဘာလဲ

Bug Report Prompt ဆိုတာ Agent ကို bug ရှာခိုင်းတဲ့ prompt ပါ။ ဒါပေမယ့် bug report prompt က code ပြင်ခိုင်းတဲ့ prompt မဟုတ်သေးပါဘူး။

အရေးကြီးဆုံး rule—

```text
Debug first. Code မပြင်သေးပါနဲ့။
```

Bug report prompt ရဲ့ရည်ရွယ်ချက်—

```text
1. Bug ကိုနားလည်စေ
2. Reproduce လုပ်နည်းပေး
3. Expected / Actual ခွဲပြ
4. Error evidence ပေး
5. Root cause candidates ရှာခိုင်း
6. Minimal fix plan တောင်း
7. Approval ရမှ code ပြင်စေ
```

Bad debug prompt—

```text
Error fix လုပ်။
```

Good debug prompt—

```text
Mode:
Debug first. Code မပြင်သေးပါနဲ့။

Bug:
Registration form submit လုပ်တဲ့အခါ page refresh ဖြစ်ပြီး success message မပြပါ။

Expected:
Form submit ပြီး success message ပြရမယ်။ Page refresh မဖြစ်ရဘူး။

Actual:
Button နှိပ်ပြီး page refresh ဖြစ်သွားပါတယ်။

Steps:
1. index.html ကိုဖွင့်
2. Name/phone ဖြည့်
3. Register နှိပ်

Likely area:
app.js form submit event handler.

Task:
Root cause candidates ပေးပါ။
Minimal fix plan ပေးပါ။
Approval မရသေးဘဲ code မပြင်ပါနဲ့။
```

# 4. Bug Report Anatomy

Bug report ကောင်းတစ်ခုမှာ အောက်ပါအပိုင်းတွေပါနိုင်တယ်။

```text
1. Title
2. Summary
3. Context
4. Environment
5. Steps to Reproduce
6. Expected Result
7. Actual Result
8. Error Message / Logs
9. Screenshots or Observations
10. Recent Changes
11. Suspected Area
12. What I Already Tried
13. Scope / Allowed Files
14. Do-not-touch List
15. Debug Instructions
16. Output Format
17. Approval Gate
```

အားလုံးကို အမြဲထည့်စရာမလိုပါ။ Bug ရှုပ်လေလေ အချက်အလက်ပိုလိုလေလေဖြစ်တယ်။

---

## 4.1 Title

Bug ကိုတစ်ကြောင်းနဲ့ပြောပါ။

မကောင်း—

```text
Error
```

ကောင်း—

```text
Register button click does not show success message
```

မြန်မာလို—

```text
Register button နှိပ်သော်လည်း success message မပြသော bug
```

---

## 4.2 Summary

Bug ကိုတိုတိုရေးပါ။

```text
Registration form submit လုပ်တဲ့အခါ expected success message မပြဘဲ browser console error တက်နေပါတယ်။
```

---

## 4.3 Context

ဘယ် project / feature / file / chapter ထဲက bug လဲပြောပါ။

```text
Context:
ဒီ bug က playground/chapter-11-coding-agent static HTML/CSS/JS lab app ထဲမှာဖြစ်နေပါတယ်။
Backend API မရှိပါ။
```

Context မပါရင် Agent က project type မှားနားလည်နိုင်တယ်။

---

## 4.4 Environment

Bug ဖြစ်တဲ့ environment ကိုရေးပါ။

```text
Environment:
- OS: Windows 11 / macOS / Linux
- Browser: Chrome / Edge / Firefox
- App type: static HTML/CSS/JS
- Run method: opened index.html directly
- Node version: node -v output, if relevant
- npm command: npm run dev / npm run build, if relevant
```

Frontend static file bug အတွက် Node/npm မလိုနိုင်ပါ။ Build bug အတွက် Node/npm version လိုနိုင်တယ်။

---

## 4.5 Steps to Reproduce

Bug ကိုတစ်ဆင့်ချင်းပြန်ဖြစ်အောင်ရေးပါ။

မကောင်း—

```text
Register နှိပ်ရင်မရဘူး။
```

ကောင်း—

```text
Steps to Reproduce:
1. Open playground/chapter-11-coding-agent/index.html in Chrome.
2. Enter “Aung Aung” in Name.
3. Enter “09123456789” in Phone.
4. Click “Register လုပ်မည်”.
```

Steps က တခြားသူ/Agent လိုက်လုပ်လို့ရရမယ်။

---

## 4.6 Expected Result

ဘာဖြစ်ရမလဲရေးပါ။

```text
Expected:
“Registration အောင်မြင်ပါသည်။” message ပြရမယ်။
Form fields clear ဖြစ်ရမယ်။
Browser console error မရှိရဘူး။
```

---

## 4.7 Actual Result

တကယ်ဘာဖြစ်နေလဲရေးပါ။

```text
Actual:
Message မပြပါ။
Console မှာ “Cannot read properties of null” error တက်ပါတယ်။
```

Expected နဲ့ Actual က bug report ရဲ့နှလုံးသားပါ။

---

## 4.8 Error Message / Logs

Error message ကိုတိတိကျကျထည့်ပါ။

Frontend console error example—

```text
Console Error:
Uncaught TypeError: Cannot read properties of null (reading 'addEventListener')
    at app.js:12:6
```

Terminal error example—

```text
Terminal Error:
npm ERR! Missing script: "dev"
```

API error example—

```text
Network:
POST /api/events/1/register
Status: 422
Response:
{
  "error": "Phone is required"
}
```

သတိ—

```text
Secret values ပါတဲ့ logs မ paste ပါနဲ့။
.env content မ paste ပါနဲ့။
Database URL / API key / token ကို redact လုပ်ပါ။
```

---

## 4.9 Screenshot / Observation

Screenshot မပေးနိုင်ရင် observation ရေးပါ။

```text
Observation:
Message area is empty.
Button click seems to refresh the page.
URL changes from index.html to index.html?name=Aung&phone=...
```

ဒီ observation က root cause ကိုခန့်မှန်းစေနိုင်တယ်။ ဥပမာ page refresh ဖြစ်တာက `event.preventDefault()` မပါလို့ဖြစ်နိုင်တယ်။

---

## 4.10 Recent Changes

Bug မဖြစ်ခင်နောက်ဆုံးဘာပြင်ခဲ့လဲရေးပါ။

```text
Recent Changes:
- app.js ထဲ validation function ကိုပြင်ခဲ့တယ်။
- HTML form id ကို registrationForm ကနေ registration-form ပြောင်းခဲ့တယ်။
- style.css မပြင်ခဲ့ပါ။
```

Recent changes က bug root cause ရှာရာမှာအလွန်အသုံးဝင်တယ်။

Git ရှိရင်—

```bash
git diff
```

ကို review လုပ်ခိုင်းနိုင်တယ်။

---

## 4.11 Suspected Area

သင်ထင်တဲ့နေရာရှိရင်ရေးပါ။ ဒါပေမယ့် Agent ကိုအတင်းမယုံခိုင်းပါနဲ့။

```text
Suspected Area:
app.js form selector or submit event listener might be wrong.
```

Agent ကိုပြောရန်—

```text
ဒီ suspected area ကိုစစ်ပါ။ ဒါပေမယ့် တခြား root cause candidates ကိုလည်းစဉ်းစားပါ။
```

---

## 4.12 What I Already Tried

ကိုယ်စမ်းထားတာတွေပါရင်ရေးပါ။

```text
Tried:
- Browser refresh
- Checked index.html script tag
- Confirmed app.js file exists
- Console error still appears
```

ဒါက Agent ကို duplicate advice မပေးအောင်ကူညီတယ်။

---

## 4.13 Scope / Allowed Files

Bug fix မှာ Agent ကို scope ထိန်းပါ။

```text
Allowed files:
- playground/chapter-11-coding-agent/index.html
- playground/chapter-11-coding-agent/app.js
```

Do-not-touch—

```text
Do not touch:
- package.json
- .env
- node_modules/**
- backend files
```

---

## 4.14 Debug Instructions

```text
Debug first. Code မပြင်သေးပါနဲ့။
Root cause candidates ပေးပါ။
Minimal fix plan ပေးပါ။
Approval ရမှ edit လုပ်ပါ။
```

---

## 4.15 Output Format

```text
Output Format:
## Bug Summary
## Reproduction Understanding
## Root Cause Candidates
## Most Likely Cause
## Minimal Fix Plan
## Verification Steps
## Approval Needed
```

Output format မရှိရင် Agent response ရှုပ်နိုင်တယ်။

# 7. Frontend UI Bug Prompt

UI ပေါ်မမှန်တာ, layout ပျက်တာ, button မပေါ်တာ စတာတွေအတွက်သုံးမယ်။

```text
Role:
သင်က frontend debugging assistant ဖြစ်တယ်။

Mode:
Debug first. Code မပြင်သေးပါနဲ့။

Bug:
Event cards desktop မှာ 3 columns ဖြစ်ရမယ့်အစား 1 column ပဲပြနေပါတယ်။

Context:
Static HTML/CSS/JS event list lab app.

Environment:
Browser: Chrome
Screen width: 1280px

Steps to Reproduce:
1. Open index.html.
2. Resize browser to desktop width.
3. Look at event cards.

Expected:
Desktop width မှာ event cards 3 columns ဖြစ်ရမယ်။

Actual:
Event cards 1 column ပဲဖြစ်နေပါတယ်။

Suspected Area:
style.css grid / media query.

Allowed Files:
- index.html
- style.css

Do Not Touch:
- app.js
- package.json
- .env

Task:
Root cause candidates နဲ့ minimal CSS fix plan ပေးပါ။
Approval မရသေးဘဲ code မပြင်ပါနဲ့။
```

စစ်ရမယ့်အချက်—

```text
CSS file linked?
class name matches?
media query min-width correct?
grid-template-columns correct?
```

# 8. JavaScript Behavior Bug Prompt

Button click မလုပ်တာ, validation မမှန်တာ, localStorage bug စတာတွေအတွက်။

```text
Role:
သင်က JavaScript debugging assistant ဖြစ်တယ်။

Mode:
Debug first. Code မပြင်သေးပါနဲ့။

Bug:
Register button နှိပ်ပေမယ့် success message မပြပါ။

Context:
Static HTML/CSS/JS registration lab app.

1. Open index.html.
2. Fill name and phone.
3. Click Register.

Success message appears.

Actual:
No message appears.

Console Error:

Root cause candidates ပေးပါ။
```

ဒီ bug မှာ Agent က စစ်သင့်တာ—

```text
form id in HTML
querySelector in JS
script tag path
script tag location
DOM loaded timing
```

# 9. API Request Bug Prompt

Frontend က backend API ခေါ်တဲ့အခါ status code error ဖြစ်တာတွေအတွက်။

```text
Role:
သင်က full-stack API debugging assistant ဖြစ်တယ်။

Mode:
Debug first. Code မပြင်သေးပါနဲ့။

Bug:
Registration form submit လုပ်တဲ့အခါ API က 422 validation error ပြန်ပါတယ်။

Context:
Frontend React app calls backend registration API.

Request:
POST /api/events/1/register

Request Body:
{
  "name": "Aung Aung",
  "phone": "",
  "email": "aung@example.com"
}

Expected:
If phone is required, frontend should show “ဖုန်းနံပါတ်ဖြည့်ပါ။” before API call.

Actual:
Frontend sends request with empty phone and backend returns 422.

Response:
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Phone is required"
  }
```

# 33. Chapter Summary

ဒီအခန်းမှာ **Bug Report Prompts** ကိုလေ့လာခဲ့တယ်။

```text
Bug report = Agent ကို root cause ရှာနိုင်အောင်ပေးတဲ့ evidence package
```

```text

Context
Environment
Steps to Reproduce
Expected Result
Actual Result
Error / Logs
Recent Changes
Allowed Files
Do-not-touch List
Debug Instructions
Output Format
Approval Gate
```

အရေးကြီးဆုံး rule—

```text
“မရဘူး” လို့မရေးပါနဲ့။
Expected / Actual / Steps / Error နဲ့ရေးပါ။
```

Debug workflow—

```text

→ root cause candidates
→ most likely cause
→ minimal fix plan
→ approval
→ fix
→ verification
→ regression test
→ git diff review
→ commit
```

Agentic Coding အတွက် takeaway—

