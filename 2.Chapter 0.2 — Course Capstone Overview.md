[ ] ဒီ course ရဲ့ capstone project က ဘာလဲ ရှင်းပြနိုင်မယ်
[ ] Event Management Platform ရဲ့ အဓိကရည်ရွယ်ချက်ကို နားလည်မယ်

[ ] Course တစ်ခုလုံးမှာ ဘယ် part က capstone project ရဲ့ ဘယ်အပိုင်းကိုတည်ဆောက်မလဲ မြင်နိုင်မယ်

## 2. Beginner Explanation

ဒီ course တစ်ခုလုံးမှာ သင်တည်ဆောက်မယ့် final project က—


Event Management Platform ဖြစ်ပါတယ်။

အလွယ်ဆုံးနားလည်မယ်ဆိုရင် Event Management Platform ဆိုတာ—


ပွဲစီစဉ်သူတွေက event တစ်ခုဖန်တီးနိုင်မယ်။
လူတွေက event ကိုကြည့်ပြီး register လုပ်နိုင်မယ်။
Register လုပ်ပြီးရင် ticket code ရမယ်။
ပွဲနေ့မှာ staff က ticket code နဲ့ attendee ကို check-in လုပ်နိုင်မယ်။
Admin/Organizer က event တွေ၊ attendee တွေကိုစီမံနိုင်မယ်။


ဆိုတဲ့ system ပါ။

ဥပမာ—

**
Myanmar Tech Meetup 2026
Startup Networking Night
AI Workshop for Beginners
Community Charity Event
Music Night
*

   လူတွေ event ကြည့်ပြီး register လုပ်တဲ့နေရာ

2. Backend API
   Data သိမ်း၊ validation လုပ်၊ ticket code ထုတ်၊ check-in logic လုပ်တဲ့နေရာ

   Code တကယ်အလုပ်လုပ်လားစစ်တဲ့ automated/manual tests

7. Agentic Workflow
   SPEC, skills, subagents, verifier, auditor, deployer နဲ့ project ကိုထိန်းတဲ့ workflow

8. Deployment
   Project ကို staging / production ပေါ်တင်တဲ့အပိုင်း

9. MCP
   AI Agent က external tools/data/workflows တွေနဲ့ချိတ်နိုင်အောင်လုပ်တဲ့အပိုင်း
*

## 4. Why Event Management Platform?

ဒီ project ကို capstone အဖြစ်ရွေးတဲ့အကြောင်းရင်း ၇ ခုရှိပါတယ်။

## 4.1 Beginner အတွက်နားလည်လွယ်တယ်

Event ဆိုတာ လူတိုင်းနားလည်တယ်။

**
ပွဲရှိတယ်
လူတွေ register လုပ်တယ်
ပွဲနေ့မှာ check-in လုပ်တယ်
Organizer က attendee စာရင်းကြည့်တယ်
*

Domain က ရှုပ်ထွေးလွန်းတာမဟုတ်လို့ beginner အတွက် စိတ်ကူးယဉ်ရလွယ်တယ်။

## 4.2 Frontend သင်ဖို့ကောင်းတယ်

Event list, event detail, registration form, ticket confirmation page တွေက frontend beginner အတွက် perfect practice ဖြစ်တယ်။

**
Card UI
List rendering
Empty state
Loading state
Error state
Form validation
Responsive design
Myanmar localization
Accessibility
*

အားလုံးလေ့ကျင့်လို့ရတယ်။

## 4.3 Backend သင်ဖို့ကောင်းတယ်

Backend API မှာ real-world logic တွေပါလာတယ်။

**
GET events
POST registration
Generate ticket code
Prevent duplicate registration
Check-in attendee
Prevent duplicate check-in
Admin CRUD
*

ဒီ logic တွေက backend foundation သင်ဖို့လုံလောက်တယ်။

## 4.4 Database သင်ဖို့ကောင်းတယ်

Event Management မှာ relational data တွေသဘာဝကျကျရှိတယ်။

**
Event တစ်ခုမှာ attendee အများကြီးရှိနိုင်တယ်
Attendee တစ်ယောက်မှာ ticket code ရှိတယ်
User တစ်ယောက်မှာ role ရှိတယ်
Admin က event တွေစီမံတယ်
Staff က check-in လုပ်တယ်
*

ဒါကြောင့် table, relation, primary key, foreign key, unique constraint, index တို့ကို သင်ဖို့ကောင်းတယ်။

## 4.5 Mobile/PWA သင်ဖို့ကောင်းတယ်

ပွဲနေ့ check-in app က mobile-first use case ပါ။

**
Staff က ဖုန်းနဲ့ attendee ရှာမယ်
Ticket code ရိုက်မယ်
Check-in button နှိပ်မယ်
Internet မကောင်းရင် offline queue ထဲသိမ်းမယ်
နောက်မှ sync လုပ်မယ်
*

ဒါကြောင့် PWA, offline storage, sync strategy ကို လက်တွေ့သင်နိုင်တယ်။

## 4.6 Security/Privacy သင်ဖို့ကောင်းတယ်

Event registration မှာ name, phone, email, ticket code လို personal data ပါလာတယ်။ Admin dashboard မှာ role-based access control လိုတယ်။

ဒီ project က security ကို theory မဟုတ်ဘဲ real feature အနေနဲ့သင်နိုင်စေတယ်။

**
Who can view attendees?
Who can create event?
Who can check in attendee?
Should phone number be logged?
What happens if someone spams registration API?
*

## 4.7 Portfolio အတွက်ကောင်းတယ်

Final project ပြီးသွားရင် သင်ပြနိုင်မယ့် demo က real-world ဆန်တယ်။

**
Visitor က event ကြည့်ပြီး register လုပ်တယ်
Ticket code ရတယ်
Admin က attendee ကို dashboard မှာမြင်တယ်
Staff က PWA app နဲ့ check-in လုပ်တယ်
Verifier report pass ဖြစ်တယ်
Auditor report ထွက်တယ်
Project ကို staging ပေါ် deploy လုပ်ထားတယ်
*

ဒါက “todo app” ထက် portfolio value ပိုကြီးပါတယ်။

## 5. User Roles

ဒီ project ထဲမှာ user role ၄ မျိုးရှိမယ်။

## 5.1 Public Visitor

Public Visitor ဆိုတာ website ကိုလာကြည့်တဲ့ ordinary user ပါ။

လုပ်နိုင်တာ—

**
- Event list ကြည့်နိုင်
- Event detail ကြည့်နိုင်
- Event registration form ဖြည့်နိုင်
- Ticket code ရနိုင်
*

မလုပ်နိုင်တာ—

**
- Admin dashboard မဝင်နိုင်
- Attendee list မကြည့်နိုင်
- Event မဖန်တီးနိုင်
- Check-in မလုပ်နိုင်
*

## 5.2 Event Staff

Staff ဆိုတာ ပွဲနေ့မှာ attendee check-in လုပ်တဲ့သူပါ။

လုပ်နိုင်တာ—

**
- Staff login ဝင်နိုင်
- Event တစ်ခုရွေးနိုင်
- Attendee ကို name / phone / ticket code နဲ့ရှာနိုင်
- Attendee ကို check-in mark လုပ်နိုင်
- Already checked-in warning မြင်နိုင်
*

မလုပ်နိုင်တာ—

**
- Event create/delete မလုပ်နိုင်
- Admin analytics မကြည့်နိုင်
- User role မပြင်နိုင်
*

## 5.3 Organizer

Organizer ဆိုတာ event ကိုစီစဉ်တဲ့သူပါ။

လုပ်နိုင်တာ—

**
- Event create/edit လုပ်နိုင်
- Attendee list ကြည့်နိုင်
- Registration status ကြည့်နိုင်
- Attendee export လုပ်နိုင်
- Staff check-in progress ကြည့်နိုင်
*

မလုပ်နိုင်တာ—

**
- System-wide admin settings မပြင်နိုင်
- တခြား organizer ရဲ့ event မပြင်နိုင်
*

## 5.4 Admin

Admin ဆိုတာ system တစ်ခုလုံးကိုစီမံတဲ့သူပါ။

လုပ်နိုင်တာ—

**
- Event အားလုံးကိုစီမံနိုင်
- User roles စီမံနိုင်
- Organizer / Staff accounts စီမံနိုင်
- System-level reports ကြည့်နိုင်
- Deployment / audit reports ကြည့်နိုင်
*

ဒီ roles တွေကြောင့် နောက်ပိုင်းမှာ **Authentication** နဲ့ **Authorization / RBAC** ကိုသင်ရပါမယ်။

git commit
