[ ] Context window ပြည့်လာရင် Agent quality ဘာကြောင့်ကျနိုင်လဲ သိမယ်

[ ] File read, terminal output, logs, rules, skills, hooks, subagents တွေ context ထဲဘယ်လိုဝင်နိုင်လဲနားလည်မယ်
[ ] Token budget ကို task အလိုက်စီမံနိုင်မယ်
[ ] Large file / large log / large diff ကို context-safe ဖြစ်အောင် summarize လုပ်နိုင်မယ်
[ ] Subagent, skills, progressive disclosure ကို context control strategy အဖြစ်နားလည်မယ်
[ ] Long session အတွက် compact summary / fresh session packet ရေးနိုင်မယ်

## 2. Beginner Explanation

AI Agent တစ်ယောက်နဲ့စကားပြောတာကို စာရွက်တစ်ရွက်ပေါ်မှာ အလုပ်လုပ်တာနဲ့နှိုင်းယှဉ်ပါ။

စာရွက်ပေါ်မှာ—

```text
သင့် prompt
Agent response
Agent ဖတ်ထားတဲ့ files
Terminal output
Error logs
CLAUDE.md / AGENTS.md rules
SPEC excerpts
Git diff
```

အားလုံးရေးထားတယ်ဆိုပါစို့။ စာရွက်အရွယ်အစားက အကန့်အသတ်ရှိတယ်။ စာတွေများလာလေလေ စာရွက်ပေါ်မှာနေရာကုန်လေလေဖြစ်တယ်။ နေရာကုန်လာတဲ့အခါ Agent က အစောပိုင်းမှာပြောထားတဲ့ rule/decision/bug detail တချို့ကို “မမြင်တော့သလို” ဖြစ်နိုင်တယ်။

ဒီ “စာရွက်အရွယ်အစား” ကို context window လို့ခေါ်တယ်။

```text
Context Window
= Agent တစ်ကြိမ်အလုပ်လုပ်တဲ့အခါ မြင်နိုင်တဲ့ working memory limit
```

Claude Code docs က context window ကို session ထဲမှာ Claude သိနေတဲ့အရာအားလုံး — instructions, files it reads, its own responses, terminal ထဲမပေါ်တဲ့ internal content တချို့ — ပါဝင်တဲ့ working memory အဖြစ်ဖော်ပြထားပါတယ်။ Session စချိန်မှာ `CLAUDE.md`, auto memory, MCP tool names, skill descriptions စတာတွေ load ဖြစ်နိုင်ပြီး Agent အလုပ်လုပ်တဲ့အခါ file reads, path-scoped rules, hooks-related context, tool outputs တွေထပ်ဝင်လာနိုင်ပါတယ်။ ([Claude][1])

အရေးကြီးဆုံး mental model—

```text
Context window = Agent ရဲ့ active working memory
Token budget = အဲဒီ memory ထဲမှာသုံးနိုင်တဲ့နေရာ
Context control = memory ထဲကို ဘာထည့်မလဲ / ဘာဖယ်မလဲ စီမံခြင်း
```

# 3. Token ဆိုတာဘာလဲ

Token ဆိုတာ AI model ကစာသားကို process လုပ်တဲ့အပိုင်းငယ်လေးတွေပါ။ Beginner အတွက် “စာလုံး/စကားလုံးအပိုင်းအစ” လိုမှတ်နိုင်တယ်။

ဥပမာ—

```text
"Registration form validation"
```

ဒီစာကြောင်းကို model က token အပိုင်းအစတွေခွဲပြီးဖတ်တယ်။

Token တွေဖြစ်လာနိုင်တာ—

```text
User prompt
Assistant responses
File contents
Terminal output
Git diff
Error logs
Rules/instructions
Tool descriptions
Skill descriptions
```

Token များလာလေလေ—

```text
context window ပိုပြည့်
cost ပိုတက်နိုင်
latency ပိုကြာနိုင်
Agent focus ကျနိုင်
old context နဲ့ new context ရောနိုင်
```

Anthropic best-practices docs က context window ပြည့်လာတာဟာ AI coding session တွေမှာ အဓိက constraint တစ်ခုဖြစ်ကြောင်း, conversation message, file reads, command output အားလုံး context ထဲဝင်နိုင်ပြီး single debugging/codebase exploration session တစ်ခုက tokens အများကြီးသုံးနိုင်ကြောင်းဖော်ပြထားပါတယ်။ Context ပြည့်လာတဲ့အခါ Agent က earlier instructions မေ့သလိုဖြစ်နိုင်ပြီး mistakes ပိုလုပ်နိုင်ပါတယ်။ ([Claude][2])

# 4. Context Window ထဲ ဘာတွေဝင်လဲ

Claude Code context visualization docs အရ session စတင်ချိန်နဲ့ Agent အလုပ်လုပ်နေချိန်မှာ context ထဲဝင်နိုင်တဲ့အရာတွေရှိတယ်။ ([Claude][1])

## 4.1 Startup Context

Session စချိန်မှာ ဝင်နိုင်တာ—

```text
System instructions
Output style
CLAUDE.md
AGENTS.md, if imported
Auto memory
MCP tool names
Skill descriptions
Project rules
```

## 4.2 Work Context

Agent အလုပ်လုပ်နေချိန်မှာ ဝင်နိုင်တာ—

```text
User messages
Agent responses
Files Agent reads
Terminal command output
Git diff output
Error logs
Tool results
Path-scoped rules, when matching files are read
Subagent summaries
```

## 4.3 Tool Context

Agent tool သုံးတာများလာရင်—

```text
Read file result
Grep/search result
Bash output
Test output
Build output
Network result
MCP tool result
```

တွေ context ထဲထပ်ဝင်နိုင်တယ်။

Course rule—

```text
Tool result တစ်ခုချင်းစီသည် context cost ရှိတယ်။
Agent ကို file/log/diff အကုန်ဖတ်ခိုင်းတာက free မဟုတ်ဘူး။
```

# 5. Context Window ပြည့်လာရင် ဘာဖြစ်နိုင်လဲ

Context window ပြည့်လာတဲ့အခါ အောက်ပါပြဿနာတွေဖြစ်နိုင်တယ်။

```text
[ ] Agent က အစောပိုင်း rule မေ့သလိုဖြစ်
[ ] Old plan နဲ့ new plan ရော
[ ] Rejected approach ကိုပြန်အသုံးပြု
[ ] Wrong file path မှတ်
[ ] Acceptance criteria တစ်ခုမေ့
[ ] Verification မလုပ်ဘဲ Done လို့ပြော
[ ] Terminal output အဟောင်းကို current error လို့ယူ
[ ] Token/cost/latency ပိုတက်
```

Context window ပြည့်လာတာက session ရပ်သွားတာမဟုတ်ပါဘူး။ Claude Code က context limit နီးလာရင် automatic compaction လုပ်နိုင်ပြီး manual `/compact` နဲ့လည်း summary ပြုလုပ်နိုင်ပါတယ်။ ဒါပေမယ့် compact summary က ဘာကိုသိမ်းပြီးဘာကိုဖယ်မလဲဆိုတာအရေးကြီးပါတယ်။ Claude Code docs က `/compact` သည် conversation ကို structured summary နဲ့အစားထိုးနိုင်ပြီး startup content တချို့ reload ဖြစ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude][1])

# 6. Context Bloat, Context Rot, Stale Context

Context control ကိုနားလည်ဖို့ term ၃ ခုသိထားပါ။

## 6.1 Context Bloat

Context ထဲမှာ မလိုတဲ့အရာတွေများလာတာ။

ဥပမာ—

```text
Full log 2000 lines
Unrelated source files
Repeated tool outputs
Old git diff
node_modules content
package-lock.json တစ်ခုလုံး
```

## 6.2 Context Rot

Long session ကြာလာပြီး context ထဲမှာ old decisions, rejected plans, outdated errors, current task မဟုတ်တဲ့ info တွေစုလာတာ။

ဥပမာ—

```text
အရင် feature plan
အခု feature scope
old error
new error
```

အကုန်ရောနေတယ်။

## 6.3 Stale Context

အရင်ကမှန်ခဲ့ပေမယ့် အခုမမှန်တော့တဲ့ context။

ဥပမာ—

```text
Old API route: /api/register
Current API route: /api/events/:id/register
```

Agent က stale route ကိုသုံးသွားနိုင်တယ်။

Course rule—

```text
Context bloat ကို summarize လုပ်ပါ။
Context rot ဖြစ်ရင် /compact သို့ fresh packet သုံးပါ။
Stale context ကို explicit “Ignore” လုပ်ပါ။
```

# 7. Context Control Strategy Overview

Context control မှာ strategy အမျိုးမျိုးရှိတယ်။

```text
1. Pack relevant context only
2. Summarize large files/logs
3. Use repo maps and file summaries
4. Use /context to inspect usage, Claude Code
5. Use /compact for long related sessions
6. Use /clear for unrelated new tasks
7. Use subagents for large exploration
8. Use skills for progressive disclosure
9. Use fresh session packets
10. Keep CLAUDE.md / AGENTS.md concise
11. Use verification commands instead of long explanations
12. Avoid reading generated folders
```

Anthropic context engineering cookbook က context သည် finite resource ဖြစ်ပြီး diminishing returns ရှိတဲ့ resource problem လို့ဖော်ပြထားပါတယ်။ High-signal tokens အနည်းဆုံးနဲ့ desired outcome ရနိုင်အောင် subagents, compaction, tool-result clearing, memory စတဲ့ strategies တွေကိုသုံးနိုင်ပါတယ်။ ([Claude Platform][3])

# 8. `/context` — Current Context Usage စစ်ခြင်း

Claude Code မှာ `/context` command က current session context usage ကို breakdown အဖြစ်ကြည့်ဖို့သုံးနိုင်ပါတယ်။ Claude Code context docs က `/context` ကို live breakdown by category နဲ့ optimization suggestions ကြည့်ရန်သုံးနိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude][1])

Use when—

```text
[ ] Session ကြာလာတယ်
[ ] Agent response quality ကျလာတယ်
[ ] Files/logs အများကြီးဖတ်ထားတယ်
[ ] Compact/clear လုပ်သင့်လားမသေချာ
[ ] Context ဘာတွေစားနေလဲသိချင်
```

Prompt—

```text
/context ကိုစစ်ပြီး context usage summary ပေးပါ။
Code မပြင်ပါနဲ့။

စစ်ရန်:
- ဘာ category က context အများဆုံးသုံးနေလဲ
- compact လုပ်သင့်လား
- clear လုပ်သင့်လား
- subagent သုံးသင့်လား
- noisy context ဘာရှိလဲ
```

# 9. `/compact` — Summary နဲ့ session ဆက်သွားခြင်း

`/compact` ဆိုတာ current conversation အဟောင်းတွေကို summary အဖြစ်ပြောင်းပြီး context window နေရာလွတ်စေတဲ့ command ပါ။

Claude Code docs က context ပြည့်လာလျှင် automatic compaction လုပ်နိုင်သော်လည်း user က `/compact focus on ...` လို focus instruction နဲ့ manual compact လုပ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ Focus instruction ပေးထားရင် summary က automatic compaction မှန်းဆတာထက် သင်ရွေးထားတဲ့အရာကိုပိုထိန်းနိုင်ပါတယ်။ ([Claude][1])

Codex CLI မှာလည်း `/compact` command သုံးပြီး long exchange ပြီးနောက် conversation so far ကို summarize လုပ်ကာ critical details သိမ်းပြီး context freeing လုပ်နိုင်ကြောင်း official slash command docs ကဖော်ပြထားပါတယ်။ ([OpenAI Developers][4])

## 9.1 Compact သုံးသင့်တဲ့အချိန်

```text
[ ] Same task ဆက်လုပ်မယ်
[ ] Session ကြာလာပြီ
[ ] Files/logs အများကြီးဖတ်ပြီးပြီ
[ ] Current goal မပြောင်းသေး
[ ] Context ထဲက important decisions သိမ်းချင်
[ ] Fresh start မလိုသေး
```

## 9.2 Compact မလုပ်ခင် focus instruction ပေးပါ

Bad—

```text
/compact
```

Better—

```text
/compact focus on Registration History feature, current files changed, accepted decisions, failing criteria, and next safe step.
```

သို့မဟုတ် manual prompt—

```text
Compact summary ရေးပါ။

Keep:
- current goal
- accepted decisions
- files changed
- acceptance criteria status
- verification results
- open bugs
- next safe step

Discard:
- old rejected plans
- repeated logs
- irrelevant file exploration
```

## 9.3 Compact ရဲ့အန္တရာယ်

Compaction က summary ဖြစ်တဲ့အတွက် detail တချို့ပျောက်နိုင်တယ်။ Anthropic engineering post မှာ compaction သည် context window contents ကို high-fidelity summary အဖြစ် distill လုပ်နိုင်ပေမယ့် အလွန် aggressive ဖြစ်ရင် နောက်မှအရေးကြီးလာနိုင်တဲ့ subtle context ပျောက်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Anthropic][5])

Course rule—

```text
Compact မလုပ်ခင် important decisions / current goal / open bugs ကို explicit ပြောပါ။
```

# 10. `/clear` — Fresh Start လုပ်ခြင်း

`/clear` ဆိုတာ old conversation context ကိုဖယ်ပြီး fresh chat စတာပါ။

Claude Code docs က unrelated work ပြောင်းတဲ့အခါ `/clear` သုံးရန်အကြံပြုထားပြီး old conversation က next task အတွက်လိုတဲ့ files/context ကို crowd out လုပ်နိုင်ပြီး message တိုင်း token cost တိုးစေနိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude][1])

Codex CLI docs မှာ `/clear` သည် terminal ကို clear လုပ်ပြီး fresh chat စတင်နိုင်ကြောင်း, `Ctrl+L` က terminal view ကိုသာ clear လုပ်ပြီး current chat ကိုမပြောင်းပေမယ့် `/clear` သည် conversation အသစ်စတာဖြစ်ကြောင်းဖော်ပြထားပါတယ်။ ([OpenAI Developers][4])

## 10.1 Clear သုံးသင့်တဲ့အချိန်

```text
[ ] Task အသစ် totally unrelated
[ ] Old session မှာ rejected plans များ
[ ] Agent repeated failed fixes
[ ] Context rot ဖြစ်နေ
[ ] Independent review လို
[ ] Commit တစ်ခုလုပ်ပြီး task အသစ်စမယ်
[ ] Project/folder ပြောင်းမယ်
```

## 10.2 Clear မလုပ်ခင် handoff summary ထုတ်ပါ

```text
Fresh session handoff summary ရေးပါ။

Include:
- current branch
- current goal
- completed work
- files changed
- verification result
- open issues
- next task
- constraints
```

ပြီးမှ—

```text
/clear
```

သို့မဟုတ် Codex မှာ—

```text
/clear
```

လုပ်ပါ။

## 10.3 Clear vs Compact

| Action       | ဘာလုပ်တာလဲ                                   | ဘယ်အချိန်သုံးမလဲ             |
| ------------ | -------------------------------------------- | ---------------------------- |
| `/compact`   | current conversation ကို summary အဖြစ်လျှော့ | same task ဆက်လုပ်မယ်         |
| `/clear`     | fresh chat စ                                 | unrelated task / context rot |
| Fresh packet | manual summary နဲ့ session အသစ်စ             | important handoff လို        |
| Subagent     | large read ကို separate context ထဲလုပ်       | main context မပွစေရန်        |

# 11. `/new`, `/fork`, `/side` — Codex Context Control

Codex CLI မှာ session control slash commands တွေရှိတယ်။ `/new` က same CLI session ထဲမှာ fresh conversation စနိုင်ပြီး `/fork` က current conversation ကို thread အသစ်အဖြစ် clone လုပ်ကာ alternative approach စမ်းနိုင်တယ်။ `/side` က parent task ကိုမပျက်စေဘဲ ephemeral side conversation ဖွင့်နိုင်ကြောင်း Codex docs ကဖော်ပြထားပါတယ်။ ([OpenAI Developers][4])

Use cases—

```text
/new
= task အသစ်စရန်

/fork
= current context ကိုထားပြီး alternative plan စမ်းရန်

/side
= main task မနှောင့်ယှက်ဘဲ quick review question မေးရန်
```

Beginner rule—

```text
Main implementation context ထဲမှာ unrelated research မထည့်ပါနဲ့။
Codex သုံးရင် /side သို့ /fork စဉ်းစားပါ။
Claude သုံးရင် subagent / fresh session စဉ်းစားပါ။
```

# 12. `/mention`, `/diff`, `/memory`, `/skills` — Context Steering

Codex CLI slash docs က `/mention` သည် specific file/folder ကို conversation ထဲ attach လုပ်နိုင်ပြီး follow-up turns မှာ အဲဒီ file ကို reference လုပ်စေကြောင်းဖော်ပြထားပါတယ်။ `/diff` သည် staged/unstaged/untracked changes ကို Git diff အဖြစ်ကြည့်နိုင်စေပြီး `/skills` သည် selected skill context ကို next request မှာထည့်နိုင်ပါတယ်။ ([OpenAI Developers][4])

Useful patterns—

```text
/mention src/RegisterPage.jsx
→ ဒီ file ကို next prompt မှာ focus လုပ်စေ

/diff
→ commit မလုပ်ခင် current changes review

/memories
→ Codex memory behavior configure, if using memories

/skills
→ task-specific skill ကိုအသုံးပြု
```

Claude Code မှာ—

```text
/context
/memory
/compact
/clear
/hooks
/permissions
```

တို့ကို context/safety control အတွက်သုံးနိုင်တယ်။

# 13. Subagents for Context Isolation

Large codebase exploration ကို main context ထဲမှာလုပ်ရင် file reads/logs များပြီး context bloat ဖြစ်နိုင်တယ်။ Subagent ကိုသုံးရင် subagent က သီးခြား context window ထဲမှာ files အများကြီးဖတ်ပြီး main conversation ဆီ summary ပဲပြန်ပေးနိုင်ပါတယ်။ Claude Code context docs က subagent က large file reads ကို main context ထဲမသွင်းဘဲ own separate context window ထဲမှာ handle လုပ်နိုင်ပြီး summary and metadata trailer ပဲပြန်လာကြောင်းဖော်ပြထားပါတယ်။ ([Claude][1])

Use when—

```text
[ ] Large folder explore လုပ်မယ်
[ ] Error log အရှည်ကြီး analyze လုပ်မယ်
[ ] Repo map ထုတ်မယ်
[ ] Security audit read-only လုပ်မယ်
[ ] Dependency impact analysis လုပ်မယ်
```

Subagent prompt—

```text
Use a read-only subagent.

Task:
Explore projects/event-management/services/api for registration-related files.

Do not edit code.
Do not read .env.

Return only:
- relevant file list
- one-line responsibility per file
- suspected root cause areas
- files main agent should read next
```

Main context ထဲမှာ raw file content မလာဘဲ summary ပဲလာတာက context control အတွက်ကောင်းတယ်။

# 14. Skills & Progressive Disclosure

Skills တွေက context window ကိုမပွအောင်သုံးနိုင်တဲ့ strategy တစ်ခုပါ။ Skill metadata — name, description, file path — ကိုပဲအရင် context ထဲမှာသိမ်းပြီး full `SKILL.md` instructions ကို skill သုံးဖို့လိုတဲ့အချိန်မှ load လုပ်တာကို progressive disclosure လို့ခေါ်တယ်။ Codex skills docs က skills use progressive disclosure: Codex starts with each skill’s name, description, and file path, then loads full `SKILL.md` only when it decides to use that skill လို့ဖော်ပြထားပါတယ်။ ([OpenAI Developers][6])

Why useful?

```text
CLAUDE.md / AGENTS.md ထဲမှာ procedure အရှည်ကြီးမထည့်ရ
Task-specific knowledge ကိုလိုအပ်မှ load
Context window မပွ
Reusable workflow ဖြစ်
```

Example—

```text
AGENTS.md:
Run tests before Done.

Test Skill:
Detailed testing workflow, commands, fixtures, output format.
```

AGENTS.md ထဲ test workflow အရှည်ကြီးမထည့်ဘဲ skill ထဲထားတာ context control အတွက်ပိုကောင်းတယ်။

Course rule—

```text
Always-needed rules → AGENTS.md / CLAUDE.md
Task-specific procedure → Skill
Frequently used prompt → Command
Large exploration → Subagent
Hard guardrail → Permission/Hook
```

# 15. Tool Result Clearing / Context Editing Concept

API-level agents မှာ tool results များလာရင် tool-result clearing သုံးနိုင်တယ်။ Anthropic context editing docs က tool result clearing သည် old tool results — file contents/search results လို re-fetchable results — ကို context မှဖယ်ပြီး placeholder ထားနိုင်ကြောင်းဖော်ပြထားပါတယ်။ Context editing သည် context ကို actively curate လုပ်ဖို့ဖြစ်ပြီး irrelevant content က model focus ကို degrade လုပ်နိုင်ကြောင်း docs ကရှင်းပြထားပါတယ်။ ([Claude Platform][7])

ဒီ course မှာ API-level context editing ကို code အဖြစ်မလုပ်သေးပါဘူး။ Beginner mental model အနေနဲ့သိထားရမယ်။

```text
Old tool output = re-fetchable
Important decision = keep
Raw log = summarize
File content = read again if needed
```

# 16. Memory vs Context Window

Memory နဲ့ context window မရောပါနဲ့။

## Context Window

```text
Active session working memory
Session ထဲမှာ currently visible
Fills up
Compact/clear လုပ်နိုင်
```

## Memory

```text
Persistent notes / rules / project facts
Session အပြင်မှာသိမ်း
CLAUDE.md, AGENTS.md, auto memory, NOTES.md
Next session မှာပြန် load/use လုပ်နိုင်
```

Anthropic cookbook က memory ကို structured note-taking အဖြစ်ဖော်ပြပြီး agent က persistent external storage ထဲ notes ရေးထားခြင်းကြောင့် active context window ထဲမှာအရာအားလုံးမထားဘဲ long-running tasks ကိုဆက်နိုင်ကြောင်းရှင်းပြထားပါတယ်။ ([Claude Platform][3])

Course rule—

```text
Current task detail → context
Long-term project rule → memory
Completed work summary → lab note / NOTES.md
Sensitive secret → never memory/context
```

# 17. Context Hygiene Rules

Context hygiene ဆိုတာ context ကိုသန့်ရှင်းအောင်ထိန်းတဲ့ habit ပါ။

```text
Rule 1 — Task အသစ်စချိန် context packet သုံးပါ
Rule 2 — Unrelated task ပြောင်းရင် /clear
Rule 3 — Same task long session ဖြစ်ရင် /compact focus
Rule 4 — Large file/log ကို full paste မလုပ်ပါ
Rule 5 — Error log first meaningful error ပဲပေးပါ
Rule 6 — Secret/PII redact လုပ်ပါ
Rule 7 — Old rejected plans ကို context ထဲမထည့်ပါ
Rule 8 — Agent response quality ကျလာရင် /context စစ်ပါ
Rule 9 — Large exploration ကို subagent ခိုင်းပါ
Rule 10 — Repeatable workflows ကို skills/commands ထဲထားပါ
```

# 26. Common Errors

## Error 1 — Context window ကြီးတာက အမြဲကောင်းတယ်ထင်ခြင်း

Context ကြီးတာက အမြဲပိုကောင်းတာမဟုတ်ပါ။ Irrelevant context က Agent focus ကိုချနိုင်တယ်။ Anthropic context editing docs က context သည် finite resource with diminishing returns ဖြစ်ပြီး irrelevant content သည် model focus ကို degrade လုပ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([Claude Platform][7])

## Error 2 — `/compact` ကို focus မပေးဘဲသုံးခြင်း

Focus မပေးရင် important details ပျောက်နိုင်တယ်။

Fix—

```text
/compact focus on [current goal, decisions, files changed, open bug, next step]
```

## Error 3 — `/clear` မသုံးဘဲ unrelated tasks ဆက်လုပ်ခြင်း

Old task context က new task ကိုလမ်းလွဲစေနိုင်တယ်။

Fix—

```text
Task unrelated ဖြစ်ရင် handoff summary → /clear.
```

## Error 4 — Raw log အရှည်ကြီး paste လုပ်ခြင်း

Fix—

```text
First meaningful error + relevant lines only.
```

## Error 5 — Full package-lock / node_modules read ခိုင်းခြင်း

Fix—

```text
package diff summary only.
Never read node_modules.
```

## Error 6 — Skills/procedures အားလုံး CLAUDE.md ထဲထည့်ခြင်း

Fix—

```text
Always-needed rules in CLAUDE.md.
Detailed procedure in skill/command.
```

# 30. Chapter Summary

ဒီအခန်းမှာ **Context Window & Token Control** ကိုလေ့လာခဲ့တယ်။

အရေးကြီးဆုံး mental model—

```text
Context window = Agent ရဲ့ active working memory
Token budget = အဲဒီ memory ထဲမှာသုံးနိုင်တဲ့နေရာ
Context control = ဘာကိုထည့်မလဲ၊ ဘာကို summarize/clear လုပ်မလဲ စီမံခြင်း
```

Context window ထဲဝင်နိုင်တာ—

```text
prompts
responses
CLAUDE.md / AGENTS.md
files read
terminal output
error logs
git diff
tool results
skill descriptions
subagent summaries
```

အရေးကြီးဆုံး tools / strategies—

```text
/context
= current context usage inspect

/compact
= same task ဆက်လုပ်ရန် summary ဖြင့် context လျှော့

/clear
= unrelated task / context rot အတွက် fresh start

Subagents
= large exploration ကို separate context ထဲလုပ်

Skills
= detailed procedure ကိုလိုအပ်မှ load

Fresh Session Packet
= new session အတွက် source of truth packet
```

အရေးကြီးဆုံး rule—

```text
Context များတာက အမြဲကောင်းတာမဟုတ်ပါ။
High-signal context ကိုပဲထားပါ။
Noisy context ကို summarize လုပ်ပါ။
Dangerous context ကို redact လုပ်ပါ။
Stale context ကိုဖယ်ပါ။
```

Agentic Coding အတွက် takeaway—

```text
Prompt က Agent ကိုဘာလုပ်ရမလဲပြောတယ်။
Context window က Agent ဘာကိုမြင်နေလဲသတ်မှတ်တယ်။
Agent quality ကိုမြှင့်ဖို့ prompt ကိုသာမက context window ကိုလည်းစီမံရမယ်။
```
