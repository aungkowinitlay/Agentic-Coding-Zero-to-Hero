## 2. Beginner Explanation

React component ဆိုတာ UI အပိုင်းတစ်ခုကို ကိုယ်ပိုင်နာမည်ပေးထားတဲ့ JavaScript function ပါ။

ဥပမာ HTML မှာ ဒီလိုရေးနိုင်တယ်။

```html
<article class="event-card">
  <h2>AI Workshop</h2>
  <p>Yangon</p>
</article>
```

React မှာ ဒီ UI ကို component အဖြစ်ပြောင်းနိုင်တယ်။

```jsx
function EventCard() {
  return (
    <article className="event-card">
      <h2>AI Workshop</h2>
      <p>Yangon</p>
    </article>
  );
}
```

ပြီးရင် App ထဲမှာ ဒီလိုသုံးနိုင်တယ်။

```jsx
<EventCard />
```

အဓိက mental model—

```text
Component = UI အပိုင်းတစ်ခု
App = components တွေကိုပေါင်းထားတဲ့ screen
Props = parent component က child component ဆီပေးတဲ့ data
```

React official docs မှာ React သည် markup, CSS, JavaScript ကို custom “components” ဆိုတဲ့ reusable UI elements တွေအဖြစ်ပေါင်းစည်းနိုင်စေပြီး, components တွေကို HTML tags လို compose/nest လုပ်ကာ pages/apps တည်ဆောက်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ React component သည် JavaScript function ဖြစ်ပြီး markup ကို return လုပ်တယ်လို့လည်း docs ကရှင်းပြထားပါတယ်။ ([React][1])

# 3. Component ရဲ့ Rule ၃ ခု

React component ရေးတဲ့အခါ beginner အတွက် အရေးကြီးဆုံး rule ၃ ခုရှိတယ်။

```text
Rule 1 — Component name သည် Capital Letter နဲ့စရမယ်
Rule 2 — Component သည် JSX return လုပ်ရမယ်
Rule 3 — Component ကို JSX tag အဖြစ်သုံးနိုင်တယ်
```

## 3.1 Capital Letter

ကောင်း—

```jsx
function EventCard() {
  return <article>AI Workshop</article>;
}
```

မကောင်း—

```jsx
function eventCard() {
  return <article>AI Workshop</article>;
}
```

React docs recap ထဲမှာ React components တွေသည် regular JavaScript functions ဖြစ်ပေမယ့် component names တွေက capital letter နဲ့စရပြီး JSX markup ကို return လုပ်ရကြောင်းဖော်ပြထားပါတယ်။ ([React][1])

## 3.2 JSX Return

Component က UI ကို return လုပ်ရမယ်။

```jsx
function PageHeader() {
  return (
    <header>
      <h1>Event Management</h1>
      <p>Frontend foundation</p>
    </header>
  );
}
```

## 3.3 JSX Tag အဖြစ်သုံးခြင်း

```jsx
function App() {
  return (
    <main>
      <PageHeader />
    </main>
  );
}
```

# 4. Parent Component / Child Component

Component တစ်ခုက အခြား component ကိုခေါ်သုံးနိုင်တယ်။

```jsx
function PageHeader() {
  return <h1>React Components</h1>;
}

function App() {
  return (
    <main>
      <PageHeader />
    </main>
  );
}
```

ဒီမှာ—

```text
App = parent component
PageHeader = child component
```

Component tree mental model—

```text
App
├── PageHeader
├── EventList
│   ├── EventCard
│   ├── EventCard
│   └── EventCard
└── Footer
```

React docs မှာ components တွေကို compose, order, nest လုပ်ပြီး pages တည်ဆောက်နိုင်ကြောင်း, project ကြီးလာတဲ့အခါ reusable components တွေကိုပြန်သုံးနိုင်တာကြောင့် development ပိုမြန်စေကြောင်းဖော်ပြထားပါတယ်။ ([React][1])

# 5. Props ဆိုတာဘာလဲ

Props ဆိုတာ parent component က child component ဆီ data ပေးတဲ့နည်းပါ။

ဥပမာ—

```jsx
function EventCard({ title, location }) {
  return (
    <article>
      <h2>{title}</h2>
      <p>{location}</p>
    </article>
  );
}

function App() {
  return (
    <main>
      <EventCard title="AI Workshop" location="Yangon" />
      <EventCard title="React Meetup" location="Mandalay" />
    </main>
  );
}
```

ဒီမှာ—

```text
title = prop
location = prop
EventCard = props ကိုလက်ခံတဲ့ child component
App = props ပေးတဲ့ parent component
```

React official docs မှာ props ကို JSX attributes လို component ဆီပေးနိုင်ပြီး child component ထဲမှာ `function Avatar({ person, size })` လို destructuring syntax နဲ့ဖတ်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ Props တွေဟာ read-only snapshots ဖြစ်ပြီး child component က props ကိုပြောင်းမရေးသင့်ကြောင်းလည်း React docs ကရှင်းပြထားပါတယ်။ ([React][2])

# 6. JSX ထဲ JavaScript သုံးခြင်း

JSX ထဲမှာ JavaScript variable/value တွေကို `{}` နဲ့ထည့်ပြနိုင်တယ်။

```jsx
function EventCard({ event }) {
  const remainingSeats = event.capacity - event.registeredCount;

  return (
    <article>
      <h2>{event.title}</h2>
      <p>ကျန်ရှိသောနေရာ: {remainingSeats}</p>
    </article>
  );
}
```

JSX ထဲမှာ string attribute ဖြစ်ရင် quotes သုံးတယ်။

```jsx
<p className="eyebrow">Frontend Foundation</p>
```

JavaScript variable ကိုသုံးမယ်ဆိုရင် braces သုံးတယ်။

```jsx
<h2>{event.title}</h2>
```

React docs မှာ JSX သည် JavaScript file ထဲမှာ HTML-like markup ရေးနိုင်တဲ့ syntax ဖြစ်ပြီး dynamic property သို့ JavaScript expression ကို JSX ထဲထည့်ချင်ရင် curly braces `{}` သုံးနိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([React][3])

# 7. Conditional Rendering

Conditional rendering ဆိုတာ condition အပေါ်မူတည်ပြီး UI တစ်ခုကိုပြမလား၊ မပြဘူးလား ဆုံးဖြတ်တာပါ။

ဥပမာ event တစ်ခုမှာ seat ပြည့်နေပြီဆိုရင် button disabled လုပ်မယ်။

```jsx
function RegisterButton({ isFull }) {
  if (isFull) {
    return (
      <button type="button" disabled>
        Seats ပြည့်ပါပြီ
      </button>
    );
  }

  return <button type="button">Register လုပ်မည်</button>;
}
```

Ternary သုံးနိုင်တယ်။

```jsx
<p>{isFull ? "Seats ပြည့်ပါပြီ" : "Register လုပ်နိုင်သည်"}</p>
```

`&&` သုံးနိုင်တယ်။

```jsx
{remainingSeats <= 10 && <p>နေရာအနည်းငယ်သာကျန်ပါသည်။</p>}
```

React docs မှာ React ထဲ branching logic ကို JavaScript နဲ့ထိန်းနိုင်ပြီး `if`, ternary operator `{cond ? <A /> : <B />}`, logical AND `{cond && <A />}` စတဲ့နည်းလမ်းတွေသုံးနိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([React][4])

# 8. Lists Rendering

Event list တစ်ခုကို data array နဲ့ render လုပ်မယ်။

```jsx
const events = [
  {
    id: "evt-ai-workshop",
    title: "AI Workshop",
    location: "Yangon"
  },
  {
    id: "evt-react-meetup",
    title: "React Meetup",
    location: "Mandalay"
  }
];
```

Render—

```jsx
function EventList({ events }) {
  return (
    <section>
      {events.map((event) => (
        <EventCard key={event.id} event={event} />
      ))}
    </section>
  );
}
```

ဒီမှာ—

```text
events.map(...)
= event တစ်ခုချင်းစီကို EventCard component အဖြစ်ပြောင်း

key={event.id}
= React ကို list item တစ်ခုချင်းစီကိုခွဲသိစေတဲ့ unique key
```

React docs မှာ array data ကို `map()` နဲ့ JSX nodes/components အဖြစ်ပြောင်းပြီး list render လုပ်နိုင်ကြောင်း, `filter()` နဲ့ specific items ကိုရွေးနိုင်ကြောင်းဖော်ပြထားပါတယ်။ JSX elements တွေကို `map()` ထဲမှာ render လုပ်တဲ့အခါ unique `key` လိုအပ်ပြီး key က React ကို array item တစ်ခုချင်းစီကို match လုပ်နိုင်စေကြောင်း docs ကရှင်းပြထားပါတယ်။ ([React][5])

# 9. Component Extraction Strategy

Beginner အတွက် component ခွဲတဲ့အခါ “အစကတည်းက file အများကြီးမခွဲ” ပါနဲ့။

အကောင်းဆုံး workflow—

```text
Step 1 — App.jsx ထဲမှာ component functions ခွဲရေး
Step 2 — Browser မှာအလုပ်လုပ်ကြောင်းစစ်
Step 3 — Build pass ဖြစ်ကြောင်းစစ်
Step 4 — Component ပိုကြီးလာမှ separate files ခွဲ
```

React docs မှာ components တွေ nesting များလာတဲ့အခါ different files တွေခွဲတာက files တွေ scan လုပ်ရလွယ်ပြီး components တွေကိုနေရာအများကြီးမှာ reuse လုပ်ရလွယ်စေကြောင်းဖော်ပြထားပါတယ်။ ([React][6])

Course rule—

```text
Beginner stage:
components in App.jsx first

Builder stage:
src/components/ folder ထဲခွဲ

Production stage:
feature folders + shared components
```

# 11. Replace `src/App.jsx`

Chapter 33 lab project ထဲက—

```text
playground/chapter-33-react-vite-starter/react-vite-starter/src/App.jsx
```

ကို အောက်ပါ code နဲ့ replace လုပ်ပါ။

```jsx
import "./App.css";

const events = [
  {
    id: "evt-ai-workshop",
    title: "AI Workshop",
    date: "2026-08-01",
    location: "Yangon",
    category: "AI",
    capacity: 100,
    registeredCount: 25
  },
  {
    id: "evt-react-meetup",
    title: "React Meetup",
    date: "2026-08-15",
    location: "Mandalay",
    category: "Frontend",
    capacity: 80,
    registeredCount: 72
  },
  {
    id: "evt-pwa-checkin",
    title: "PWA Check-in Lab",
    date: "2026-09-05",
    location: "Online",
    category: "Mobile Web",
    capacity: 60,
    registeredCount: 60
  }
];

function App() {
  const totalCapacity = events.reduce((total, event) => total + event.capacity, 0);
  const totalRegistered = events.reduce(
    (total, event) => total + event.registeredCount,
    0
  );
  const openEvents = events.filter((event) => event.registeredCount < event.capacity);

  return (
    <main className="page">
      <PageHeader
        eyebrow="Part 6 — Frontend Foundation"
        title="React Components"
        description="Event Management project အတွက် UI ကို reusable components တွေအဖြစ်ခွဲရေးပါမယ်။"
      />

      <StatsSummary
        totalEvents={events.length}
        totalCapacity={totalCapacity}
        totalRegistered={totalRegistered}
      />

      <EventList events={events} openEventCount={openEvents.length} />
    </main>
  );
}

function PageHeader({ eyebrow, title, description }) {
  return (
    <section className="hero">
      <p className="eyebrow">{eyebrow}</p>
      <h1>{title}</h1>
      <p>{description}</p>
    </section>
  );
}

function StatsSummary({ totalEvents, totalCapacity, totalRegistered }) {
  const remainingSeats = totalCapacity - totalRegistered;

  return (
    <section className="stats" aria-label="Event summary">
      <StatCard label="Events" value={totalEvents} />
      <StatCard label="Registered" value={totalRegistered} />
      <StatCard label="Remaining Seats" value={remainingSeats} />
    </section>
  );
}

function StatCard({ label, value }) {
  return (
    <article className="stat-card">
      <p>{label}</p>
      <strong>{value}</strong>
    </article>
  );
}

function EventList({ events, openEventCount }) {
  if (events.length === 0) {
    return <EmptyState message="Event မရှိသေးပါ။" />;
  }

  return (
    <section className="event-section" aria-labelledby="events-heading">
      <div className="section-header">
        <div>
          <p className="eyebrow">Upcoming Events</p>
          <h2 id="events-heading">လာမည့် Event များ</h2>
        </div>

        <p className="section-note">
          Register လုပ်နိုင်သော event {openEventCount} ခု ရှိသည်။
        </p>
      </div>

      <div className="event-grid">
        {events.map((event) => (
          <EventCard key={event.id} event={event} />
        ))}
      </div>
    </section>
  );
}

function EventCard({ event }) {
  const remainingSeats = event.capacity - event.registeredCount;
  const isFull = remainingSeats === 0;

  return (
    <article className="event-card" aria-labelledby={`${event.id}-title`}>
      <div className="event-card-header">
        <div>
          <p className="event-category">{event.category}</p>
          <h3 id={`${event.id}-title`}>{event.title}</h3>
        </div>

        <StatusBadge isFull={isFull} remainingSeats={remainingSeats} />
      </div>

      <dl className="event-meta">
        <EventMetaItem label="ရက်စွဲ" value={event.date} />
        <EventMetaItem label="နေရာ" value={event.location} />
        <EventMetaItem label="ကျန်ရှိသောနေရာ" value={remainingSeats} />
      </dl>

      <PrimaryButton disabled={isFull}>
        {isFull ? "Seats ပြည့်ပါပြီ" : "Register လုပ်မည်"}
      </PrimaryButton>
    </article>
  );
}

function EventMetaItem({ label, value }) {
  return (
    <div>
      <dt>{label}</dt>
      <dd>{value}</dd>
    </div>
  );
}

function StatusBadge({ isFull, remainingSeats }) {
  if (isFull) {
    return <span className="status-badge status-badge-full">Full</span>;
  }

  if (remainingSeats <= 10) {
    return <span className="status-badge status-badge-warning">Few seats</span>;
  }

  return <span className="status-badge status-badge-open">Open</span>;
}

function PrimaryButton({ disabled = false, children }) {
  return (
    <button className="primary-button" type="button" disabled={disabled}>
      {children}
    </button>
  );
}

function EmptyState({ message }) {
  return (
    <section className="empty-state">
      <p>{message}</p>
    </section>
  );
}

export default App;
```

# 12. ဒီ App.jsx ထဲမှာ ဘာတွေသင်နေလဲ

## 12.1 Data ကို array အဖြစ်ထားတယ်

```jsx
const events = [
  { id: "evt-ai-workshop", title: "AI Workshop", ... },
  ...
];
```

# 18. Props Design Rules

Props ရေးတဲ့အခါ—

```text
Good:
<EventCard event={event} />

Good:
<EventMetaItem label="ရက်စွဲ" value={event.date} />

Too many props:
<EventCard title={} date={} location={} category={} capacity={} registeredCount={} />
```

EventCard အတွက် `event` object တစ်ခုလုံးပေးတာက ဒီအဆင့်မှာပိုရှင်းတယ်။ Small component ဖြစ်တဲ့ `EventMetaItem` အတွက် `label`, `value` တိတိကျကျပေးတာကောင်းတယ်။

Props design mental model—

```text
Page-level component
= data object/array ပေးနိုင်

Small display component
= specific props ပေး
```

Props ကို child ထဲမှာပြောင်းမရေးပါနဲ့။

မကောင်း—

```jsx
function EventCard({ event }) {
  event.title = "New title";
}
```

ကောင်း—

```jsx
function EventCard({ event }) {
  const remainingSeats = event.capacity - event.registeredCount;
}
```

Props docs မှာ props တွေသည် read-only snapshots ဖြစ်ပြီး component က props ကိုပြောင်းမရေးသင့်ကြောင်းဖော်ပြထားပါတယ်။ ([React][2])

# 19. Common Errors

## Error 1 — Component name lowercase

မကောင်း—

```jsx
function eventCard() {
  return <article />;
}
```

ကောင်း—

```jsx
function EventCard() {
  return <article />;
}
```

## Error 2 — Props destructuring မမှန်

မကောင်း—

```jsx
function EventCard(event) {
  return <h2>{event.title}</h2>;
}
```

ဒီ code မှာ `event` သည် props object တစ်ခုလုံးဖြစ်နေတယ်။

ကောင်း—

```jsx
function EventCard({ event }) {
  return <h2>{event.title}</h2>;
}
```

သို့မဟုတ်—

```jsx
function EventCard({ title }) {
  return <h2>{title}</h2>;
}
```

## Error 3 — map() ထဲ key မထည့်

မကောင်း—

```jsx
{events.map((event) => (
  <EventCard event={event} />
))}
```

ကောင်း—

```jsx
{events.map((event) => (
  <EventCard key={event.id} event={event} />
))}
```

React docs မှာ `map()` call ထဲရှိ JSX elements တွေမှာ keys လိုအပ်ပြီး key မရှိရင် console warning တက်နိုင်ကြောင်းဖော်ပြထားပါတယ်။ ([React][5])

## Error 4 — Arrow function block body မှာ return မမေ့ပါနဲ့

မကောင်း—

```jsx
{events.map((event) => {
  <EventCard key={event.id} event={event} />;
})}
```

ကောင်း—

```jsx
{events.map((event) => {
  return <EventCard key={event.id} event={event} />;
})}
```

သို့မဟုတ်—

```jsx
{events.map((event) => (
  <EventCard key={event.id} event={event} />
))}
```

React rendering lists docs မှာ arrow function မှာ `{}` block body သုံးရင် explicit `return` ရေးရမယ်, မရေးရင်ဘာမှ return မဖြစ်နိုင်ကြောင်းသတိပေးထားပါတယ်။ ([React][5])

## Error 5 — class instead of className

မကောင်း—

```jsx
<section class="hero">
```

ကောင်း—

```jsx
<section className="hero">
```

## Error 6 — isFull တွက်ချက်မှုမှား

မကောင်း—

```jsx
const isFull = event.capacity === 0;
```

ကောင်း—

```jsx
const remainingSeats = event.capacity - event.registeredCount;
const isFull = remainingSeats === 0;
```

# 26. Common Beginner Questions

## 26.1 Component တစ်ခုကို ဘယ်အချိန်ခွဲမလဲ?


ခွဲဖို့မလိုသေးတဲ့အချိန်—

```text

props ပိုရှုပ်သွား
```

## 26.2 Props object တစ်ခုလုံးပေးသင့်လား, field တစ်ခုချင်းစီပေးသင့်လား?

Beginner rule—

```text
Big display component → object တစ်ခုလုံးပေးနိုင်
Small reusable component → specific props ပေး
```

Example—

```jsx

```

## 26.3 key ကိုဘာကြောင့် id နဲ့သုံးရလဲ?

React က list item တစ်ခုချင်းစီကိုခွဲသိဖို့ key လိုတယ်။ Data ထဲမှာ unique id ရှိရင် id ကို key အဖြစ်သုံးပါ။ React docs မှာ keys ဟာ array item တစ်ခုနဲ့ component ကို match လုပ်ဖို့ကူညီပြီး items insert/delete/reorder ဖြစ်နိုင်တဲ့အခါအရေးကြီးကြောင်းဖော်ပြထားပါတယ်။ ([React][5])

## 26.4 Separate file ခွဲတာကောင်းလား?

အမြဲမဟုတ်ပါ။ Beginner အတွက် App.jsx ထဲ component functions ခွဲရေးပြီး အလုပ်လုပ်ကြောင်းသေချာမှ separate files ခွဲတာပိုကောင်းတယ်။

# 29. Chapter Summary

ဒီအခန်းမှာ **React Components** ကိုလေ့လာခဲ့တယ်။

အရေးကြီးဆုံး mental model—

```text
React component = reusable UI piece written as a JavaScript function returning JSX
```

React component rules—

```text
Component name သည် Capital Letter နဲ့စ
Component သည် JSX return လုပ်
Component ကို <ComponentName /> အဖြစ်သုံး
Props ကို parent က child ဆီပေး
Props ကို child က read-only အဖြစ်သုံး
List render လုပ်ရင် stable key ထည့်
```

ဒီအခန်းမှာ Chapter 33 starter app ကို—

```text
PageHeader
StatsSummary
StatCard
EventList
EventCard
EventMetaItem
StatusBadge
PrimaryButton
EmptyState
```

ဆိုတဲ့ reusable components တွေအဖြစ်ခွဲခဲ့တယ်။

React official docs အရ UI ကို buttons, text, images စတဲ့ small units အဖြစ်ခွဲနိုင်ပြီး React က အဲဒီ units တွေကို reusable/nestable components အဖြစ်ပေါင်းစည်းခွင့်ပေးတယ်။ Component တွေက regular JavaScript functions ဖြစ်ပြီး names ကို capital letter နဲ့စကာ JSX markup ကို return လုပ်ရတယ်။ ([React][7])
